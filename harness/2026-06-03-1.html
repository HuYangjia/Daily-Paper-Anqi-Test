一句话核心贡献
把"大规模技能选择"从 embedding 相似度匹配提升为类型化有向图上的结构检索——以 propose-then-commit 协议让图在执行中累积可信结构。
摘要（中文翻译，忠实原文）

当 LLM agent 采用大型技能库时，挑选合适的子集成为结构问题，而非相似度匹配问题：技能之间彼此依赖、冲突、特化或重复，这种结构既看不见于全量枚举，也看不见于 embedding 相似度。

我们提出 SkillDAG，把 skill 间关系建模为类型化有向图，并对 LLM agent 暴露为推理时可调用的结构检索接口——它在执行过程中被查询和演化，而不是预先固定到检索管道里。每次查询都返回三类信号：向量匹配、类型化邻居（typed-edge neighbors）、冲突信号（conflict signals）；同时通过 propose-then-commit 协议，agent 可以注册由"实际执行后果"支撑的边，使图结构跨多个 episode 不断累积。

在 ALFWorld 与 SkillsBench / MiniMax-M2.7 上，SkillDAG 取得 67.1% 成功率 / 27.3% reward，比已知最强 Graph-of-Skills 基线高 +12.8 / +8.6 pp；该优势可迁移到 gpt-5.2-codex；本征 SkillsBench Ret@K 在匹配 query 下 65.5 → 78.2。这些增益可追溯到两条独立机制：(i) 候选排序在 pool 扩 10× 时仍鲁棒，而固定的 seeding-diffusion 管道会退化；(ii) 集合单调的在线编辑扩大了 ground-truth recall，且不会驱逐已命中条目。

核心内容解读（结构问题立论 / 类型化 DAG IR / propose-then-commit / 鲁棒性来源）

问题再框架：从相似度到结构。过去一年（2025-2026）"agent skill library"是 harness 工程的高速增长方向：MetaGPT skill stash、SkillSmith、AgentFloor 都把"把会做的事情存起来留给下次"作为基本叙事。但当 skill 数从几十涨到上千时，embedding 相似度就开始失效——本文给出了三种典型失败：(a) 依赖（dependency） skill A 必须先做 B 才能跑，但 embedding 上 A、B 远离；(b) 冲突（conflict） A 与 C 互相覆盖输出，相似度反而很高；(c) 特化与重复（specialize/duplicate） A 与 D 几乎是同一件事，但 D 多了个边界条件。这三类关系都不是相似度可观测，只能靠"执行后果"显式记录——这就是 SkillDAG 立论的核心。

类型化有向图 IR。SkillDAG 形式化为四元组 G = (V, E, T, σ)：节点 V 是技能；边 E 是有向关系；类型 T = {DEPENDS_ON, CONFLICTS_WITH, SPECIALIZES, DUPLICATES, OVERRIDES …}；σ 是边的可信度（来自历次执行回放）。关键设计：图本身不是离线一次性构造的，而是 agent 推理时可调用的"检索 + 编辑"接口（agent-callable structural retrieval interface），与今日 #1 序列里 5-26 SkillOpt、5-25 Code as Agent Harness 的"code-as-IR"立论遥相呼应。

检索原语：每次返回 3 路信号。当 agent 用 query q 检索时，SkillDAG 同时输出：

信号语义下游用法
向量匹配 (vector matches)语义近邻 top-k 候选常规相似度 baseline，与历史 RAG 兼容
类型化邻居 (typed-edge neighbors)沿 DEPENDS_ON / SPECIALIZES 等边扩展补全先决条件、找到更精准的特化版本
冲突信号 (conflict signals)沿 CONFLICTS_WITH / OVERRIDES 边告警排除会互相打架的组合，避免回路

这意味着 agent 一次查询不是拿到"top-k similar"列表，而是拿到一个"带健康检查的 frontier"——这是与传统 RAG 最根本的区别。

Propose-then-commit 协议（图自演化的关键）。新边不是 LLM 自由提议就能加入，而要走两步：(1) propose：agent 在执行时产生候选边，附带前提与后果；(2) commit：必须有"execution-backed evidence"（执行结果与预期一致），才会被持久化进图。这点关键——它把"幻觉新增的虚假关系"挡在图外，且天然带上 provenance（每条边都能追到一次具体执行）。这与 5-19 ANNEAL（FDKA + governed 符号补丁）的"有审计的图编辑"是同一审稿人偏好——结构修改要有外部凭证。

鲁棒性的两个来源。论文显式分析了 SkillDAG 增益的"原因"：(i) 候选排序对池子规模的鲁棒性——把 skill pool 扩大 10× 时，固定 seeding-diffusion 管道（即"先选种子再传播"的传统图检索）会退化，而 SkillDAG 的 typed-edge expansion 不退化，因为它始终沿"语义有意义"的边走；(ii) 集合单调的在线编辑——commit 一条新边只会扩大 ground-truth recall，不会驱逐已命中条目（这与 LRU/类 LRU cache 截然相反），意味着 SkillDAG 是纯增益的演化结构。

与昨日 5-28 #1 的连接。5-28 #1 提出的 agent runtime layer 四原语 observe / score / predict / act，SkillDAG 恰好是 predict 与 act 之间的共享 IR——下一个 skill 不再是"下一个最像的 token"，而是"沿类型化边可达的下一个 frontier"。把两者合起来看，是过去 2 周 harness 方向"把 IR 从 token / vector 抬升到 typed graph"的连续演进。

  • Pool 扩 10× 鲁棒——固定 seeding-diffusion 管道在此条件下退化，SkillDAG 候选排序保持稳定。
  • 集合单调编辑——commit 新边只扩大 ground-truth recall，不驱逐先前命中——纯增益累积。
  • 查询返回 3 路信号——vector matches + typed-edge neighbors + conflict signals 一次给齐，告别"top-k similar"单维信号。

Harness Engineering（直击毕设）

SkillDAG 与你毕设"稀疏注意力 KV-cache 预取 + 双水位线 GPU 热缓存"在 IR 抽象上有可借鉴的位移：把"下一段最可能命中的 token 块"从相似度（embedding）抬到类型化有向图——这正是预取问题的二阶版本。建议：在毕设 Discussion 引入 SkillDAG 的"typed-edge expansion"作为对照组，论证你的 prefetcher 也是把"相似性预取"升级为"结构性预取"——agent role / session_id / system prompt 是天然的类型化边。这与 5-28 #1 提出的 predict 原语共同构成"结构性 prefetch"路线。同时 propose-then-commit 协议给"预取候选必须由真实命中验证"提供了直接借鉴——你毕设 Q.md 关于"预取与缓存数据一致性"的争议，可以引用此协议作为 reference 形式化解。

Agent Skills Safety

类型化边里 CONFLICTS_WITH、OVERRIDES 是天然的"safety 拒绝信号"——SafeHarbor / SkillSmith / SKILL.md 的"不安全 skill 不能调用"约束，可以一一映射成 SkillDAG 上的 typed edges。这给"policy-as-graph"提供了可执行的载体：拒绝不再是 prompt 字符串，而是可被检索的结构化反例。结合今日方向 #2 的 compliance bias 与三类 abstention gap，SkillDAG 的 conflict signals 恰好可以作为 specification gap / verification gap / authority gap 的结构化承载体——这是把"该不该做"问题图结构化的钥匙。

Safety Benchmark

SkillDAG 暴露的本征评测维度 Ret@K 是一个"检索系统正确性"的可独立测量量——这与 5-28 #1 的命中率 / TTFT / 吞吐三联可以拼成"性能 + 行为"双维 benchmark。建议：未来 agent benchmark 应在每个任务上同时报告 task-success 与 retrieval-correctness（沿真实 typed-edge 的命中率）——这能区分"偶然走对"与"结构上正确"。这与今日 #3 的"benchmark 污染检测可靠性差距"形成对仗——前者要求 benchmark 自身是 typed graph，后者要求 benchmark 数据来源是 verifiable provenance。
