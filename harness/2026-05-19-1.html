摘要（中文翻译，忠实原文）

基于 LLM 的 agent 能从单次执行错误中恢复，但当底层流程知识——operator 模式、前置条件、约束未被修补时，它们会在同一故障上反复跌倒。现有 self-evolving 方法通过更新 prompt、记忆或模型权重来缩小这个 gap，但没有任何一个直接修补"任务如何执行"的符号结构，且很少能给出安全部署所需的治理保证。

我们提出 ANNEAL——一个神经符号 agent，不动 foundation model 权重，把反复出现的失败转化为对 process knowledge graph 的可治理符号编辑。其核心机制 Failure-Driven Knowledge Acquisition (FDKA) 做三件事：（1）定位责任 operator；（2）通过约束式 LLM 生成类型化补丁；（3）在 commit 之前用多维评分 + 符号护栏 + canary 测试验证提案。每一笔被接受的编辑都附带完整 provenance 与确定性回滚能力。

跨 4 个域 × 27 多 seed 跑次，ANNEAL 是唯一能 commit 持久结构性修补的系统——ReAct 与 Reflexion 这类强基线虽然单次恢复良好，但在 holdout 复发故障上仍保留 72%–100% 失败率，而 ANNEAL 在受测的 recurring-failure 设置下把这一比例降为 0%。消融显示：移除 FDKA，结构性修补全部消失，成功率最多下滑 26.7 个百分点。

核心内容解读（背景·方法·差异）

解决了什么问题（why now）：过去三周 harness 主线已经把"评测面"做到很细——5-13 declarative semantics（Evaluation Differential）、5-14 prospective metacognition（TRIAGE）、5-15 negative ablation（ChromaFlow）、5-18 recovery ladder（CAX-Agent）。但 CAX-Agent 的"修错梯子"依然是事件驱动的局部恢复：每一次失败按 rule→model→context→human 的强度爬一格，恢复完就回到原状。ANNEAL 的关键转变在于——不修任务、不修 prompt、不修权重，而是修"任务如何被执行"的那张图。它把"修错"做成了一笔可审计的、永久写回到符号知识库的提交，下次相同故障根本不会再发生。这是 harness 工程从"运行时控制"向"运行时学习"的一次结构升级。

核心方法：FDKA = 定位 + 合成 + 治理

  • 定位 responsible operator：从失败 trace 反推到流程知识图上的某个节点（operator）——这相当于把 SRE 的"root-cause attribution"形式化为图上的归因运算。
  • 约束式 LLM 生成类型化补丁：让 LLM 在schema 受限的语法下生成"对该 operator 的 precondition / constraint 的 typed edit"——而不是放任 LLM 自由改图。这一步把 LLM 当成"补丁建议者"，但禁止它直接落盘。
  • 三道治理闸门：（a）multi-dimensional scoring——补丁要在多维上拿到阈值；（b）symbolic guardrails——硬约束检查（类型/前置/不变量）；（c）canary testing——在小流量上跑通才能 commit。三道全过才允许写入 KG。
  • provenance + 确定性回滚：每条 commit 携带可追溯的 provenance（谁、何时、为何、基于哪些证据），并支持确定性回滚——这把 ANNEAL 的"自我演化"做成了 git-like 的有状态历史，而不是模糊的"我学了一些东西"。

核心数据亮点 / 关键论点
  • 4 域 × 27 多 seed · 复发故障率 0%：ANNEAL 是唯一在 holdout recurring-failure setting 下把残留失败率压到 0% 的系统——对照 ReAct/Reflexion 72–100%，差距不是几个 pp，而是定性差异。
  • 消融：去 FDKA 成功率 ↓ 26.7 pp——结构性修补全部消失。说明"修补流程图"是 ANNEAL 的不可移除核心，不是装饰功能。
  • 三闸门 + canary：补丁不是"LLM 改完就走"，而是被多维评分 + 符号 guardrail + canary 三道审核——这是把 LLM-as-author 与 verifier 解耦的工程化设计。
  • typed edit + provenance + 回滚：每笔 commit 可追溯、可回退——这是 self-evolving agent 第一次有 git-like 的状态历史，而不是黑盒"自我提升"。
  • 不动权重：foundation model 完全冻结，所有"学习"发生在符号层——这意味着学习成本与模型规模解耦，且能跨 backbone 迁移。
  • 方法学外推：FDKA 的"locate → propose → verify → commit"四段式可直接迁到 KV-cache predictor、prefetch policy、tool-routing rule、scheduler hint——只要这些"运行时决策"能被表述为图节点，就能套同一份治理。
对你三个研究方向的启发
Harness Engineering（直击毕设）

ANNEAL 给了你 KV-cache offload 工程一个"持久学习层"的样本。当前你的 prefetch + Top-K 是无状态规则 + 学习预测器——它在每个 step 都重做决策，没有把"过去踩过的坑"沉淀下来。可以把整个 offload runtime 看成一张图：节点是 layer/operator + cache state，边是"在该状态下应该 prefetch 哪些 K 行"。当某条 layer 序列反复触发 H2D 阻塞，就让一个轻量 LLM/policy 在受 schema 约束的 patch language 里写一条 typed rule（如"在 layer L 命中率 < 0.6 且 prefetch_k=8192 时，将该 layer 的 prefetch 提前到 layer L-2"），经 canary（小批 step）通过后写回 policy KG。这把毕设的 ablation 从"单条曲线"升级为"可累积的 policy commit 历史"——每一次"延迟降低"都对应一次可审计的图编辑，而不是某轮 grid search 的随机命中。

Agent Skills Safety

三道闸门（multi-dim score → symbolic guardrail → canary）是 agent skill 安全审核的范本。你之前关注的 SKILL.md 供应链攻击（5-13 报告）正是缺少 typed edit + canary 的反面教材：技能注册表允许"自由文本即生效"。把 ANNEAL 的契约搬过去——任何 skill 注册或更新都必须是 typed edit、过 symbolic guardrail、跑 canary——可以一次性把 86%/77.6%/100% 的三阶段攻击成功率打回到接近 0。这是 self-evolving 与 self-installing skill 都该共享的最小治理契约。

Safety Benchmark

对 benchmark 的启示：当前所有 agent benchmark 都只测"开机即战"的 raw capability，没有测"复发故障率"。ANNEAL 给了一个新坐标——同一类故障重复注入 N 次后，agent 的失败率曲线。把这个坐标加到 SaaS-Bench / TOBench / ClawForge 等任务上，相当于在 benchmark 里测"agent 学不学得乖"。这个指标特别能区分"看似强的 ReAct/Reflexion"与"真有持久学习的 ANNEAL 级系统"——值得作为 next-gen agent benchmark 的一档新维度。
