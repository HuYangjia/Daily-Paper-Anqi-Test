摘要（中文翻译，忠实原文）

工业广告推荐模型依靠架构演化 持续改进——RankMixer、TokenMixer-Large、MixFormer 这些升级都说明更好的结构 仍是质量与业务收益的关键来源。但要在生产系统里开发这种升级，极度依赖专家、难以规模化。已有的自动化手段都不够：AutoML 只是调超参，而真正有效的收益往往需要在严格约束下做跨模块改动；通用 LLM 编码 agent 只优化"能跑"，但能跑不等于这是一个合法的推荐架构——候选可能在本地测试里看似通过，却在线上造成沉默失败（silent failure）让性能恶化。

我们提出 NOVA：一个 level-aware 的 agent harness，用于验证感知 的架构演化。NOVA 使用 architecture gradient——一个 SGD 启发、非可微 的更新信号，聚合之前的修改、验证诊断、度量反馈与轨迹记忆，引导下一步修改。verification cascade 检查结构语义、本地可执行性、离线有效性与线上影响；非法候选被早期拦截，失败模式被记录为禁止方向。L1–L4 任务级控制让自动化匹配任务复杂度与风险，把高风险任务路由到 Copilot 接受人工监督。部署在工业广告系统里，NOVA 在 L2 ScaleUp 与 L3 Literature-to-Production 任务上分别拿到最高 54.5% / 60.0% 的有效通过率，相比 coding-agent 基线显著降低 silent failure，单次 literature-to-production 循环的人工时间缩短 13× 以上。线上 A/B 中，被选中的 L3 候选在三条 pCVR 目标上分别带来 +1.25%、+1.70%、+2.02% GMV 提升，pCVR 偏差同时减少 58.8%、66.7%、37.3%。

核心内容解读（架构梯度 / 验证级联 / L1–L4 / silent failure 反例）

动机：silent failure 才是工业 harness 的真正敌人。2024–2026 H1 的 agent harness 论文链条一直在强调"能跑通"——OpenHands、SWE-agent、ReActAgent 都是基于 sandbox runnable 来判定成功。但 NOVA 团队直接指出：在工业广告推荐 这种已经被人类工程师反复打磨过的高敏感系统里，"能跑的代码" 完全可能结构上不是一个合法的推荐架构——它能通过本地 unit test，但在 AUC / GMV / pCVR Bias 上表现塌方，而由于本地测试通过，失败信号被吞掉了。这就是 silent failure。已有 coding agent baseline 在 L2 ScaleUp 任务上的 SFR 高达 72.7%（Optuna-TPE）/ 80.0%（OpenHands），意味着过 4 个候选才有一个真正有效。

架构梯度：非可微的 SGD-like 更新信号。NOVA 的核心抽象是 architecture gradient——把 harness 的每一次"下一步修改的方向" 形式化为一个聚合函数 gt = Grad(et−1, Vt, ΔJt, Ht)，四个输入分别是上一次的修改、验证诊断、离线度量变化、历史轨迹记忆。这个梯度不是数值梯度，而是 LLM 在 reasoning 中产出的三类结构化信息：(1) weak components—— G/φ/F 中可能瓶颈的部分；(2) modification directions—— 下一步候选修改方向；(3) forbidden directions—— 应避免的失败模式。架构状态 At = (Gt, φt, Ft) 包含模型图、结构超参、特征配置；更新规则 At+1 = Apply(At, et*)；离线目标 Joffline = AUC，线上目标 Jonline = Σ wi · mi（mi ∈ {GMV, Bias}）。这是 harness 工程链条第一次把"下一步该怎么改" 写成数学符号——它把 6-15 HarnessX 的可演化 / 6-25 Interplay 的训练联动推进到形式化的迭代算子。

验证级联：四级 gate 同时是梯度去噪源。NOVA 的另一个关键设计是四级 verification cascade：

阶段检查内容作用
1. Structure-semantic gate修改逻辑 / shape & dtype / feature-to-token / attention 方向 / mask 语义 / logit 融合训练前捕捉可运行但结构错误 的候选
2. Local testing gate单机可执行性离线训练前拦截工程性失败
3. Offline inner loopAUC（阈值 ΔAUC > 0.001）线上部署前评估
4. Online outer loopGMV / pCVR Bias生产环境最终验证

关键在于：被拒绝的候选其失败模式 不会被丢掉，而是写入轨迹记忆 H，作为禁止方向 反馈到下一轮架构梯度——"验证即梯度去噪"。这把 6-25 Hacker-Fixer Loop 的"verifier 加固 benchmark" 视角进一步推进：verifier 不再只是审计员，而是harness 内循环里的梯度源。

L1–L4 任务级控制：把自动化匹配到风险层级。NOVA 把架构演化任务分为四级：L1 原子结构调优（如 RankMixer 层数 → AutoRun）；L2 约束感知 ScaleUp（联合缩放耦合变量、FLOPs/参数预算 → AutoRun）；L3 文献到生产迁移（把 TokenMixer-Large / AttentionRes / MixFormer / OneTrans 论文模块适配生产模型 → AutoRun 或 Copilot）；L4 开放式创新（基于趋势 / 业务需求 / 历史证据提出新结构 → Copilot）。模式（AutoRun vs Copilot）由 skill-specification 覆盖度决定，而非仅由级别决定——这是 6-23 AOHP 中"OS 级 secure information flow" 思想在组织层 的对偶：把"谁有权决定下一步" 抽象成可配置的路由策略。

关键数据 ① L2 ScaleUp。NOVA EPR 54.5%（vs Optuna-TPE 4.7%，OpenHands 6.7%，ReActAgent 12.5%，Human Expert Loop 49.3%）；LPR 99.0%、SFR 45.5%。NOVA 是第一个超越人类专家 的 baseline。

关键数据 ② L3 Literature-to-Production。NOVA EPR 60.0%（vs Human Expert 31.1%，OpenHands 10.2%，ReActAgent 7.1%）；LPR 86.7%、SFR 30.8%。单次周期的人工时间 从 54u 降到 4u（13.5×），端到端 wall-clock 从 82u 降到 37u（2.2×）。这是 harness 论文第一次拿出带 wall-clock 数字 的工业证据。

关键数据 ③ 线上 A/B。NOVA 选出的 L3 TokenMixer-Large 迁移候选在 3 条 pCVR 目标上：GMV +1.25 / +1.70 / +2.02%；pCVR Bias −58.8 / −66.7 / −37.3%（双指标同向改善 = 校准与业务收益没有 trade-off）。这是 harness 工程链条第一次把线上 GMV 写进 abstract。

消融实验 — 每个模块都必要。(a) w/o Paper Reproduction：EPR 60.0 → 33.3%（SFR 30.8 → 63.6%）；(b) w/o Solution Design：60.0 → 18.2%；(c) w/o Multi-Candidate：60.0 → 25.9%；(d) w/o Quality Assessment：60.0 → 21.9%；(e) w/o Architecture-Gradient Feedback：60.0 → 37.5%。最后一行直接证明 architecture gradient 是 NOVA 的"梯度去噪核心"—— 去掉它 EPR 折半。

位置：把 harness 工程的 "四联拍" 扩成 "五联拍"。6-23 AOHP 给了 OS 降维，6-23 AgentLens 给了表征下沉，6-23 Benchmark Illusion 给了评测升维，6-25 Interplay 给了训练联动。NOVA 给的是验证联动—— 把 harness 内循环里的 verifier 从纯 gate 升级为梯度信号源，并第一次在工业 5% 流量 A/B 中证明 harness 工程可以直接带来 +1–2% GMV。这意味着 H2 的 harness 工程线条将进入"L1–L4 任务级路由 × verification-as-gradient × 跨业务复用" 的新阶段。

对你三个研究方向的启发
Harness Engineering（直击毕设）

NOVA 是 harness 工程链条里第一篇把工业 GMV 写进 abstract 的论文，对你 KV-cache / SGLang 系统毕设的启发极其具体：(1) architecture gradient 思想 可直接迁移到 KV-cache 策略搜索—— eviction policy / placement policy / prefetch heuristic 的下一步修改可以形式化为 g = Grad(prev-config, profile-diagnostics, perf-delta, traj-memory)，从而避开 RL 设计奖励的工程难题；(2) verification cascade 四级 直接对偶 KV-cache 系统的"语义检查（cache 一致性）→ 单机性能 → 端到端 throughput → 在线 SLA" 四级，每一级失败模式都应该回写为下一轮策略搜索的禁止方向——这是 SGLang KV-cache 调优论文中明显缺失的方法学；(3) silent failure 概念—— KV-cache 系统也有 silent failure（cache 命中但 sparse pattern 与算子不兼容、显存峰值超过 SLA、attention numerical error 不被 logit 监控捕捉），"能跑 ≠ 合法 KV-cache 系统" 是你毕设的天然角度；(4) L1–L4 任务级路由—— KV-cache 系统的 patch / kernel rewrite / kernel new arch / new memory hierarchy 也可以分四级，AutoRun 跑前两级、Copilot 处理后两级；(5) 统一基础模型对照—— NOVA 把所有方法都跑在 Claude 4.6 上，意味着你做 KV-cache benchmark 时也应固定 LLM，让差异只来自 cache 系统。

Agent Skills Safety

NOVA 的 verification-as-gradient 思想对 agent safety 有直接启发：(1) safety 不应该只是"refuse / pass" 二值 gate，而应该把 unsafe trajectory 的失败模式作为"下一轮训练 / 对齐的禁止方向"——这与今日 #2 AgentDoG 1.5 的 influence-function purification 形成姊妹方法（两种"用失败数据回写训练信号" 的范式）；(2) silent failure 在 safety 里对应"looks-safe but actually unsafe" 模式（如 6-23 AgentLens 的 covert tool-use、6-25 SkillSafetyBench 的 skill-mediated 攻击）—— NOVA 提示我们 safety 也需要四级 verifier cascade（语义安全 → 工具调用合规 → 长程后果 → 真实业务影响）；(3) L1–L4 任务级路由—— 安全任务也应该按风险分级：低风险 AutoRun（refuse-only），高风险 Copilot（HITL）；(4) 架构梯度 思想可平移到 safety policy gradient——用 LLM 推理出"下一条 safety rule 应该补在哪里"，把 safety alignment 从纯 RLHF / SFT 推进到带 verifier feedback 的迭代算子。

Safety Benchmark

NOVA 把 6-25 Hacker-Fixer Loop 的"verifier 加固 benchmark" 视角推进到验证级联——这给 safety benchmark 设计带来三条新原则：(1) 从单一 ASR / EPR 升级到多级验证级联结果矩阵——一个候选不只是 pass/fail，而是哪一级失败（语义 / 可执行 / 离线 / 在线），从而能定位失败根因；(2) silent failure rate (SFR) 是 benchmark 设计的关键被忽视指标——它衡量"local pass 但 effective fail" 的占比，远比 ASR 更能反映 benchmark gaming 风险；(3) 线上 A/B 作为 outer verifier—— Safety benchmark 应该至少配置一个真实部署度量（不只是 sandbox metric），让 benchmark gaming 在最外层被 silent failure 检测出来；(4) L1–L4 风险分级—— Safety benchmark 也应该按任务风险分级，让评估器知道哪些任务必须由人审计；(5) 与 6-23 Benchmark Illusion 的多变量评测 思想 mutually reinforcing—— NOVA 提供了"多变量 + 多级" 的实证模板。
