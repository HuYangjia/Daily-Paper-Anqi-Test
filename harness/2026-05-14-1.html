一句话核心贡献

摘要（中文翻译，忠实原文）

把语言模型部署为 autonomous agent，所要求的远不止"每个任务的准确率"：当 agent 面对一个排队的问题流、并被限制在一个有限 token 预算内时，它必须在执行反馈出现之前就决定：要做哪些、以什么顺序、以及给每一题分多少算力。这正是人类认知科学研究了几十年的 prospective metacognitive control（前瞻性元认知控制），但语言模型究竟有没有这种能力，至今没被测过。

我们提出 TRIAGE——一个新的评测框架：模型收到一个任务池、再加上一个"按它自身基线消耗校准过的 token 预算"，然后一次性 commit 一个有序计划，这个计划同时编码 selection（挑哪些）、sequencing（顺序）和 per-problem allocation（每题分配多少 token）。这个计划在一个知道每题 solvability 与代价的 oracle 下被打分，得到一个共同度量尺度上的 triage efficiency ratio。

我们在竞赛数学、研究生科学、代码生成、专家多学科知识四个域评测前沿 / 开源模型，并在开/关 reasoning 两个设定下测量：当前的语言模型在前瞻性元认知控制上有显著差距，揭示出一条此前未被度量的能力维度——它对"资源高效的 agent 部署"有直接含义。

核心内容解读（背景·方法·差异）

解决了什么问题（why now）：过去 12 天 harness 主线的"外化决策面"路线已经把 endpoint（Token Arena）、scaffolding（More Is Not Always Better）、能力 ladder（AgentFloor）、工具拓扑（ComplexMCP）和声明语义（Evaluation Differential）依次显式化。所有这些工作都默认一个事——任务是逐题独立的、token 预算是无限的。但真实部署里，agent 拿到的是一队列问题、一个有限预算，必须 在执行前 决定怎么分配。TRIAGE 是首次把这条隐含假设搬到台面，把"前瞻性元认知"作为第七个 harness 维度立起来——前面六维都还是逐题层面的。

核心方法（task pool + self-calibrated budget + triage plan + oracle scoring）：

  • Self-calibrated budget：预算 B 不是任意设的，而是用模型自己在该任务上的基线 token 消耗校准——避免"小模型被锤、大模型轻松"的 budget 偏置，使得跨模型可比。这是把 Token Arena 的"成本即评测变量"思路推进了一步。
  • Triage plan = (S, π, a)：模型 commit 一个三元组——选中子集 S ⊆ task pool、子集上的执行顺序 π、每题 token 配额 a_i。关键约束：必须在看到任何执行反馈前一次性给出，因此真正在测"前瞻"。
  • Oracle 对齐：oracle 知道每个问题对该模型的 solvability 与最优代价，它生成的计划是该模型可达的全局最优解——而非"理论最优"，避免拿超模型能力的题去苛责。
  • Triage efficiency ratio：模型计划下的期望解题数 / oracle 的期望解题数，归一到 [0,1] 公共尺度——使得"小模型在小预算下"和"前沿模型在大预算下"可以同台对比。
  • 四个评估域：竞赛数学（如 MATH/AIME 类）、研究生科学（GPQA-类）、代码生成（HumanEval/MBPP-类）、专家多学科（MMLU-Pro-类）。这四个域的 solvability/代价分布完全不同，故能拆出"前瞻性元认知是否跨域稳定"。
  • 双模式：with / without reasoning：把 reasoning enable 当作受控变量，量化 chain-of-thought 是否改善前瞻性元认知——这与"reasoning 改善 retrospective 但不一定改善 prospective" 的认知科学假设直接对照。

核心数据亮点 / 关键论点
  • 三合一约束首次落地：selection / sequencing / per-problem allocation 历史上一直被分别测，TRIAGE 把它们绑成一个不可分割的 commitment。这意味着从今往后讨论"agent 在真实部署的资源效率"必须报这三件事的联合分布，而不是分别报准确率与平均开销。
  • baseline-calibrated budget：用模型自己的基线 token 消耗校准 B——把"评测公平性"从手工调参变成结构化机制。这是过去 budget-aware harness（如 Token Arena）一直缺的一块。
  • oracle-grounded 评分：所有计划相对于该模型可达的全局最优 plan 打分而非理论最优——避免"模型力所不及的题"被错误归到 metacognition gap。这是 sound 的可比尺度。
  • 显著差距：摘要明确报告"current language models exhibit substantial gaps in prospective metacognitive control"——这意味着 reasoning enable 之后，仍有相当部分 efficiency loss 来自计划阶段而非执行阶段。
  • 四域设计 → 跨域稳定性：竞赛数学、研究生科学、代码、多学科四域的 solvability/代价分布差异巨大；若 efficiency ratio 在四域上一致退化，则证明 metacognition 缺陷是"架构级"而非"领域级"的——这是评判该 harness 影响力的关键经验问题。
  • 方法学定位：在 evaluation 工程已成熟到能"外化几乎所有变量"之后，下一步是把决策作为评测对象。TRIAGE 是 harness 第一次把"前瞻"做成可度量量。
对你三个研究方向的启发
Harness Engineering

TRIAGE 给"资源约束 harness"做了一次清晰的奠基。对你正在做的 ZipperQuant W4A4 e2e 实验和 KV-Cache offloading 毕设来说，这是非常直接可借的范式：把"给定固定 KV-budget，让模型自己决定保留哪些层 / 哪些 token / 各分多少 cache"做成 budget-calibrated harness——这不就是 prefetch / evict 决策的前瞻性元认知版本吗？同时论文 v3 的分层画像 + Coverage 实验也可加一节"前瞻 vs 回顾"对照：让模型在看到完整序列前 commit 一个 Top-K plan，再对照 oracle Top-K，得出 layer-wise prospective metacognitive ratio——这比单独报 recall 更能解释"为什么有的层稳定、有的层漂移"。

Agent Skills Safety

TRIAGE 与今日 Agent Safety 选定的 HAM³ 形成自然对位：前者要求 agent 在执行前合理分配资源，后者证明在执行前 agent 的多模态/通信/推理三层都能被攻破。把两者合成一个红队设定——在 metacognitive plan 阶段注入对抗 perturbation——可以测出 agent "前瞻性元认知"对扰动的脆弱性。这与昨日 SKILL.md 攻击 也可拼接：metacognitive 阶段恰是 skill registry 被查询的位置，plan 时可被诱导去 commit 高额预算到恶意 skill。

Safety Benchmark

TRIAGE 给整个 safety benchmark 社区提了一个迟早要面对的命题：所有的 safety/agent benchmark 都是逐题独立的，没有 budget。但真实 deployment 里 safety policy 往往是预算受限的（safety filter 也是 token 成本）。借 TRIAGE 思路可设计一类 budget-aware safety benchmark：在 token-cap 下让 agent 同时挑哪些任务做、哪些拒答、哪些升级——把"safety 与 efficiency"做成 joint trade-off。今日 Benchmark 选定的 BenchJack 揭露的 reward-hacking 多与"无成本约束的 score"绑定；TRIAGE 的预算机制恰是反 reward-hack 的一种结构化补丁。
