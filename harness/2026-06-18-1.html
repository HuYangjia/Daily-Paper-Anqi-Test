摘要（中文翻译，忠实原文）

AI 系统正在自动化越来越多的科研工作流，但是把先验证据 / 生成的想法 / 实验 / 最终主张连接起来的推理常常仍然隐藏在模型的 inference 内部。本文引入 Xcientist，一个把研究合成与实验验证外化为可审视、合约治理过程的research harness。Xcientist 把文献证据、想法状态、实施计划、消融记录、修复轨迹组织成持久的研究 artifact，使生成的机制可以被基于证据地落地、执行、测试与修订，而不丢失其证据基础。我们识别出 claim drift——一种自动化研究的失效模式，可执行 artifact 不再支撑最初声称的机制——并在 training-free 记忆系统、图结构交通预测和多尺度物理信息神经网络三个领域上演示 Xcientist 保留了从问题表述到机制设计、验证与有界修订的可追溯轨迹。这些结果表明：AI scientist 不应仅以最终 artifact 评估，还应以"合成与验证过程是否仍然可归因、可审视、有科学问责性"评估。

核心内容解读（三层架构 / Claim Drift / 三大案例 / 9 维 audit）

立论：AI Scientist 必须按"过程的可问责性" 评估，不仅按最终 artifact。2025 H2 至 2026 H1 涌现的 AI Scientist 系统（AI-Scientist-v2 [5]、DeepScientist [15]、AI-Researcher [16]、InternAgent-1.5 [18]、EvoScientist [7]、AlphaEvolve [20] 等）共同问题：把"科研推理" 全部塞回 model inference。Xcientist 团队首次把这个问题形式化命名为 Claim Drift——可执行的 artifact 还在，但保留的证据已不足以将观察到的结果归因于最初声称的机制。论文给出三种漂移类型：语义漂移（操作虽实现但只是浅层文本/时间更新，如 memory case）；实验漂移（可执行系统不再测试声称的干预，如 graph forecasting case）；机制漂移（数值收益无法归因于声称组件，缺少 controls 与 diagnostics，如 PINN case）。这把 6-12 #2 Containment Gap"主流框架的遏制原则不合规" 和 6-14 #1 ToolSense"知识—检索分离" 的姿态延展到科研全链路。

系统设计一：三层架构（§2 / Fig. 3）。Layer 1 Paper Graph Infrastructure：50,000 篇 Semantic Scholar 索引论文，用 MinerU [30] 全文结构化解析 + 三轮约束式抽取（ideation_resource / graph_data / 引用对齐），最终图含~72,000 核心方法节点 + ~250,000 基线节点 + ~63,000 数据集节点 + ~1,150,000 类型化边。回答"一个 claim 可以从哪里获得支持"。Layer 2 Research Harness（核心贡献层）：4 模块——文献综述（DeepSurvey [31] 子流程）、想法生成（5 taste modes + Memory-Guided MCTS + Idea Fusion）、实验验证（三大原则 validator supremacy / separation of concerns / workspace encapsulation）、报告写作（write-audit-repair 循环）；以 ideation–validation–evolution loop 串联。Layer 3 System User Interface：topic-centric workflow + 三 lane（review / ideation / experiment）+ snapshot-stream 状态模型。

系统设计二：5 种 idea taste mode + Memory-Guided MCTS。每个想法状态包含 problem framing / core mechanism / supporting components / expected advantages / risks / preliminary validation plan 6 个字段；5 种先验取向——high-risk exploration / cross-domain transfer / conservative engineering / ambitious-but-implementable / evidence-first。MCTS 在 4 类编辑算子上搜索（adding / removing / replacing / rewiring 模块 + 引入验证协议），评估器接收 9 维评估信号：novelty / surprise / impact / feasibility / clarity / alignment / complexity / risk / protocol quality。Idea Fusion 阶段每个 taste mode 选最佳候选，host idea 主导论点，其它候选组件仅在支持论点 / 强化验证 / 修复弱点且不创造冲突 novelty claim 时保留——并显式记录被选/被拒组件、冲突解决决策、最小验证计划。

系统设计三：实验验证三大原则 + 收敛判定。准备阶段产出 prepare_target_inventory.json（已验证 repos / dataset 文件 / model identifiers / env vars / benchmark entrypoints）+ prepare_idea.md（canonical component list）。执行阶段写代码到 project/，为每个 canonical component 暴露 ablation switch，以 full-chain smoke test 收尾。标准 science：baseline + full-method 写到 standard_results_dir。消融 science：每次禁用 1 个 canonical component。收敛判定："所有必需阶段携带 validator-backed pass status + 无阻塞问题 + 完整 component-to-evidence 映射存在"。报告阶段触发 9 维 audit：content quality / engineering depth / source fidelity / research integrity / E-E-A-T signals / citation readiness——其中 source fidelity 要求每个提到的 function 名、参数值、文件路径必须匹配实际实现/工件；research integrity 要求被引论文必须存在于 workspace 且支持声称——这两条直接是 Claim Drift 检测的工程化落地。

三大案例的实证（§3.2 / 3.3 / 3.4）。(1) Training-free memory system：基线 A-Mem [24] textual F1=0.306 / token=2844.1；MCTS + 修复后 v4.2 final F1=0.391 / token=1017.2（token 长度 ↓64.2%）。(2) Graph spatio-temporal forecasting（PEMS-BAY）：从 iter 0 UCA-PCSA avg MAE=1.723（attention 完全替换 diffusion 严重 regression）→ iter 4 final 引入 innovation coverage cell α = ⟨a, e⟩/(⟨a, a⟩+λ)，avg MAE=1.556 / horizon-12 MAE=1.908；iter 3 DORA-GWNet 经消融发现"functionally inert"——硬正交投影移除后 clean MAE 仅变化 <0.12%——直接演示了 mechanism drift 检测能力。(3) Multi-scale PINN：6 版迭代，平均排名 v1→6.33 / v6→2.33；v6 final 用 fixed complement contract + Fourier-enriched coordinates 取代自适应 backbone-conditioned 投影，heat1d_multiscale 0.0672 ± 0.0118，pinnacle_heat 0.4307 ± 0.0139（heat2d_multiscale 仍逊于 MMPINN [27] 与 MultiscalePINNs [28] 强外部基线）。三案例展现"traceable trajectory + bounded revision"在不同科研类型上的一致性。

方法学姿态：harness 概念跨过 agent runtime 进入科研生命周期。把 6-12 HarnessBridge / 6-15 HarnessX / 6-14 ToolSense / 6-17 SEAGym 路线上的"harness 是类型化代数对象 / 是可学习模块 / 是诊断工具 / 是 5 视图评测面"延展到一个新维度：harness 还是 AI Scientist 的"合约式研究全链路"。在 Table 1 中，Xcientist 与 8 个对比 AI Scientist 系统在 6 维（Structured Knowledge / Structured Idea Evolution / Contracted Validation / Evidence-driven Repair / Component Attribution / Claim-boundary Audit）全部为显式系统级机制。Limitation：评估为案例式（case-based），未建立通用自主发现能力；未来方向是claim-drift rate 与 attribution completeness 等量化过程指标——这两个未来指标本身就是 6-17 SEAGym 5 视图协议的科研版自然延伸。

对你三个研究方向的启发
Harness Engineering（直击毕设）

Xcientist 把 harness 的边界从"agent 运行时"延展到"科研生命周期"——这意味着"harness"作为一个研究 primitive 已经具备从 agent 工程到 AI Scientist 工程的同构。对你的 KV-cache 系统层毕设的启发是：(1) 借鉴 合约治理——把 cache 系统每个组件（eviction policy / placement policy / prefetcher）外化为 contract（例如"本 policy 在某 workload 上需将 hit-rate 提高 X% 以上 + 延迟降低 Y% 以下"），并且每个 contract 必须由独立 validator 检查；(2) 借鉴 Claim Drift 检测——cache 工程论文的常见失败正是"语义/实验/机制漂移"（声称是 prefetcher 起作用但实际只是 buffer size 变了）；(3) 借鉴 9 维 audit——把 cache 改进从"final hit-rate" 升级为多维评估（含 source fidelity / research integrity）；(4) 借鉴 paper graph——你的相关工作部分可基于 paper graph 自动生成 baseline / dataset 对齐表；(5) 借鉴 MCTS + taste mode——cache 策略空间搜索引入 5 种 taste mode（保守工程 / 高风险 / 跨域迁移 / 雄心实现 / 证据优先）后，搜索过程本身 也成为论文产出的一部分。毕设级建议：把 KV-cache 改进做成"论文级 case study"挂载到 Xcientist——既是工程毕设也是 AI Scientist 的可复用 case。

Agent Skills Safety

Xcientist 的"研究 harness" 与今日方向 #2 SciRisk-Bench"科研安全风险维度" 形成"研究行为—研究风险" 双轴对偶——前者关心"研究是否可信"，后者关心"研究是否安全"。统一启发：未来 AI Scientist 必须同时承担科学问责（Xcientist 的 9 维 audit + Claim Drift 检测）与安全问责（SciRisk-Bench 的 10 风险维度 / 31 子学科 / safety omission / dual-use / regulatory blind spot）。Xcientist 的 contract-governed validator 可被直接扩展为 safety contract——每个生成的 idea / experiment / claim 在通过科学问责后，再通过 safety reflection（参见今日方向 #2 SRP 思想）+ risk-dimension audit 才能提交。"research integrity audit" 在 safety 上对应"moral / lawful / non-dual-use audit"——这给 6-09 ELK 不可能定理 / 6-14 "Did you lie?" / 6-17 Rift 三件套之外补上一条 科研场域 safety contract 路径。Xcientist 列出的 workspace encapsulation（每个实验声明 paths / permissions / datasets / env vars / config）天然就是 sandbox 安全边界——把它从"科研可复现" 扩展到"科研可遏制" 即可。

Safety Benchmark

Xcientist 的三大案例（A-Mem 记忆系统 / GraphWaveNet 时空预测 / multi-scale PINN）形成"同一 harness 跨学科 case study" 范式——这与今日方向 #3 SciRisk-Bench"跨 7 学科 31 子学科 10 风险维度" 异曲同工。统一启发：未来 benchmark 设计应同时具备 (a) 跨学科覆盖（SciRisk-Bench 7+31 / Xcientist 3 案例）；(b) 风险/失效维度交叉（SciRisk-Bench 10 维 / Xcientist 9 维 audit + Claim Drift 三类）；(c) 过程级问责（Xcientist contract-governed / SciRisk-Bench LLM-as-judge with risk taxonomy）；(d) 真实科研工件（Xcientist 持久 artifacts / SciRisk-Bench 政策法规标准抽取）。Xcientist 提议的未来过程指标（claim-drift rate / attribution completeness）是 benchmark 方向最直接的"下一代评测面" 候选——把它与 6-17 SEAGym 5 视图（train/val/ID/OOD/replay）对齐，可形成"科研问责 5 视图"：claim 起源 / claim 依据 / claim 跨域迁移 / claim OOD 验证 / claim 历史回放。这把 benchmark 从"能力衡量器" 推向"科学问责审计器"。
