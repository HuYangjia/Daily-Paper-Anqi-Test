摘要（中文翻译，忠实原文）

LLM agent 已经在代码库、浏览器、操作系统、日历、文件系统和工具生态上实际工作，但用来评测它们的 benchmark 是碎片化的：每一个都强调一种不同的"度量单位"——最终任务成功率、tool-call 合法性、重复-pass 一致性、轨迹安全性、或攻击稳健性。2024–2025 年的一批工作已经开始达成共识：对部署级 agent 来说，单列 accuracy 不再是合适的比较单位。

AgentAtlas 在此线上进一步提出 4 个组件：

  • (i) 一套 6 状态 control-decision taxonomy：Act / Ask / Refuse / Stop / Confirm / Recover；
  • (ii) 一套 9 类 trajectory-failure taxonomy，带两条正交的层级标签——primary_error_source 与 impact；
  • (iii) 一种 taxonomy-aware vs taxonomy-blind 方法学：测量"模型表观能力中有多少是 prompt 监督带来的"；
  • (iv) 一份 15 个 agent benchmark 的覆盖审计，沿 6 条行为轴对照映射，找出共同盲点。

方法学验证：在固定 8 模型集（1,342 项 · 4 frontier closed + 4 open-weight）双 prompt 模式下跑——移除显式标签菜单使每个模型的 trajectory accuracy 下降 14–40 pp，无论模型族落入 0.54–0.62 的紧致带；没有任何单一模型在 control accuracy / trajectory diagnosis / tool-context utility retention 三轴上同时领先。作者强调这是一次 "measurement-protocol demonstration, not a benchmark release"。

核心内容解读（背景·方法·差异）

解决了什么问题（why now）：5-13 的 Evaluation Differential 提出"declarative semantics"，5-14 的 TRIAGE 把"前瞻性元认知"做成第七维，5-15 的 ChromaFlow 用 negative ablation 否定盲目堆 orchestration，5-19 的 ANNEAL 把 recurring failure 翻译成 governed symbolic patch，5-20 的 DecisionBench 揭示"mean quality 几乎不可分辨而 routing fidelity 7.5–29.5%"。AgentAtlas 把这条主线推到了一个更上层的诊断：不是再造一个 benchmark，而是先回答"评测面本身是不是搭对的"——通过审计 15 个现有 benchmark 沿 6 条行为轴的覆盖空白，正面挑战"准确率列即比较单位"这个隐含假设。

核心方法：4 件可拆零件 + 双 prompt 协议

  • 6 状态 control-decision taxonomy（Act / Ask / Refuse / Stop / Confirm / Recover）：一个 agent 在每一步决策"做什么类的事"——这是"动作语义层"，把 tool-call 之外的 ask/refuse/stop 全部纳入一等公民。
  • 9 类 trajectory-failure taxonomy（双层级）：失败被同时贴两个正交标签——primary_error_source（错从哪里发源） × impact（错带来什么后果）。这让"同一种现象由不同根因导致"被显式可分。
  • Taxonomy-aware vs taxonomy-blind 双 prompt 模式：同一项任务、同一模型，跑两次——一次把 6 状态/9 类菜单显式贴进 prompt，一次完全去掉。差距即"模型自身的能力 - prompt 监督带来的能力"。
  • 15 benchmark 行为轴覆盖审计：把 GAIA、τ-bench、AgentBench、SWE-bench、WebArena、OSWorld、ToolBench 等 15 个 benchmark 的"测什么"按 6 条行为轴映射成可视化矩阵——直接暴露共同盲区（如 Confirm/Recover 普遍缺测）。

核心数据亮点 / 关键论点
  • 1,342 生成项 × 8 模型 × 双 prompt 模式——用最小固定模型集做 protocol demonstration，避免"自家 benchmark 自家刷"的套路。
  • 移除 taxonomy 标签菜单 → trajectory accuracy 下降 14–40 pp：模型表观能力中有非常大一部分是 prompt 监督蒸出来的，不是模型自身行为能力。
  • 0.54–0.62 紧致带：所有 8 模型在 blind 模式下都跌入这一窄带——意味着 "frontier vs open-weight" 的差距在没有 prompt 监督时被压扁。
  • 无单一模型同时领先三轴（control accuracy / trajectory diagnosis / tool-context utility retention）——这是"没有 universal best agent"的硬证据。
  • 6 状态 control-decision taxonomy 是少数把 Ask/Refuse/Stop/Confirm/Recover 显式列入第一公民的方法——大多数现有 benchmark 只测 Act。
  • 9 类 trajectory-failure × 2 轴正交标签：让"同一现象不同根因"被分解；这是后续做归因实验的工具底座。
  • 15 benchmark 覆盖审计第一次把"benchmark 沿 6 条行为轴的盲点"做成可视化——直接告诉社群"下一步该补哪一条"。
对你三个研究方向的启发
Harness Engineering（直击毕设）

AgentAtlas 给你 KV-cache 仿真评测一个直接对位的"双模式 protocol"思路：把"Top-K prefetch + 多种驱逐"运行两遍——一次把 access pattern 的 oracle 信号显式喂进策略（taxonomy-aware），一次完全 blind。如果在 blind 模式下 hit rate 也跌入紧致带，就证明你"看似有效"的策略其实是 oracle 信号蒸出来的，不是 working-set 估计本身的能力。再进一步，把 6 状态 control-decision 类比成"cache 决策状态"——Keep / Evict / Prefetch / Refuse-prefetch / Confirm-on-miss / Recover-from-spill——给毕设第三章添加一个比单一 hit rate 更立体的"cache 行为面"评测。9 类 trajectory-failure 也可借鉴：把"miss / wasted prefetch / late prefetch / pollution / thrashing" 等失败建成正交两轴（root cause × impact）。

Agent Skills Safety

对 agent safety 的迁移路径：把 SKILL.md 安全也做成 aware vs blind——一次把 6 状态决策表（含 Refuse / Confirm）显式贴入 system prompt，一次去掉。如果 blind 模式下 unsafe 调用率暴涨，就证明现有"安全 agent"的安全大部分来自 prompt 监督，而非内化策略——这跟 5-13 SKILL.md 攻击（86%/77.6%/100% 绕过）以及 5-20 ECA authorization-failure 框架完全同调，AgentAtlas 提供了一个可量化"prompt-stripped 安全余量"的评测协议。

Safety Benchmark

对 benchmark 方向是直接弹药："taxonomy-aware vs blind 双 prompt"是任意 safety benchmark 都该补上的协议——HarmBench / SafetyBench / GrandGuard 都应该报告 blind 下 ASR 是否回弹。同时，15 benchmark 行为轴覆盖审计是一个可立刻搬到 safety 域的方法学：把 30+ 现有 safety benchmark 沿"过度拒答 / 误调工具 / 链路 hijack / authorization 错位"4 条轴映射，找共同盲区。这跟 5-20 POLAR-Bench 的 5×5 诊断面是同一文化的两种实现——AgentAtlas 提供了"为什么要这么做"的方法学骨架。
