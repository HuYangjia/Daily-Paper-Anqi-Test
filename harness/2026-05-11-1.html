摘要（中文翻译，忠实原文）

生产环境的 agentic 系统对每个用户请求都要做多次模型调用，其中绝大多数都是短的、结构化的、例行的。这带来一个现有 evaluation 没有直接回答的实际 routing 问题：在 agent 工作流中，到底哪些环节真的需要前沿大模型，哪些可以由小模型胜任？

我们提出 AgentFloor——一个 deterministic 的 30 任务 benchmark，按 6 阶能力 ladder（capability tier）组织，覆盖 instruction following、tool use、multi-step coordination、long-horizon planning under persistent constraints。我们评测了 16 个开源模型（0.27B 到 32B 参数）加上 GPT-5，总计 16,542 条 scored run。

结果揭示了一条清晰的"必要性边界（boundary of model necessity）"：小型与中型开源模型已足以胜任真实 agent pipeline 的大部分短时长、结构化工具使用任务；汇总来看，最强开源模型在 AgentFloor 上的得分与 GPT-5 持平，运行成本和延迟显著更低。差距最明显的是长时计划任务——需要在多步骤上保持持续协调和可靠的约束追踪——这里前沿模型仍然占优，但双方都没达到强可靠性。

我们还发现这条边界不能仅由参数规模解释：部分失败可以通过定向干预改善，但效果是 model-specific 而非普适。这些发现给 agentic 系统提示了一条务实的设计原则：把广泛的 routine 操作交给小型开源模型，把真正需要更深规划与控制的窄类任务保留给大型前沿模型。我们公开了 benchmark、harness、sweep configurations 和完整 run corpus。

核心内容解读（背景·方法·差异）

解决了什么问题：过去一年的 agent harness 研究主要是"把外延做大"——Token Arena 把 (provider, model, SKU) endpoint 当评测单元，Uno-Orchestra 用 RL 端到端学 (model, primitive) 决策面，More Is Not Always Better 把组件子集做成 2⁵ 全因子。AgentFloor 攻的是另一条线：把"该用哪个模型"这件事建成可路由的 deterministic 评测。它把"小模型够不够用"这一个工业界天天在问、却没有 benchmark 直接回答的问题，落到 30 任务 × 6 tier 的固定坐标系上，跑出 16,542 条评分；从而让 router 设计从"猜"变成"读图"。

核心方法（6 阶 ladder × deterministic harness）：

  • 能力 ladder 6 阶：从最低阶 instruction following，到 tool use（结构化调用）、multi-step coordination、long-horizon planning under persistent constraints。每阶对应一组任务的难度面，可拼成完整能力曲线。
  • 30 个 deterministic 任务：关键修饰词是 deterministic——任务结果可由确定性 verifier 判定，避开了 LLM-judge 的随机性偏差，也保证了 16,542 条 run 之间可比。
  • 17 个模型横切：16 开源模型从 0.27B 到 32B（覆盖了 SLM/MLM/LLM 三个量级）+ GPT-5；同步 sweep 多种 sweep configurations（temperature/top-p/system prompt 模板）。
  • 报告"边界"而非"排名"：核心交付不是"模型 leaderboard"，而是 capability frontier——每阶 tier 上"何处开源已够、何处必须 frontier"的边界图。
  • negative result 与 model-specific 干预：作者明确报告"边界不由 scale 单独解释"——某些失败响应定向 prompt 干预，效果却是 model-specific，不可普适，这是工程上比一句"scale up"更实用的指南。
  • 完整发布：benchmark + harness + sweep configs + full run corpus，所有 16,542 条 trajectory 都可复跑——这是真正"can you reproduce my router?"级别的开源。

核心数据亮点 / 关键论点
  • 16,542 条 scored run（30 任务 × 17 模型 × sweep）——deterministic verifier 全部可复跑，是当前规模最大的 small-vs-frontier 直接对比之一。
  • 开源 ≈ 前沿（短时结构化任务）：在低-中阶 tier（instruction following、tool use、短-multi-step）上，最强开源模型在 AgentFloor 上的得分与 GPT-5 持平，且更便宜更快。
  • 差距集中于长时规划：在 long-horizon planning under persistent constraints 上，前沿模型仍占优，但双方都未到达强可靠性——这是 agent harness 研究的下一个明确战场。
  • 边界不是 scale-only：作者明确指出"this boundary is not explained by scale alone"——某些失败响应定向干预（prompt/scaffolding 调整），但效果 model-specific，非普适。
  • 16 模型 × 0.27B – 32B：跨越 SLM 到中型 OSS 的完整覆盖（包括 Qwen / Llama / Phi / Gemma 系列），确保 ladder 上每一阶都有候选可比较。
  • 设计原则："broad base of routine actions → small open-weight；narrower class of tasks demanding deeper planning → frontier"——这是工业界 router 设计可以直接落地的工程结论。
  • 开源完整度高：benchmark + harness + sweep configurations + full run corpus，即"可以基于它直接复跑路由器"。
对你三个研究方向的启发
Harness Engineering

AgentFloor 给 harness 工程添了一个新的"决策面变量"：model size routing。它和 More Is Not Always Better 的"组件子集"+ Token Arena 的"endpoint SKU"组合起来，可以构成一个三轴坐标系 (model size, scaffolding subset, endpoint)。对你正在写的 ZipperQuant W4A4 e2e benchmark：可以直接借鉴 AgentFloor 的"capability tier × deterministic verifier"思路——把 (Qwen3-8B, prefill_len, decode_len) 测试点按"短上下文/长上下文/超长上下文"分 tier，用 BF16 baseline 做 verifier reference，让你的 ZipperEngine 跑分以"在哪个 tier 上 W4A4 还能匹配 BF16"的形式呈现，比单纯报 throughput 更有说服力。NeurIPS 投稿 ablation 也可以参考它的 sweep config 公开方式。

Agent Skills Safety

AgentFloor 的 "boundary not explained by scale alone" 对 agent safety 是关键提示：routing 决策本身就是一个安全 surface——把高敏感任务（如代码执行、文件操作）误路由到能力不足的小模型上，会直接放大 hallucination/误用工具的安全风险。今日 Agent Safety 报告中的 Skills as Verifiable Artifacts 提出"skill 必须按验证级别加 capability gate"——把 capability gate 和 AgentFloor 的 capability ladder 直接对接，可以得到"模型能力 × skill 信任级别"的二维 admission policy，比单维 skill 信任更稳健。

Safety Benchmark

30 deterministic tasks + 6 tier 给 safety benchmark 一个直接可学的形态：把 ASR / NSR / refusal-correctness 也做成 deterministic scoring（不依赖 LLM-judge 的 ASR 偏移），并按 capability tier 切片。今日 Benchmark 方向 ML-Bench&Guard 已经在做"政策对齐的多语种 safety benchmark"——下一步自然是把"能力 tier × 政策合规"两维交叉，让评测显示"小模型在低 tier 上能不能也合规拒绝"。这是更接近真实 agent 部署的评估形态。
