摘要（中文翻译，忠实原文）

AI Agent 正被越来越多部署到不同领域，承担长时序、高风险动作的复杂工作流。由于其能力强、灵活性高，这类 Agent 引发显著的安全和合规担忧。一系列真实事故已证明：对手可以轻易诱导 Agent 执行危险动作——泄露 API key、删除用户数据、发起未授权交易等等。

评测 Agent 安全性本身极具挑战性——它们运行在动态、不可信、含外部工具、异构数据源、频繁人机交互的环境里。但是真实、可控、可复现的大规模风险评估环境，迄今仍未充分建立。

为此，本文提出 DecodingTrust-Agent Platform (DTap)——首个可控可交互的 AI Agent 红队平台，覆盖 14 个真实域，包含 50+ 仿真环境，复现 Google Workspace、PayPal、Slack 等被广泛使用的系统。在 DTap 之上，作者进一步提出 DTap-Red——首个自主红队 Agent，能系统化探索不同注入向量（prompt / tool / skill / environment / combinations），针对不同恶意目标自主发现攻击策略。基于 DTap-Red，作者构造了 DTap-Bench——一个跨域大规模红队数据集，每个样本配套可验证的 judge 用以自动核验攻击结果。

通过 DTap，作者对运行在不同 backbone 模型之上的主流 AI Agent 进行了大规模评测，覆盖安全策略 / 风险类别 / 攻击策略三轴，揭示出系统性的脆弱模式，为下一代安全 Agent 的开发提供洞见。

核心内容解读（背景·方法·差异）

解决了什么问题：这条 Safety Benchmark 主线最近一年逐步把"题集"两个字撑大——BenchGuard 审 benchmark 自身、Claw-Eval-Live 引入实时刷新、CarryOnBench 把 safety×utility 接成多轮、RedVLA 把红队推到物理 VLA、SafetyALFRED 接厨房物理隐患、AcademiClaw 让"学生出题"。DTap 给出的是结构上完全不同的一档贡献：把 benchmark 从"静态题集"升级为"由自主红队 Agent 持续生成新题的工业级平台"，并由 DecodingTrust 团队（Bo Li / Dawn Song / Percy Liang / Sanmi Koyejo 牵头，该方向最具号召力的作者群）背书。

三层架构：

层组件作用
仿真层DTap Platform（14 域 / 50+ 环境）覆盖 Google Workspace / PayPal / Slack 等真实系统的可控复现，让 Agent 在受控沙盒里"真打"
红队层DTap-Red（自主红队 Agent）系统化探索 5 类注入面（prompt / tool / skill / environment / combinations），自主收敛攻击策略
评测层DTap-Bench（带 verifier 的大规模数据集）每条样本附自动 judge，攻击结果可被复现验证；分维度（policy / risk / attack）切片打分

5 类注入向量的工程意义：

  • Prompt 注入——经典对话攻击，文献基础最多。
  • Tool 注入——污染 Agent 可调用的工具描述/返回值。
  • Skill 注入——污染 Agent 加载的"技能/记忆"内容（与 HarmfulSkillBench 同源）。
  • Environment 注入——污染外部网页、文件、邮件。
  • Combinations——上述 4 类组合攻击；正是过往 benchmark 最弱的一档。

与现有工作的核心差异：

  • 对比 AgentBench / SafetyBench / HarmBench：它们是静态评测；DTap 把"生成攻击"也做成 Agent，使 benchmark 自带"持续刷新"能力——这是 Safety Benchmark 在 2026 年最重要的范式跃迁。
  • 对比 Claw-Eval-Live：Claw-Eval-Live 是"任务侧 live"，DTap 是"攻击侧 live"——两者正好正交，可以叠加。
  • 对比 RedVLA：RedVLA 在 VLA 物理域；DTap 在数字域（SaaS/办公协作）；两者构成"物理红队 + 数字红队"双轴。
  • 报告体量：论文 279 页 + 148 图——这是该领域近一年最大的单篇技术报告，参考 DecodingTrust 系列前作的传统（Bo Li 团队的旗舰评测系列），通常会持续维护版本。

对你三个研究方向的启发
Harness Engineering

DTap 的"自主红队 Agent + verifier"组合，本质上是把"评测 harness"搬到了攻防两侧——"测对象"和"出题人"双方都是 Agent。把它和今日 Harness 报告 Uno-Orchestra 的"编排即可学习"结合，下一阶段 harness 应该天然支持对抗自演化：编排策略 vs 红队 Agent 在同一 harness 上对抗演化，类似 GAN 但目标是 trajectory-level 的安全 + 效用双目标。

Agent Skills Safety

DTap 的 5 类注入面与今日 Agent Safety 报告 AgentTrust 的 4 类 verdict 几乎是天然配对：DTap 出题、AgentTrust 应试。建议把 DTap-Bench 的 case 直接灌入 AgentTrust 评测，再把 AgentTrust 误判反哺 DTap-Red 的攻击搜索——这能形成"红队 → 守卫 → 红队"的闭环数据飞轮，是 Agent Safety 真正可工程化的标志事件。

Safety Benchmark

DTap 让 Safety Benchmark 进入"第三代"——第一代是题集（HarmBench），第二代是题库刷新（Claw-Eval-Live），第三代是自主出题。下一阶段的 benchmark 标准应包含：(1) 注入面分类法 ≥ 4；(2) 自动 verifier；(3) 攻击 Agent 的可演化性。结合 Judge Sensitivity 的 ASR 24.2 pp 偏移结论，未来 benchmark 必须同时报告"攻击 Agent 配置 + Judge 配置"两轴稳定性。
