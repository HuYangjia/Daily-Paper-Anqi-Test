摘要（中文翻译，忠实原文）

科学 AI agent 越来越多地检索证据、跨源推理、合成关键决策中所用的结论。然而，它们在健康等高风险领域执行此类任务的能力仍不清楚。我们提出 SciConBench，一个由 9.11K 问题与系统综述中专家撰写的结论 构成的大规模在线基准，用以评估开放域科学结论合成。

该基准依赖一条经专家验证的自动评估管线，将结论分解为原子事实（atomic facts），并通过事实精确率与召回率衡量正确性与完备性。为缓解数据泄漏，我们进一步提出 SciConHarness——一个 clean-room（净室）评估 harness，让 agent 在受控的网络交互下被评估，以保证测量的有效性。

在评估 8 个前沿模型与深度研究 agent 后，我们发现事实质量仍然很低：在 净室设置下，最佳 agent 的事实 F1 仅为 0.337。我们的净室设置始终降低了相对非约束评估的性能，提示数据泄漏会虚高估计模型真实的合成能力。

最后，我们对面向消费者的 agent（例如 Google AI Overview、OpenEvidence）进行审计，发现即使在地面真值答案可获取的情况下，它们仍频繁生成不完整甚至自相矛盾的结论。

综合而言，我们的结果表明：科学结论的可靠合成仍是开放挑战，且净室评估对评估开放域 AI agent 至关重要。

核心内容解读（净室评测 / 原子事实分解 / 数据泄漏问题 / 消费级审计）

立论：当 agent 拿到 PubMed / Wikipedia / Google Scholar / Cochrane 这些公开科学数据库的浏览权时，它们的"评测分数" 究竟反映"能力" 还是"记忆"？这是 2026 年 agent benchmark 圈最深刻的方法学危机之一。所有带网络访问的 agent benchmark（包括 GAIA、Mind2Web-Live、AssistantBench 等）都默认 agent 会"诚实搜索 + 真合成"，但是 — 当 benchmark 题目的答案早已存在 agent 的训练语料里时，agent 实际上是从内存中输出训练时见过的结论，而不是从证据中合成结论。SciConBench / SciConHarness 是首个把这条混淆变量正面解决的工作。

9.11K 系统综述题：高风险领域 + 专家撰写答案 + 不易被泄漏。选择系统综述（systematic reviews）作为问题源是关键的方法学决策：

系统综述题源 vs 其他 benchmarkSciConBench 选择带来的方法学优势
问题分布从 Cochrane / PRISMA / 顶级医学期刊 systematic reviews 抽取真实临床 / 公共健康问题 — high-stakes domain
答案来源专家在 systematic review 中撰写的结论部分非众包 / 非自动生成 — 答案是 ground truth 的金标准
规模9,110 个问题 — 比同期 benchmark 大一个数量级统计功效足够支撑细分领域 ablation
动态更新"live benchmark" — 持续注入新发表的 systematic review抗污染 — 训练截止日之后的题不可能在训练集里
评估方式原子事实分解 + 事实精确率 / 召回率解决"结论模糊措辞导致评分主观" 的老问题

SciConHarness 的关键工程：受控网络交互。普通的 agent benchmark harness（如 GAIA / WebArena 等）让 agent 自由访问开放互联网；但当问题来自 systematic review，原文很可能已被 agent 在训练时索引，agent 可以一搜直接命中原始结论文本——这就让"合成" 变成"检索 + 复制"。SciConHarness 设计的受控网络交互则在 agent 外部加了一道clean-room 屏障：哪些域名可访问、哪些段落可见、是否要求 agent 用证据出处链回答……这让 agent 必须从原始论据合成而不能直接复制结论。这条工程哲学与 6-09 MAC-Bench 的 SERV holographic sandbox、6-10 PreAct-Bench 的"预测时刻评测面" 同源——都是把评测面本身 立为 first-class 工程对象。

核心发现：净室设定下能力降级一致且大。"净室设置下最强 agent 事实 F1 仅 0.337" 这一数字本身已是震撼——但更震撼的是系统性对比：同样的 agent 在非约束评估下分数显著高于净室。这就把"数据泄漏使评估虚高" 从猜想升级为跨 8 个 agent 的稳健实证。这条结论与 6-03 Reliability Gap in Benchmark Auditing "污染检测仅 59.4% 正确" 完美互补——前者证明"检测污染很难"，本论文证明"污染对评估的扭曲很大"。两者一起为"所有评测必须假设污染存在并设计 clean-room 防护" 提供双重论据。

消费级 agent 审计：Google AI Overview / OpenEvidence 即使能拿到 ground truth 也常给不完整 / 自相矛盾的结论。这是论文最具公共政策意义的部分。当一个普通用户在 Google 搜索"X 治疗对 Y 病的效果如何"，AI Overview 会自动生成结论摘要——但研究表明这些摘要 常常错过关键限制条件 / 误读相反证据 / 给出被原综述明确反对的结论。这条审计直接对接公共健康风险——和 6-03 What Benchmarks Don't Measure "compliance bias 让用户看到 89.2% blocking + 87.5% usability 双高" 形成对照：一边是"太能拒"，一边是"太能编"——两者都是 RLHF 后部署时的结构性失败模式。

方法学定位：把 benchmark 设计的"方法学严谨度" 立为可比较维度。SciConBench / SciConHarness 之所以是 2026 H1 最重要的 benchmark 工作之一，不在于"分数" 本身，而在于它把"benchmark 自身的可信度" 立为一等讨论对象。这条姿态对位 6-08 AARR/AARRI-Bench"把 harness 与模型解耦报告"、6-09 MAC-Bench "Beyond Goodhart's Law"、6-10 PreAct-Bench "禁前评测面"——共同构成 2026 H2 benchmark 圈的"方法学三轴"：(a) 评测面在哪里？(b) 评估对象是否真在被测？(c) benchmark 自身可信吗？

研究队伍：含 USC 计算社会学家 Aleksandra Korolova 与 EPFL 信息学家 Manoel Horta Ribeiro——两人都是计算社会学 + AI 政策圈的著名学者，本论文有强烈的"把 AI 评测从 ML benchmark 推向 public-impact accountability" 的政策导向。这一定位让本工作在 ICML / NeurIPS 之外，也很可能进入 FAccT 2026 / AAAI AIES 2026 等政策导向会议。

Harness Engineering（直击毕设）

SciConHarness 的 clean-room evaluation harness 概念对你的稀疏注意力 KV-cache 预取毕设带来直接启发：任何"cache 命中率提升" 报告都必须考虑数据泄漏的对应问题 — 即"训练分布上的 cache 模式是否被 over-fit"。建议：(1) 在毕设里设计一个holdout-prompt 套件，与训练时使用的 prompt distribution 完全独立；(2) 报告时分两栏 — "训练分布上的 cache 命中率" vs "holdout 分布上的 cache 命中率"；(3) 把两者的 gap 作为"策略对训练分布的过拟合" 度量。这条做法本质上是"系统层 clean-room evaluation"——把 SciConHarness 的方法学引入 ML systems 评测，能直接让你的论文具备 6-08 AARR/AARRI-Bench 那种"评测面诚实" 的方法学品质。

Agent Skills Safety

SciConHarness 与今日 #2 Impossibility of ELK 形成深刻互补：ELK 不可能定理在理论层证明仅靠行为反馈无法保证诚实，SciConHarness 在评测层提供"受控信息访问" 的工程范式——后者实际上提供了一个测量"不诚实合成" 的具体工具。把两者拼起来：未来 agent honesty 评测必须包含 (a) clean-room 屏障 + (b) 表征级 probe + (c) 原子事实分解——三件套缺一不可。为什么消费级 agent（Google AI Overview / OpenEvidence）在能拿到 ground truth 时仍生成矛盾结论？因为它们的 RLHF 目标是"让用户看到流畅、有信心的答案"——这正是 ELK 不可能定理刻画的"讨好 generalisation"。
