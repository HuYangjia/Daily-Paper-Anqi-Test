摘要（中文翻译，忠实原文）

AI agents 越来越多地被开发用于加速科学发现，但其在真实研究场景中的实际能力仍然知之甚少。现有的 AI agent benchmark 很少捕捉科学工作所需的复杂性、异构性与扩展推理；而科学任务的 benchmark 又常常把研究简化为静态、直接的问题，对交互式评估支持有限。

本文引入 SciAgentArena——一个用于评估 AI agents 在真实科学研究场景中表现的系统化 benchmark，任务来自多个学科的新兴需求。SciAgentArena 包含约 200 个任务，配有步骤级验证（stepwise verification），以及一个交互式、agent-agnostic 的环境 用于评估多样化的 AI agents。

使用该 benchmark，我们发现当前 agents 能在规范化数据分析工作流中作出有效贡献，特别是 当任务结构与评估标准明确时。但是，它们的表现在不同科学场景下极不均匀：agents 难以生成真正新颖的洞察、难以维持自主探索、难以为开放式研究问题给出鲁棒解决方案。

我们进一步刻画了不同 agents 间的共同失败模式，并识别出在可靠性、自主性与科学推理三方面的改进机会。综合来看，SciAgentArena 为测量科学 AI agents 进展 和指导未来设计能应对复杂科学挑战的 agents 提供了实用框架。

核心内容解读（双 gap 立题 / 步骤级验证 / 交互环境 / 三层失败模式）

立论：当前 agent benchmark 与 scientific task benchmark 都失之偏颇。SciAgentArena 用双 gap 立题：(a) AI agent benchmark（如 AgentBench / GAIA / WebArena）不捕捉科学复杂性——任务通常是 web 浏览、表格问答、简单代码，缺乏跨步骤异构推理；(b) scientific task benchmark（如 SciCode / MMLU-Science / GPQA）把研究简化为静态问题——给一道题、给一个答案，缺乏探索、决策、修改的交互 维度。这两类 benchmark 之间存在结构性"科研评测断层"——没有 benchmark 在真实科研形态 下评估 agent。SciAgentArena 正是为填补这一断层而设计——这条立题与 6-09 #1 SciConBench"系统综述真实科研问题"（arXiv 2606.11337）、6-08 #3 AARRI"研究行为本身是新瓶颈"（arXiv 2606.07462）形成"三层科研评测梯度" —— SciAgentArena 跨学科真实新兴需求 / SciConBench 系统综述精度 / AARRI 研究员级行为素养。

方法学贡献：把"步骤级验证" 与"agent-agnostic 交互环境" 立为 benchmark 双轴。

核心发现一：agent 在结构清晰任务上"有效贡献"——但这是低天花板。论文坦诚承认 agent 在"规范化数据分析工作流" 上能作出有效贡献——这与 6-12 #1 HarnessBridge / ToolSense 反映的"规范化任务上 SOTA" 共同构成 2026 H1 agent 能力共识。但 SciAgentArena 提出尖锐质疑："规范化数据分析" 是科研工作的低天花板部分——真正的科研价值在新颖洞察、自主探索、开放式问题 三层。这条姿态与 6-08 #3 AARRI"研究员级专业素养" 共振——两者都在说"过去 agent benchmark 用的是研究实习生 / 助理级任务"，但研究员级 的"提出问题" 才是真正的科研瓶颈。

核心发现二：agent 失败的共同模式 是结构性而非个例。论文系统刻画了不同 agents 间的共同失败模式——这是论文的核心方法学贡献。这意味着失败不是某种特定 model / harness / prompt 的偶然 bug，而是当前 agent 范式的结构性缺陷。论文识别三个具体改进机会：(a) 可靠性——任务执行的鲁棒性不足；(b) 自主性——agent 难以维持长时间无指导探索；(c) 科学推理——agent 在提出而非解答 科学问题上能力弱。这与 6-08 #3 AARRI"研究行为本身是新瓶颈" 的判断互证——两篇都在说当前 agent 范式在能力维度上 已遇到结构性瓶颈，需要范式跃迁。

核心发现三：interactive 环境改变了"什么算 SOTA" 的定义。论文用 stepwise verification + interactive 环境同时评测，意味着同一个 agent 在不同 评测形态下分数差距巨大——比如某个 agent 在静态评测上 SOTA，但在 interactive 评测上失败模式高度暴露。这条性质与 6-14 #1 ToolSense（arXiv 2606.12451）"知识—检索分离"、6-12 #1 HarnessBridge"harness 影响 agent 评测" 共同推动 2026 H2 评测方法学姿态：未来 agent benchmark 必须显式声明 (a) 任务是静态还是 interactive；(b) 评估是任务级还是 stepwise；(c) agent 接入是 fixed 还是 agnostic——这三档姿态共同构成 2026 H2 评测设计的必要 报告维度。

哲学定位：把 benchmark 从"能力衡量器" 升级为"失败模式诊断器"。SciAgentArena 最深远的影响可能是方法学层——它不主要追求"谁是 SOTA"，而是主要追求"当前 agent 范式有哪些共同 失败模式"。这种"诊断器" 姿态在 2026 H1 后期成为新主流——6-11 #1 SciConBench"clean-room evaluation"、6-12 #1 MLUBench"多模态对齐保持" 失败诊断、6-14 #1 ToolSense"知识—检索分离" 失败诊断，三者合起来构成"诊断型 benchmark" 的新范式——SciAgentArena 把这一姿态扩展到跨学科真实科研 场景。

开源价值：跨学科真实科研评测基础设施。项目主页 sciagentarena.github.io 公开全部任务、代码、数据集——这意味着任意 agent 团队可以在自己的系统上跑一遍 SciAgentArena，独立报告 step-level 分数。这条工程价值在科研能力评测 领域尤其关键——过去这类评测要么需要 PI 级专家手工评估（不可扩展），要么用静态题目（评测形态退化）。SciAgentArena 用 interactive + agnostic + 步骤级验证三件套打开了"可扩展真实科研评测" 这条路径。

Harness Engineering

SciAgentArena 给 harness 工程方向最深远的启发是"agent-agnostic 交互环境" 立为 harness 设计第一公民——传统 harness 设计姿态是"为某种 agent 优化某种环境"；SciAgentArena 提出更高的姿态："设计能让任意 agent 接入并被公平评测的环境"。具体到你的 KV-cache 毕设：(a) 借鉴agent-agnostic 思想——你的 cache 系统应该对接口 解耦，不绑定某个具体 model 或 inference framework；(b) 借鉴stepwise verification——你的 cache 评测应该有每个 cache 决策的独立追踪，而非仅看 end-to-end 吞吐；(c) 借鉴interactive 形态——你的 cache trace 应来自真实多租户长尾访问 而非合成均匀工作负载。论文写作姿态：把 SciAgentArena 列为"agent-agnostic interactive 评测姿态" 的方法学先例，与 6-14 #1 ToolSense"诊断 harness" 共同支撑你的"cache 评测姿态升级" 论点。

Agent Skills Safety

SciAgentArena 与今日方向 #2 "Did you lie?" 形成互补——SciAgentArena 把 agent 在科研开放式形态 下的失败模式系统刻画；"Did you lie?" 把 agent 在说谎—信念差距 形态下的检测器失效系统刻画。两篇合起来对 agent skills safety 的影响：失败模式刻画姿态 是 2026 H2 safety 的统一方法学——不再追求"检测某种特定危害"，而追求"系统刻画 agent 失败模式空间"。建议：未来 agent skills safety 评测应同时声明：(a) 任务是结构清晰还是开放式；(b) 失败模式是任务级还是 step 级；(c) 失败是个例还是跨 agent 共享。这三档姿态与 6-12 #1 Containment Gap"架构层 safety"、6-14 #1 "Did you lie?""白盒检测器失效" 共同推动 2026 H2 agent safety 从"对齐 + 检测" 升级为"对齐 + 检测 + 架构 + 失败模式刻画" 四联架构。

Safety Benchmark（直击）

SciAgentArena 把 safety benchmark 方向推向"失败模式诊断器" 这一新姿态——传统 safety benchmark（HarmBench / TrustLLM / SafetyBench / AIR-Bench）追求"有害响应率" 这种单数字 SOTA；SciAgentArena 提出更高的姿态："刻画跨 agent 共享的失败模式空间"。具体落地：(a) 未来 safety benchmark 应同时报告 (i) 总体性能 (ii) 失败模式分布 (iii) 跨 agent 共享性；(b) 失败模式应在结构层（可靠性 / 自主性 / 推理）而非表面 层（特定 prompt 类型）；(c) 评测应有步骤级颗粒度，区分"错在哪一步" 与"下游误用"。这条姿态与 6-12 #1 MLUBench"多模态对齐保持" 失败诊断、6-11 #1 SciConBench"clean-room evaluation"、6-14 #1 ToolSense"知识—检索分离" 共振，构成 2026 H2 safety benchmark"诊断型评测" 主流姿态。SciAgentArena 把这一姿态扩展到跨学科真实科研 场景，是 2026 H2 评测设计的方法学拐点。
