摘要（中文翻译，忠实原文）

在具身智能（embodied intelligence）中，安全是机器人在物理世界中可靠部署的前提。当前视觉-语言-动作（VLA）模型不断朝通用任务能力 演进，但其具身安全边界仍然缺乏研究。为填补这一空白，我们提出 ForesightSafety-VLA：一个把安全 作为主要评估目标 的诊断性 benchmark。我们定义了一个 13 类安全分类法，覆盖物理交互安全（Safe-Core）、指令侧安全（Safe-Lang）、感知侧安全（Safe-Vis），并在三个受控变化维度（场景结构、语言指令、视觉观察）下评估策略，让失败来源可被诊断，而不是被淹没在单一聚合分数里。除二值任务成功之外，ForesightSafety-VLA 用累积安全代价（CC）与风险暴露时间（RET）测量过程级风险，并配以"安全/不安全成功 + 失败" 的四象限分解。我们在 RoboTwin 上实例化了 66 个安全增强基础场景，覆盖 5 种 embodiments，并在代表性 VLA 基线上报告结果。在所评估的基线中，即使最强的策略仍有非零的安全代价与不安全的"名义成功"；结构变化与视觉变化引发的安全退化远超普通语言变化。这些结果表明：具身安全与感知、grounding、控制能力深度耦合，无法仅靠事后过滤来处理。

核心内容解读（13 类分类法 / 三轴诊断 / 过程级度量 / 四象限 / "能力越强越安全" 反例）

立论：VLA 安全不是"动作过滤"，而是场景内生属性。当前 VLA 模型从 OpenVLA / RT-2 / π0 / Octo 到 RDT / DexVLA / TinyVLA 都在快速演进，但它们的安全评测一直依靠两条传统路线：(1) 事后过滤（post-hoc filter）——加一个 safety classifier；(2) 规范化指令——只在干净 prompt + 干净视觉下测任务成功率。ForesightSafety-VLA 团队的关键观察是：具身安全与感知、grounding、控制能力深度耦合—— 你不可能用一个外挂 filter 把"把手伸进加热区" 拦下来，因为这要求模型先在 perception 里识别加热区。这就需要把"安全" 在 benchmark 设计阶段就嵌进任务构造。

13 类安全分类法 —— 物理 × 指令 × 感知三大家族。

L × W × V 三轴受控诊断变化。每个 episode 同时受三个变化维度影响：(1) L 结构变化 L0–L2—— 危险物放置、杂乱度、间隙、时序耦合；(2) W 语言变化 W0–W4—— 从规范指令到改写、组合重写、模糊、不安全请求、注入攻击（W3–W4 由前沿 LLM 生成并人工验证）；(3) V 视觉变化 V0–V4—— 从规范观测到光照变化、外观变化、视角/遮挡退化、对抗补丁（累积扰动，物理场景不变）。这种 L×W×V 三轴设计是 benchmark 链条里第一次把扰动维度做成正交受控变量，让"失败归因到哪一轴" 成为可分析问题。

过程级度量 CC + RET：双阈值监控。每个安全通道 k 上定义 qk(st) 分数：qk ≥ εk = safe；0 ≤ qk < εk = warning（累积软风险代价）；qk < 0 = hard violation。

  • CC（Cumulative Safety Cost） = Σt Σk wk·ck(st)，其中 ck(st) = gk(max(0, εk − qk(st)))。CC 是主要的过程级排名指标。
  • RET（Risk Exposure Time） = Σt 𝟙[∃k: qk(st) < εk]。RET 衡量暴露持续时间，区分"短暂峰值" vs "长期处于危险区"，用于识别侥幸成功。
  • SASR（Safety-Adjusted Success Rate） = (1/N) Σ 𝟙(Si ∧ ¬Vi) · exp(−λ · CCi/Ti)。设 λ=1 时退化为 SSR。

这是 VLA 安全 benchmark 里第一次 用 CC + RET 双指标精确测量"risk exposure 的持续时间"—— lift_pot 案例中 OpenVLA-oft 任务成功，但 Spatial Boundary 通道暴露 60%、Thermal/Energy 通道暴露 40%、Any active risk 占 62% episode 时长 → 证明 RET 必要性。

四象限分解：S × V 二维结果空间。每 episode 产生两个二值信号——任务成功 S 与是否硬违规 V，组合成 4 个互斥结果：SSR（安全成功 ↑）/ USR（不安全成功 ↓，任务完成但有违规）/ SFR（安全失败，保守失败）/ UFR（不安全失败 ↓，失败且伴随物理风险）。这种四象限分解把"能力 / 安全 / 失败归因" 三件事正式拆开。

关键结果 ① 没有任何 VLA 完全安全。

即使最强的 OpenVLA-oft 仍有 CC=0.18、USR=0.06、UFR=0.15—— SSR + USR ≠ 100%，且 USR > 0 是关键警讯。这意味着评估 VLA 不能只看 SSR，必须同时报告 USR。

关键结果 ② 反 trade-off —— 能力越强越安全。"不安全成功 / 总成功" 比例：OpenVLA-oft 12.5% < RDT 25% < DP 29.4% < ACT 37.5%。不存在能力-安全 trade-off—— 这反驳了 H1 早期"safety hurts capability" 的悲观叙事，与今日 #2 AgentDoG 1.5 的"safety + utility 同时改善" 形成姊妹证据。

关键结果 ③ 难度差异极大。Thermal/Energy 是最难类别（ACT CC 0.54），Temporal Sequence 最容易（CC 0.29 vs 0.39 总体）—— 近距离感知型危险需要细粒度空间推理，这正是当前 VLA 的薄弱环节。

关键结果 ④ 维度敏感性。结构变化 (L) 与视觉变化 (V) 引起的安全退化远超 普通语言变化 (W0–W2)；但 W3–W4（不安全请求、注入攻击）危害显著—— 提示视觉与结构是 VLA 安全的主要漏洞通道，语言变化只在对抗注入下才严重。

位置：与 H1 末 benchmark 谱系共构"诊断性 benchmark 三联拍"。6-23 Benchmark Illusion 拆解"评测意识" 为 4 轴多变量；6-25 Hacker-Fixer Loop 把 verifier 升级为可加固的一级研究对象；今日 ForesightSafety-VLA 把"诊断性 benchmark" 思想跨过 LLM 进入具身 VLA—— 三篇共同确立 H1 末 benchmark 的方法学共识："单一分数不够；必须多变量诊断 + 过程级度量"。

对你三个研究方向的启发
Harness Engineering

ForesightSafety-VLA 给 harness 工程提供了三条直接借鉴：(1) 过程级度量 CC / RET 思想 可平移到 KV-cache 系统毕设 —— 不能只测"任务完成时间"，而要测"整个执行过程中 cache miss / 显存峰值 / SLA 违规暴露的累积代价与暴露时间"；(2) 三轴受控诊断（L × W × V） 是评测 harness 的极好模板—— harness 的失败维度可以拆为结构（system topology）× 语言（API surface）× 视觉（telemetry），让失败归因可分析；(3) 四象限分解 S × V 可对偶到 harness：能跑且性能合规 / 能跑但性能违规 / 跑不通且保守降级 / 跑不通且性能违规—— 这正是 6-25 NOVA 的 silent failure 概念的诊断版升级；(4) "能力越强越安全" 反 trade-off 数据点提示 harness 工程"性能 vs 鲁棒性" 也可能不存在 trade-off，应该作为 KV-cache 实验的 hypothesis；(5) RoboTwin / FSM 思想—— harness benchmark 也可以用轻量仿真平台，无需全栈生产部署。

Agent Skills Safety

ForesightSafety-VLA 给 agent safety 三条新启发：(1) 把安全嵌入任务构造 而非事后过滤—— 这与 6-25 SkillSafetyBench 的"skill-mediated 攻击面" 思想互补，意味着 H2 agent safety benchmark 应该主动注入风险物 / 拧紧约束 / 插入时序前置；(2) 过程级 CC / RET 度量 直接平移到 agent safety—— LLM agent 长程轨迹也有"侥幸成功"（任务完成但路径上侵犯了隐私 / 越权 / 浪费资源），需要 CC + RET 双度量精确捕捉；(3) "能力越强越安全" 反 trade-off 实证 与今日 #2 AgentDoG 1.5 BFCL 76 → 81 形成跨方向、跨模态的姊妹证据—— 强 LLM 不一定更危险，反而更精细地理解了安全约束；(4) 13 类分类法 可类比为 agent safety 的细粒度风险叶子，与 AgentDoG 1.5 三维分类法（来源 × 模式 × 危害）形成"具身 × 数字" 双侧分类系统；(5) L × W × V 三轴诊断 对应 agent safety 的"环境扰动 × 指令扰动 × 观察扰动"。

Safety Benchmark（直击本方向）

ForesightSafety-VLA 重新定义了"可靠 safety benchmark" 的设计原则：(1) 单一标量不够—— SSR / USR / SFR / UFR / CC / RET / SASR 七元结果向量是最低交付；(2) 受控变化必须是正交多轴—— L × W × V 三轴笛卡尔积让"哪一轴最危险" 成为可分析问题；(3) 过程级 + 端点级双层度量—— 不只测最终成功，还测过程中暴露时长；(4) "能力 vs 安全" 不应被绑定—— benchmark 应该同时报告 两个轴，让相关性自然显现而非预设；(5) 与 6-23 Benchmark Illusion / 6-25 Hacker-Fixer Loop 共构 H1 末"诊断性 benchmark 三联拍"—— LLM 评测幻象 / verifier 加固 / VLA 具身诊断—— 三篇共同确立"benchmark 必须是诊断工具而非排行榜"的方法学共识；(6) 跨学科扩展—— ForesightSafety-VLA 把诊断性 benchmark 思想从 LLM 推到 VLA，提示 H2 benchmark 链条会进一步覆盖具身 / 多模态 / 物理世界。
