一句话核心贡献
用同一 WorkBench 基准跨 2024 / 2026 两年评测 21 个前沿模型——任务完成率 43% → 88.8%，非预期有害副作用率 26% → 2.5%；Claude Opus 4.8 双双最佳；得出反共识结论 "capability 与 safety 在工作场所 agent 上同向提升而非权衡"——前沿 agent "几乎完全以无害方式失败"；同时透明披露 56 任务（约 14%）ground truth 修订与 ReAct→native tool-calling 工程改进（同模型修订前后差 8pp）；六巨头跨地缘格局：前沿高价区全是西方专有模型，廉价高效区全是中国开源模型，西方廉价梯队（Haiku / GPT-mini/nano）被双面挤压。
摘要（中文翻译，忠实原文）

2024 年 3 月，WorkBench 上表现最好的 agent GPT-4 完成了 43% 的任务，但在其中 26% 的任务中执行了非预期的有害动作（如把邮件发给错误的人）。我们在 2026 年 6 月重新评测该基准，发现迄今最佳 agent Claude Opus 4.8 完成 88.8%，非预期有害动作率仅 2.5%。除了前沿 agent 性能的显著进步，论文指出三大要点：

第一，能力与安全在 WorkBench 上协同提升而非权衡——完成任务最多的模型造成的非预期损害也最少。第二，尽管多类错误已被彻底消除，前沿模型仍偶发不可逆有害错误（如发邮件到错误地址）。第三，开源权重模型的崛起大幅拉低了原本仅专有模型才能达到的性能水准的成本，而前沿模型成本相对稳定。我们发布更新版基准，包含数据与代码质量改进、新模型分数、自 2024 年以来 agent 进展分析。

核心内容解读（纵向重测 / 三大发现 / benchmark 自审 / 跨地缘格局 / 反共识结论）

立论：纵向重测是被忽视的 benchmark 方法学。过去半年的 agent benchmark 论文几乎都是"新基准 + 新评测"姿态——5-25 #1 Boiling the Frog 立"多轮持久工作区"、6-08 #3 AARRI 立"研究实习生级" 颗粒度、6-09 #3 MAC-Bench 立"程序合规性"、6-12 #3 MLUBench 立"终身遗忘"、6-14 #3 SciAgentArena 立"失败模式诊断器"。这条主线方法学价值巨大，但同时掩盖了一个重要问题：当前 agent 进步到底有多快？相比两年前是什么倍数？capability 与 safety 是否真的在权衡？WorkBench Revisited 用最朴素的"同基准重测" 给出第一份高质量纵向证据。

核心实证一：21 模型 leaderboard 揭示前沿 agent 已突破 80% 完成率。论文评测 21 个 2024-2026 年间发布的前沿模型，按完成率从高到低：

核心实证二：capability 与 safety 在工作场所 agent 上同向提升。这是论文最有价值的反共识结论。传统直觉是"能力越强越危险"——例如 jailbreak 文献长期警告"更强模型更容易被诱导越界"。但 WorkBench 数据显示反向：完成率最高的 Claude Opus 4.8 副作用率最低（2.5%）；GPT-3.5-turbo 完成率仅 25.8% 但副作用率高达 38.7%。论文措辞精炼："the strongest agents now fail almost entirely in harmless ways"（最强 agent 现在几乎完全以无害方式失败）。这条 finding 与 6-12 #2 Containment Gap"架构层 safety 必须治理"、5-21 #2 CUGA"policy-as-code"、6-13 MINIM"客户端可信代理" 互补——三者讲的都是系统层防护，WorkBench 讲的是模型层 capability-safety 协同。两条线交叉得出 2026 H1 后期 agent safety 的新共识：capability-safety tradeoff 不是必然的，前提是 (a) 模型本身在工作场所类任务上充分对齐 + (b) 系统层叠加结构化治理。

核心实证四：benchmark 自审与 56 任务修订。论文极其透明披露原 WorkBench 的 7 类问题：(a) "过去 N 天" off-by-one 错误（today−N 应为 today−N+1）；(b) prompt 与答案键不匹配（"more than" vs "less than"）；(c) 静默零聚合 bug（idxmin 丢弃零计数）；(d) 会议推迟违反工作时间规则；(e) 邮件正文 \\n 转义不一致（10 任务全部失败）；(f) 工具描述不清晰；(g) 计数查询不可解（search_tasks 上限从 5 提到 200）。修订效应量化：56 任务（8%）ground truth 或措辞被改 + 准入规则放宽 ≈ 90-95 任务（约 14%）；同一 GPT-4 在新旧基准上完成率 49% → 57%（+8pp）。论文同时弃用 ReAct 文本解析循环，改用 native tool-calling 结构化输出，温度设 0，上限 20 步推理。这条 benchmark 自审姿态与 6-03 #3 Reliability Gap"污染检测在实战中仅 59.4% 正确" 形成"benchmark 自身可被审计 + 必须被修订" 双姿态——是 2026 H1 后期 benchmark 方法学的统一立场。

方法学价值：纵向重测应是 benchmark 标准动作。WorkBench 自 2024 年公开在 GitHub，作者承认"无法排除模型预训练时已'看过'测试数据"，建议未来设立每年刷新的私有保留集。这条 caveat 与 6-09 #3 SciConBench"clean-room evaluation" / 6-12 #3 MLUBench"live benchmark 抗污染" 共同构成 2026 H2 benchmark 防污染方法学的统一姿态。但即便存在污染担忧，纵向重测的方法学价值依然巨大——它把"静态 leaderboard" 升级为"纵向证据流"，是过去六个月 agent 进步速度量化的最权威单一证据。

核心数据亮点 / 关键论点
  • 完成率：43% → 88.8%（GPT-4 2024-03 vs Claude Opus 4.8 2026-06）——两年翻一倍。
  • 副作用率：26% → 2.5%——降至原来约 1/10，前沿 agent 几乎"无害失败"。
  • 同模型修订效应：GPT-4 在新旧基准上完成率 49% → 57%（+8pp）——证明基准修订前后同模型本身分数会变。
  • 21 模型 leaderboard：6 个模型（Anthropic / OpenAI / Google）突破 80%；Kimi-K2.6 等中国开源模型以 1/10 价格匹敌前沿。
  • 跨地缘格局：前沿高价区（>$0.07）全西方专有；廉价高效区（<$0.025 但 ≥ 60%）全中国开源；西方廉价梯队被双面挤压。
  • 56 任务 ground truth 修订（占 8%）+ 准入规则放宽 ≈ 14% 任务受影响——基准方法论级修订。
  • ReAct → native tool-calling 工程升级：弃用文本解析循环，改用结构化输出 + 温度 0 + 20 步上限。
  • 5 类错误演化：完全消除（ReAct 格式 / 日历搜索 / 事件更新）/ 几乎消除（错误邮箱）/ 减少（错误解读 / 搜索上限）/ 未改善（绘制未来日期）/ 新出现（过度积极 / 推理失败 / 轻信截断 / 时序错乱）。
  • 不可逆错误残留：GPT-5.5 仍有 1 次邮件发到 @example——前沿模型仍可能造成不可逆有害行动。
  • 反共识结论："capability and safety go together rather than trade off"——直接挑战传统 safety-capability 权衡担忧。
  • 开源建议：作者建议未来设立每年刷新的私有保留集对抗预训练污染。
对你三个研究方向的启发
Harness Engineering

WorkBench Revisited 给你的 KV-cache 毕设最直接启发："纵向重测" 应是 cache 系统评估的方法学标配。当前 KV-cache 论文绝大多数是"这个月发表 + 与上个月 SOTA 对比"姿态，缺乏"同 workload + 跨 6 个月 / 1 年的纵向证据"。WorkBench 用"同基准 + 同任务 + 跨两年" 证明 capability 翻倍 + 副作用降 10×，这条姿态对 cache 系统同样关键。毕设方法论建议：(a) 设计你的 cache benchmark 时预留纵向重测接口——记录每次重测的 ground truth 修订日志；(b) 借鉴 WorkBench 的 5 类错误分类（完全消除 / 几乎消除 / 减少 / 未改善 / 新出现）作为 cache failure mode 演化报告范式；(c) 借鉴论文对 ReAct 弃用的透明披露——cache benchmark 同样应该详细记录"测试 protocol 升级" 的同模型分数差（WorkBench +8pp）；(d) 借鉴论文跨地缘成本表——你的 cache 系统应同时报告"性能 / 成本" 而非单一指标。论文写作姿态：把 WorkBench Revisited 列为"纵向重测 + 自审基准" 方法学先例。

Agent Skills Safety

WorkBench Revisited 与今日方向 #2 MINIM 在结论上互相支撑——MINIM 用"客户端可信代理" 工程层面证明"safety 可与 utility 兼得"（TCNP-I 99.31% + TISL 0.101 双优），WorkBench Revisited 用纵向重测在模型层面证明"capability-safety 同向提升"。两者合起来构成对"capability-safety tradeoff 不可避免" 共识的双重否证。关键启发：(a) capability-safety tradeoff 是错觉——前提是 (i) 模型在 alignment 训练充分 + (ii) 系统层叠加结构化治理；(b) 不可逆错误是 safety 的硬底限——即便 Opus 4.8 副作用率 2.5%，发到 @example 仍是不可逆错误，需要前置约束（如 MINIM 的 K/A/R 三元策略 + Containment Gap 的 policy gate）；(c) 纵向重测应是 safety 评测标配——同一 jailbreak 攻击在两年间是否仍奏效是关键证据；(d) 跨地缘格局意味着 safety 评测必须同时覆盖中国开源模型，单纯评测 GPT/Claude/Gemini 已不够代表性。

Safety Benchmark（直击方向）

WorkBench Revisited 是 2026 H1 后期 benchmark 方法学最完整的纵向 + 自审 论文之一——它把 6-03 #3 Reliability Gap"污染检测不可靠" 的元层方法学反思推到具体落地："不仅要审计 benchmark，还要修订它，并透明披露修订效应"。建议：未来 safety / agent benchmark 论文应建立四档透明度标准：(1) ground truth 修订日志——每次更新明列哪些 task 被改、为什么、影响多少；(2) 同模型修订前后差——必须报告基准修订引起的同模型分数差（WorkBench +8pp）；(3) protocol 升级报告——ReAct → native tool-calling 类的工程升级必须独立报告；(4) private hold-out / 年度刷新——对抗预训练污染。这条标准在 6-09 #3 SciConBench"clean-room evaluation" / 6-12 #3 MLUBench"live benchmark" / WorkBench Revisited 三处独立成型，是 2026 H2 benchmark 设计的统一立场。另一条关键启发：safety benchmark 不应只测"当前 SOTA 的失败率"，还应测"跨期能力提升的形态"——前沿模型主要在哪些维度变好、哪些维度未改善，是 safety 政策制定的关键证据。
