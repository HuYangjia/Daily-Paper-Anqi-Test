摘要（中文翻译，忠实原文）

随着基础模型与 agent scaffolding 的不断进步，AI agent 已经在复杂的长程编码任务、甚至自主实验执行上展现出强大能力。尽管它们已从研究助理演变为自主研究 agent，但仍在领域敏感性、研究伦理与细微科学判断上存在显著局限——前沿 agent 仍无法完整取代人类研究员。

为弥合差距，作者提出 AARR（Act As a Real Researcher）基准系列。与现有主要评估"宏观执行能力"的 benchmark 不同，AARR 关注 agent 能否在细颗粒度的研究情景中模拟人类研究者所具备的专业性、严谨性与细致推理。

该系列的首个 benchmark 是 AARRI-Bench（Act As a Real Research Intern）——模拟"研究实习生"工作场景。在跨多个 frontier 模型与 agentic 系统的实验中，最佳配置 Mini-SWE-Agent + Claude Opus 4.7 仅获 68.3% 成功率，主要失败模式是频繁忽略人类研究员显而易见的微小但关键的细节。核心洞见：发展"研究员级 AI"需要更深地探究研究行为本身，而不是仅仅依赖更复杂的 scaffolding。

核心内容解读（AARR 立论 / AARRI 设置 / "harness vs scaffolding" 二轴评测 / 失败模式）

立论：把"研究员级 agent"从口号变成可被度量的对象。过去两年的科研 agent benchmark（GAIA / OpenScholar / SciCode / SciAct / TaskBench）大多偏重"能不能跑完一个研究流程"的宏观成功率；本文反过来强调"跑完之后做得专业不专业"。这是 evaluation 思路从"broad-coverage 大纲式"到"granular 实习生式"的下沉，是评估方法学层面的重要补缺。

AARRI-Bench：研究实习生工作场景的细颗粒度任务集。论文把研究生命周期切成多个细化任务，覆盖：

研究阶段实习生级任务样例评估关注点
调研 / 文献回顾找出关键引用、识别 prior work 重叠、判定可信度学术敏感度、不被过期文献误导
方法选型在多种 baseline 中识别"看似合理但实际不可比"科学判断、对照设置严谨性
实验执行正确处理超参与种子、读 logs / debug实验工程素养、错误信号敏感度
结果整理表格 / 图表、显著性判定、误差范围统计严谨、不夸大结论
研究伦理识别数据使用边界、引用规范、自我抄袭警告研究伦理与发表规范

这些维度不是"能不能完成宏观目标"，而是"过程中有没有犯研究新人不该犯的错"——这与 SWE-Bench 等"能不能 pass 测试"的 binary 信号正好互补。

"模型 × harness"二轴评测——本文的方法学创新。过去 benchmark 一般只评测一个轴：要么固定 agent scaffold 比模型，要么固定模型比 scaffold；本文坚持两个轴一起报，因此你能在 paper 里看到"Mini-SWE-Agent + Claude Opus 4.7"这样的组合最高 68.3%——这是把 harness 工程视作可被独立度量的 first-class 对象，与今日方向 #1 Lean4Agent、5-26 #1 From Model Scaling to System Scaling 的方向论遥相呼应。这是 2026 年 benchmark 的新基线规范：必须同时报模型与 harness 的组合贡献度。

失败模式：忽略人类研究员显而易见的细微但关键细节。这是核心叙事 punchline。它的影子在多个上半年工作中出现过——5-15 #1 ChromaFlow 在 GAIA-2023 L1 上"加 orchestration 反而变差"是同一现象的早期信号；5-22 系列的 multi-agent debate 工作也描述过类似 "多人对线反而漏细节"。AARRI 这次的贡献在于把这个一直在直觉里的现象放到 5 维（领域敏感、对照设置、伦理、统计、引用）的系统化考核里——以后任何科研 agent 都得在这 5 维上接受体检。

32 pp 的 gap：人类 94.0% vs 最强 agent 68.3%。这不是宏观成功率的 gap，而是"专业素养"的 gap。两者的差异在评估科学性上意义截然不同：宏观成功率可以被"更多采样 + 投票"或"scaffolding 加层数"在数月内追平；而专业素养的差距需要训练分布与反馈机制的根本调整。这给 AI 研究员的接下来 12 个月画了一条清晰的研究路线：scaffold 已经不是瓶颈，研究行为本身的训练数据 + 形式化反馈才是。

与今日 #1 / #2 的协同。把今日三篇放一起：(a) 方向 #1 Lean4Agent 提供"plan 是否可证明对"的形式化裁判；(b) 方向 #2 Attack Selection 提供"red-team 时机维度"的评估方法学修正；(c) 本文 AARRI 提供"研究员级专业素养"的 granular 评测。三者一起构成"正确 / 抗攻击 / 专业"三轴 benchmark 体系——不再是"跑通即过"的浅评估，而是"正确 + 抗时机攻击 + 学术专业"的三层校验。

对你三个研究方向的启发
Harness Engineering（直击毕设）

"模型 × harness 二轴评测"是你毕设可以直接借用的 figure 模板：把"稀疏注意力 KV-cache 预取 + 双水位线 GPU 热缓存"作为一个 harness 维度，让它和不同 base model（LLaMA / Qwen / GPT-OSS）在同一组任务上做联评。这样毕设 paper 不再是"给某个固定模型加速"，而是"该 harness 在不同模型上的边际增益曲线"——后者直接对应 AARRI 的"组合 = 68.3%"叙事范式。具体建议：把 AARR 列入 Related Work 章；用"模型 × harness 二轴"作为你 evaluation section 的组织维度；并在 Discussion 引用 AARRI 的 32 pp gap，论证你的 harness 在细颗粒度命中场景（cache miss、prefetch 误中）也存在类似的 specialization gap。

Agent Skills Safety

AARRI 的"研究伦理"维度其实就是 agent safety 的微缩版：当 agent 写论文时，识别"数据使用边界"、"引用规范"、"自我抄袭"等隐性约束，与今日 #2 Attack Selection 中蓝队 monitor 要识别的"违规时机"是同源问题——都是"不该做" 的 fine-grained 检测。建议：把 AARRI 的伦理子集与 control evaluation 拼起来，让被监控的 agent 在科研工作流中分别面对"start-policy 攻击者"和"伦理违规者"两类对手——这能给 trusted monitor 提供更立体的训练信号，也是把 AI control 从纯安全学语境向"研究伦理"扩张的自然路径。

Safety Benchmark（直接命中）

本文给 2026 年 benchmark 论文树立了三条新规范：(1) 必须报告"模型 × harness"二轴贡献度，单轴评测从此被视为不完整；(2) 必须区分"宏观成功率" vs "细颗粒度专业素养"——前者会被 scaffolding 拉满，后者才反映真实 capability gap；(3) 必须暴露"显而易见的细微细节" 失败模式，这条单点错误的覆盖率比综合分数更能体现专业级评估。建议未来 benchmark 应同时报告：(i) task-success（宏观）+ (ii) verification-yield（结构性正确率，今日 #1 启发）+ (iii) detail-coverage（专业素养，今日 #3 启发）+ (iv) attack-selection-aware safety（今日 #2 启发）四维；只有四维齐全的 benchmark，才能在"真实部署可信度"上给出有意义的判断。
