一句话核心贡献
把 AI4Science 安全评测的"风险维度"从隐含的次级标签升级为与"学科" 平行的一级组织轴——构建 7 学科 × 31 子学科 × 10 风险维度 × 350 样例的方阵，让评测能同时回答"哪个学科最危险" 与"哪类风险驱动了不安全行为" 两个问题；并在统一基准上首次实证：science-specialized 模型在大多数风险维度上反而 ASR 更高，最弱维度 Knowledge cutoff drift 74.2%、最强 Privacy leakage 12.2%；最危险学科 Engineering 57.0%、最安全 Biology ~18.8%；学科内子学科异质性显著（E-1 电气/电子 远高于 E-5 软件/系统）。
摘要（中文翻译，忠实原文）

大语言模型（LLM）正越来越多地嵌入到 AI for Science（AI4Science）工作流中，从科研问答、文献分析到实验室规划与自主科学发现。这种进展带来对安全基准的迫切需求——基准不仅要评估科研能力，还要评估模型是否能识别并避免高风险科学情境中的风险。现有 AI4Science 安全数据集涵盖了若干学科与任务格式，但底层风险维度仍然欠规范。我们提出 SciRisk-Bench，一个从两个互补视角评估 AI4Science 安全的基准：显式风险维度 与科学学科。SciRisk-Bench 覆盖 7 个学科、31 个子学科与 10 个风险维度。在实验部分我们针对主流通用 LLM 与科学专用 LLM，跨风险维度、学科和子学科进行评测，使我们能够细粒度诊断科学模型在哪里仍然不安全。

核心内容解读（10 风险维度 / 31 子学科 / ASR 表 / 三大反共识）

立论：AI4Science 安全评测必须把"风险维度" 与"学科" 同时作为一级轴。论文回顾既有 benchmark 谱系：科学能力（SciBench / ScienceQA / GPQA / SciEval / SciKnowEval）→ 领域专用安全（ChemSafetyBench / MedSafetyBench / LabSafetyBench）→ 跨域科学安全（SciSafeEval / WMDP / SOSBench / SafeScientist）→ 通用安全（TruthfulQA / HaluEval / HarmBench / SafetyBench）。共同问题：风险维度 要么欠规范，要么仅作为次级标签。SciRisk-Bench 提出：把 risk-dimension awareness 提升到一级，与 discipline 维度正交交叉——这才能"看清不安全行为是从哪个风险类别里冒出来的"。这种姿态升级与今日方向 #1 Xcientist"9 维 audit"（科学问责）和今日方向 #2 SRP"10 类 unsafe 标签"（safety reflection 类别化）共振。

系统设计二：7 学科 × 31 子学科覆盖。Astronomy 4（A-1 ~ A-4 含空间探索 / 观测 / 恒星 / 行星/宇宙学）；Biology 5（B-1 ~ B-5 生物信息合成生物 / 分子发育 / 生态进化 / 病原毒理药 / 解剖生理神经）；Chemistry 4（C-1 ~ C-4 分析 / 无机配位 / 有机合成 / 生化）；Geography 5（G-1 ~ G-5 城市经济 / 人文政治 / 气候气象 / 物理地形 / 制图 GIS）；Engineering 5（E-1 ~ E-5 电气电子 / 结构土木 / 机械制造 / 化学过程 / 软件系统）；Mathematics 3（M-1 ~ M-3 纯数学 / 统计概率 / 应用数值）；Physics 5（P-1 ~ P-5 电磁光学 / 量子力学 / 核粒子 / 热力学统计 / 经典力学）。子学科合计 31 ✅；样例 350 ✅。

核心实证一：学科级 ASR（Figure 5）——Engineering 57.0%（最高）/ Chemistry ~50% / Astronomy ~50% / Physics 中等 / Geography 中等 / Mathematics 较低 / Biology ~18.8%（最低）。三大反直觉子学科级发现（Figure 6）：E-1 电气/电子工程 子学科 ASR 最高；E-5 软件与系统工程显著低于其他工程子学科；C-1 分析化学 在化学中最高；A-1 / A-2 空间探索/观测天文 在天文中最高；G-1 城市经济地理 远高于 G-5 制图 GIS；P-1 / P-2 电磁光学/量子 在物理中最高，P-5 经典力学 最低——揭示学科内异质性远大于学科间差异。

核心实证二：science-specialized vs mainstream base 模型对比（Figure 4 / Figure 7）。论文文本未列具体模型名（细节在 Figure 2 热图与 Table 2 / Figure 4 / 7），但定性结论清晰：mainstream base 模型主要不安全维度为 knowledge cutoff drift / safety omission / laboratory safety；science-specialized 模型在上述维度依然高，并且 dual-use / authority inflation / hallucination / fringe amplification 也升高，风险轮廓更"宽且均匀"。最大差距学科：数学、物理、化学、生物（science-specialized 显著更差）；唯一例外：Astronomy（base 与 specialized 相当或 base 更高）。反共识结论：科学专精化 ≠ 更安全——领域 fluency 增加"答应能力"但未增加"风险鉴别"。这条结论与 5-13 #2 SKILL.md 系列里"skill-fluency 与 skill-safety 解耦" 形成跨场景同源警示。

核心实证三：最弱类别并非显式恶意请求。Safety omission（53.5%）/ Knowledge cutoff drift（74.2%）/ Laboratory safety（高）这三类最危险 的维度，本质都是"隐式失误"——不安全行为隐藏在正常的科学协助之中。这与今日方向 #2 SRP 在 MedSafetyWorld 上的发现"不安全行为可由安全数据泛化" 完全同源——两者都说明：safety 评测和 alignment 不能仅以"显式恶意"为靶，必须覆盖"看似良性但隐式不安全"的全谱系。

方法学姿态：把 safety benchmark 从"总分"推进到"双轴诊断器"。SciRisk-Bench 的核心方法学贡献是提升 risk dimension 为一级轴——这条路径的下一步自然就是三轴扩展（学科 × 子学科 × 风险维度 × 时间快照），把 6-15 #3 WorkBench Revisited"纵向重测" + 6-17 #1 SEAGym"5 视图" + 本论文"风险维度方阵" 合成"科学问责立体评测面"。Limitation 三项：仅文本评估（未覆盖多模态显微图像 / 地理栅格 / 分子结构 / 实验视频）；风险定义为快照（未来需动态更新）；未与领域治理标准更强集成。三项均预示后续 v2 / v3 自然演进路径。

  • 规模：7 学科 × 31 子学科 × 10 风险维度 × 350 样例（Mathematics 58 / Chemistry 50 / Biology 50 / Astronomy 50 / Physics 50 / Geography 47 / Engineering 45）。
  • 风险维度样例分布：Hallucinations 118 / Dual-use 53 / Fringe amplification 38 / Knowledge cutoff drift 27 / Regulatory blind spot 27 / Lab safety 26 / Safety omission 25 / Authority inflation 17 / Privacy leakage 11 / Geopolitical sensitivity 8。
  • 风险维度 ASR 极值：Knowledge cutoff drift 74.2%（最弱）→ Safety omission 53.5%（次弱）→ Privacy leakage 12.2%（最强）。

  • 核心反共识 1：science-specialized ≠ safer（领域 fluency 增加"答应能力" 但未增加"风险鉴别"）。

对你三个研究方向的启发
Harness Engineering

SciRisk-Bench 的"风险维度作为一级轴" 与今日方向 #1 Xcientist"9 维 audit + Claim Drift 三类" 形成"风险类型化 + 失效类型化"双轴对偶——前者把科研结果的潜在危害 类型化（10 维），后者把科研过程的可问责失效 类型化（语义/实验/机制 3 类）。统一启发：未来 research harness 应同时具备 10 risk-dimension audit（生成的 idea / experiment / claim 是否触发 dual-use / safety omission / knowledge cutoff drift 等任一维度）与 3 类 Claim Drift 检测。对你 KV-cache 毕设的启发是：cache 系统的"风险维度"也应类型化（如：缓存污染 / 侧信道泄露 / 一致性失效 / 性能 misreport / safety-relevant data eviction）——这与 6-17 #1 SEAGym 5 视图正交，可形成"5 视图 × N 风险维度"立体诊断面。

Agent Skills Safety

SciRisk-Bench 揭示的"science-specialized 模型反而更危险" 与"最危险类别是 safety omission / knowledge cutoff drift 等隐式失误"两条结论，对 agent skills safety 有直接警示：(1) skill-specific fine-tuning 不是 safety 的银弹——agent 越擅长某 skill，越容易"顺手就做"而忽略 safety；这与今日方向 #2 SRP 主张的"反思必须穿过整个生命周期" 完全同源。(2) skill safety 评测不能只测显式恶意请求——必须包含隐式失误维度（agent 调用 tool 时遗漏前置安全检查 / 调用过期 API 给出过时合规建议 / 把不确定输出当作权威建议给用户等）。(3) skill safety 评测必须区分 skill 内的子能力——SciRisk-Bench 的"子学科异质性 > 学科间差异" 在 agent 上对应"同一 skill 不同子操作的安全性差距巨大"（如同样是 web-browsing skill，表单填写 vs 下载文件 的 safety 差异可能远大于 web-browsing 与 coding 之间的差异）。

Safety Benchmark（直击毕设）

SciRisk-Bench 是 2026 H1 后期 safety benchmark 设计的方法学样板：(a) 把次级标签升级为一级轴（风险维度从隐含 → 显式平行学科）；(b) 正交多维方阵设计（学科 × 子学科 × 风险维度，350 样例就能覆盖 7×31×10=2170 元胞中的关键格点）；(c) 三粒度报告（风险维度级 / 学科级 / 子学科级）；(d) 跨模型族对比（mainstream vs science-specialized）；(e) 三级数据来源优先（法规 → 已有数据集改写 → LLM 生成）。这套方法学的"下一步" 已经清晰：(1) 多模态扩展（论文明示 limitation）—— 把 risk-dimension benchmark 拓到显微图像 / 地理栅格 / 分子结构 / 实验视频；(2) 纵向化—— 与 6-15 #3 WorkBench Revisited 同步重测，看模型升级是否在不同风险维度上同向收敛；(3) 动态更新—— 与法规更新挂钩，做knowledge-cutoff-aware benchmark refresh。SciRisk-Bench 的风险维度 × 学科方阵设计可直接用作你 safety-benchmark 毕设方向的评测协议模板。
