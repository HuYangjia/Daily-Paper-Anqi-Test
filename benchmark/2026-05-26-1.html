摘要（中文翻译，忠实原文）

由多模态大语言模型（MLLM）驱动的自治计算机使用 agent，作为完成复杂数字工作流的能干助手正在崛起。然而，真实世界的执行环境远非理想：弹窗、分辨率变化、竞争应用 频繁干扰 agent 的感知与控制。

我们引入 AgentHijack，一个旨在评测计算机使用 agent 在常见腐蚀下稳健性的 benchmark——这里"动态环境的不确定性扰乱执行流，而没有直接的对抗意图"。具体而言，AgentHijack 引入 9 种可配置的常见腐蚀，复现真实的非理想场景。

我们对一系列利用 MLLM-based agent 完成的桌面任务进行评测，发现即使是轻度腐蚀也会导致显著的性能退化——这强调了 agent 的脆弱性，以及对稳健性评测的必要性。随后，我们提出 AgentHijack-Agent——一个整合了"具备增强 grounding 能力的 action generator"和"负责行为汇总与环境检查的 onlooker"的框架。大量实验验证了它的有效性。

代码、环境、基线模型与数据均公开可用。

核心内容解读（9 类腐蚀 · onlooker 设计 · 与 robustness 经典系列的对比）

问题：为什么"无意干扰" benchmark 比 prompt 注入 benchmark 更有用？过去一年的 agent 安全 benchmark 几乎都聚焦于有意攻击——MCP tool description poisoning（5-22 候选）、TTT-Undermines-Safety、Evo-Attacker、prompt injection 等等。但真实部署中 99% 的失败不是攻击——是浏览器弹窗挡住了按钮、是 4K 屏的分辨率把 grounding 推到了像素边缘、是 IM 通知卡了一下界面、是另一个应用抢焦点。AgentHijack 的核心 framing 转变是：把 agent 的稳健性测评，从"对抗鲁棒性"扩张到"常见腐蚀（common corruptions）"——这与 vision 社区 ImageNet-C / CIFAR-C 的方向论一致：当 corruptions 是真实分布时，benchmark 的工业意义会更大。

AgentHijack-Agent 防御框架的两条柱：

  • action generator with enhanced grounding——把视觉 grounding 模型升级，使坐标 / 元素决策不被 corruption 推偏；
  • onlooker for behavior summarization & env check——这是论文最有意思的设计：用一个"旁观者 agent"周期性 summarize 行为并检查环境（"现在屏幕上有没有奇怪的东西""上一步动作有没有真的生效"），把"sanity check"从内嵌的步内逻辑挪到外部的元 agent。

与昨日 #1（PoisonForge）/ #1 报告（Boiling the Frog）的对比：PoisonForge 是"1% 训练集中毒"——攻击者侧的；Boiling the Frog 是"9 步多轮 escalation 攻击"——也是攻击者侧的。AgentHijack 是第一个把"无意环境干扰"视作 benchmark 一等对象的工作。三者拼起来正好覆盖 agent 安全评测的三个面：训练 / 多轮交互 / 环境。

  • 9 configurable common corruptions——首次为 computer-use agent 给出"类 ImageNet-C 风格"的可参数化 corruption 矩阵。
  • "轻度腐蚀即可显著退化"——论文摘要明确指出，即使是非对抗、非恶意的扰动，frontier agent 表现也会塌方。

  • 开源完整三件套：code + environment + baseline + data——是直接拿来作 benchmark 基线的最佳产物形态。

对你三个研究方向的启发
Harness Engineering

AgentHijack 的"onlooker" 与今日 #1 Harness 报告的"verification layer + governance layer"完全对得上——onlooker 就是 verification 层的具体实现形态。这给了你毕设答辩一个很妙的横向类比："就像 computer-use agent 需要 onlooker 检查环境，KV-cache 系统也需要一个旁观者去监控双水位线、热缓存命中率与卸载收益是否退化"——你可以把 prefetcher 的元监控写得比工程实现更高一个抽象层。

Agent Skills Safety

"无意干扰"是对 agent skills safety 研究的一次重要"扩域"——它告诉防御方：你的 guardrail 不能只防恶意 prompt，真实部署里的安全事故大多由"无意环境腐蚀 + 模型不稳健"共同造就。把今天 #1 Agent Safety 的 Evo-Attacker（恶意攻击）+ AgentHijack（无意腐蚀）放在一起，构成 safety 训练 / 评测的最小完备 corpus。
