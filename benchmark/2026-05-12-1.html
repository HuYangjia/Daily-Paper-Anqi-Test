摘要（中文翻译，忠实原文）

Autonomous agent 已迅速成熟为任务执行者，并通过 OpenClaw 等 harness 大规模部署。safety 已应得地获得越来越多研究关注，而藏在 safety 之下、悄然引导 agent 行为的，是价值（values）。然而现有的价值 benchmark 仍局限于 LLM，agent 的价值仍是空白。

我们从直觉、实证、理论三个角度论证：agent 的价值与其底层 LLM 的价值会发生分歧；并且 agentic 模态在数据集、评测、系统三个层面引入了纯文本协议中不存在的挑战。

我们用 Agent-ValueBench 弥合这一空白——首个专门针对 agent 价值的 benchmark：覆盖 16 个领域下的 394 个可执行环境，提供 4,335 个价值冲突任务，覆盖 28 个价值系统与 332 个维度。每个实例均经我们专门设计的端到端 pipeline 协同合成，并由专业心理学家逐例策展。每个任务附带两条极对齐黄金轨迹（pole-aligned golden trajectories），其检查点支撑了基于 rubric 的轨迹级判官。

我们对 14 个 frontier 专有/开源权重模型 × 4 主流 harness 进行 benchmark，得到三条相互呼应的发现：

(1) Agent 价值首先表现为"价值潮汐（Value Tide）"——跨模型的同质性，其下藏着可解释的反向潜流；

(2) 这一潮汐在harness 拉力下非加性地弯曲；

(3) 在嵌入式 skill 的有意 steering 下变形更为决定性。

这些结果合在一起表明：agent 对齐的杠杆正从经典的"模型对齐+prompt steering"转向"harness 对齐+skill steering"。

核心内容解读（背景·方法·差异）

解决了什么问题：过去两周的 safety benchmark 工作主要在三条线推进——XL-SafetyBench 跨语种 / 跨文化 jailbreak、Partial Evidence Bench 权限-完整性、ML-Bench&Guard 多语种 policy-grounded。这些都基于"LLM 拒绝/接受单次 prompt"的范式。Agent-ValueBench 攻的是另一条线：价值不是单次 prompt 的拒绝，而是多步 trajectory 的价值取舍——而且关键论点是：agent 的价值并不等于其底层 LLM 的价值。这是把 alignment benchmark 从"输出级"推进到"行为级"的关键一步，对应了价值哲学/伦理学早就提出但 LLM 时代尚未在 agent 层面 operationalize 的概念。

核心方法（环境-任务-价值系统三层 + 黄金轨迹）：

  • 规模空前的价值评测层级：16 域 × 394 环境 × 4,335 价值冲突任务 × 28 价值系统 × 332 价值维度。这一规模是当前 LLM 价值 benchmark（典型 1k-2k）的 2-4×，且第一次把"可执行环境"作为评测单元。
  • 专业心理学家 per-instance curation：每个任务都由专业心理学家逐例核对——这在 AI benchmark 里非常罕见，把 noise floor 显著压低，给"价值"这种主观概念注入跨学科严谨度。
  • pole-aligned golden trajectories（极对齐黄金轨迹）：每任务两条——一条朝价值 A 极端、一条朝价值 B 极端，把"价值取舍"做成可评分的轨迹空间几何。trajectory checkpoint 锚定 rubric judge，这是把价值评测做成"轨迹级"的关键工程设计。
  • 4 主流 harness × 14 模型横切：这是该工作最有 systems 风味的部分——把模型与 harness 当成两个独立变量做 ablation，第一次在大规模上看清"harness 配对模型如何改变价值表达"。
  • 三层挑战明示化：论文明确指出 agentic 模态比纯文本多三层 challenge——dataset 层（无现成 trajectory 标注）、evaluation 层（rubric 必须考虑 trajectory）、system 层（harness/skill 是 explicit variable）。这是给后来者的工程清单。

核心数据亮点 / 关键论点
  • 394 可执行环境 × 4,335 价值冲突任务 × 28 价值系统 × 332 维度——目前规模最大、维度最齐全的 agent value benchmark；专业心理学家逐例策展极大压低 label noise。
  • "Value Tide（价值潮汐）"：14 个 frontier 模型在 agent 模式下表现出强烈的跨模型同质性（共趋势），潮汐之下藏可解释的反向潜流——这是该领域第一次把"agent 价值的群体性"做成可量化现象。
  • harness 非加性弯曲：同一模型搭不同 harness，价值表达发生非线性变化——意味着"模型对齐"在 agent 部署里只是一部分，harness 是不可忽略的耦合变量。
  • skill steering 决定性：嵌入式 skill 对价值的 steering 比 harness 拉力更具决定性——这给"agent 安全 = 模型 safety"这一旧范式划下界限。
  • 对齐杠杆迁移："classical model alignment + prompt steering" → "harness alignment + skill steering"——这一论断与今日 Harness 方向 ComplexMCP 强调"工具拓扑作为 harness 第一原则"形成呼应。
  • 方法学影响：pole-aligned golden trajectory + rubric checkpoint 是对 trajectory-level evaluation 的一个高质量 reference 模板，可作为后续 agent benchmark 的范式 baseline。
对你三个研究方向的启发
Harness Engineering

"价值在不同 harness 下非加性弯曲" 给 harness 工程提出一个非常重要的结构性观察：harness 选择本身是 alignment 工具的一部分。今日 Harness 方向 ComplexMCP 把工具拓扑作为评测变量，Agent-ValueBench 进一步把 harness 作为价值变量，两者拼起来等于在说："工具拓扑 + harness 配置 + 模型选择" 是 agent system 的三轴 design space，单看一轴会得出错误结论。这给 ZipperQuant 的 e2e benchmark 一个直接启示：报 W4A4 vs BF16 时不能只在一个 harness 上比较，至少要跨 2-3 种主流 harness（如 vLLM / sglang / 自研 ZipperEngine）做 cross-harness 对照，避免单 harness "局部最优"误导结论。

Agent Skills Safety

"skill steering 比 harness 拉力更决定性" 与今日 Agent Safety 方向 PRISM 的 per-token IFC 形成完美互补：skill 是高层价值钥匙，PRISM 是低层信息流闸口。把两者拼起来，可以构建一个完整的 "skill steering + token IFC" 双层 alignment 架构——高层用 skill curation 引导价值倾向，低层用 PRISM 在 decoding 阶段拦截敏感信息流泄漏。同时 Agent-ValueBench 的"两条极对齐 trajectory"思路非常适合做 PRISM 的对抗 stress test：用一条朝"过度泄漏"价值偏移的 golden trajectory，看 PRISM 是否能稳定拦截。

Safety Benchmark

Agent-ValueBench 的 trajectory-level evaluation 给 safety benchmark 的方法论引入两个新原则：(1) 评测单元从 prompt-response 升级到 trajectory + checkpoint rubric；(2) 把 harness 当显式变量做 ablation。两者结合，未来 safety benchmark 应该报告"(模型, harness) 对在 trajectory 空间上的覆盖与漂移"，而不是单一聚合分数。从你正在筹备的 ZipperQuant e2e quality 评估来看，这一思路可以直接搬用：用 BF16 baseline 生成"golden trajectory"，看 W4A4 在 trajectory 中哪些 checkpoint 最容易漂移（特别是数学推理 / 代码执行的关键 step），把"质量损失"细化到轨迹位置而非单一困惑度均值。
