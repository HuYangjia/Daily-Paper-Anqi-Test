纳入论文逐篇分析（19 篇）

1) ArMOR: Defending Against Consistency Model Mismatches in Heterogeneous Architectures

ISCA 2015 · Conference · 检索入口

一句话：首次系统化处理异构 MCM 不匹配问题，把“跨架构一致性”变成可自动推理任务。
核心分析：方法价值在于把 heterogeneous memory models 从经验兼容转成 algorithmic translation + checking；它为后续 TriCheck 与 ILA-MCM 提供了“跨层一致性必须形式化证明”的起点。

2) Counterexamples and Proof Loophole for the C/C++ to POWER and ARMv7 Trailing-Sync Compiler Mappings

arXiv 2016 · Preprint · arXiv

一句话：证明“看似成立”的编译映射可能存在严谨性漏洞，并给出可执行反例。
核心分析：这篇工作是 TriCheck 爆发前夜的关键铺垫：它把“证明文本正确”升级为“机器可证且可反例驱动”，直接推动后续全栈 MCM 验证范式。

3) TriCheck: Memory Model Verification at the Trisection of Software, Hardware, and ISA

ASPLOS 2017 · Conference · arXiv

一句话：提出软件-ISA-微架构三岔口的全栈 MCM 验证框架，发现并推动修复 RISC-V 规范问题。
核心分析：TriCheck 的最大贡献不是单点算法，而是“验证对象重定义”：从单层 correctness 变成端到端 consistency contract；这也奠定了 Trippel 后续契约式安全验证路线。

4) CheckMate: Automated Synthesis of Hardware Exploits and Security Litmus Tests

MICRO 2018 · Conference · PDF

一句话：把硬件安全验证从“手工构造攻击”推进到“形式化攻击模式自动合成”。
核心分析：CheckMate 借助 µhb 图与 axiomatic µspec，把 exploit class 变为可判定约束；其意义在于可重复、可扩展的 security verification pipeline，而不仅是单次攻击展示。

5) ILA-MCM: Integrating Memory Consistency Models with Instruction-Level Abstractions for Heterogeneous SoC Verification

FMCAD 2018 · Conference · 检索入口

一句话：把 ILA 与 MCM 接轨，解决 heterogeneous SoC 场景下“组件局部正确但系统全局错误”的验证断层。
核心分析：这篇论文的工程价值很高：让 formal spec 能跨 IP block 组合推理，而不是停留在单核/单模块验证。

6) MeltdownPrime and SpectrePrime: Automatically-Synthesized Attacks Exploiting Invalidation-Based Coherence Protocols

arXiv 2018 · Preprint · arXiv

一句话：展示 coherence protocol 维度的自动合成攻击，扩展 Meltdown/Spectre 家族。
核心分析：它验证了 CheckMate 不是“只会复现已知攻击”，而是可发现新 exploit surface；对 arch/chip formal 的意义是：安全验证要覆盖微架构事件序和一致性协议层。

7) TransForm: Formally Specifying Transistency Models and Synthesizing Enhanced Litmus Tests

ISCA 2020 · Conference · 检索入口

一句话：把 transistency 作为可形式化对象，并自动合成更有诊断力的 litmus 测试。
核心分析：该工作延续 TriCheck 思路，进一步表明：评测质量来自模型质量；litmus 自动增强能显著提升发现边界行为的能力。

8) Synthesizing Formal Models of Hardware from RTL for Efficient Verification of Memory Model Implementations

MICRO 2021 · Conference · PDF

一句话：将 RTL 自动抽取成可验证 formal model，降低 MCM 验证的人力门槛。
核心分析：这是从“理论可验证”走向“工程可落地”的关键步骤：把 hand-written model bottleneck 变为 synthesis problem，使大规模验证成为现实。

9) Relational Models of Microarchitectures for Formal Security Analyses

arXiv 2021 · Preprint · arXiv

一句话：用关系模型抽象微架构行为，为形式化安全分析提供统一语义底座。
核心分析：该文连接了“微架构细节”与“安全性质证明”，并为后续 Axiomatic Contracts/RTL2MµPATH 等方法提供建模思想。

10) Axiomatic Hardware-Software Contracts for Security

ISCA 2022 · Conference · 检索入口

一句话：提出可验证的 HW-SW 安全契约，把“假设”升级为“可证明约束”。
核心分析：贡献在于契约化：把软件防御的硬件前提显式化、可审计化；该思路已直接影响后续 Serberus 与 MemGlue 路线。

11) Scalable Assurance via Verifiable Hardware-Software Contracts

OSCAR 2022 · Workshop · CV 来源

一句话：将契约验证推进到可扩展保证（scalable assurance）视角。
核心分析：虽然是 workshop short，但它明确了该路线的工程化目标：不仅“能证”，还要“可规模化地持续证”。

12) G-QED: Generalized QED Pre-silicon Verification beyond Non-Interfering Hardware Accelerators

DAC 2023 · Conference · 检索入口

一句话：拓展 QED 预硅验证能力到更复杂、存在交互影响的硬件场景。
核心分析：该文把传统 QED 适用边界外推，强化了 formal pre-silicon verification 在真实 SoC 复杂交互中的实用性。

13) Towards Improving Verification Productivity with Circuit-Aware Translation of Natural Language to SystemVerilog Assertions

DAV 2023 · Workshop · CV 来源

一句话：尝试把自然语言需求转成 SVA，提高形式化验证入口效率。
核心分析：方法层意义很强：它解决的是“formal specification creation”这一长期痛点，降低验证工程师写属性的成本。

14) Design for Hardware Memory Model Verification

PLARCH 2023 · Workshop · CV 来源

一句话：从设计方法角度提出“为可验证而设计”的 memory model workflow。
核心分析：这是研究策略层贡献：把验证后置纠错改成前置设计约束，有助于降低架构演进中的一致性回归风险。

15) RTL2MµPATH: Multi-µPATH Synthesis with Applications to Hardware Security Verification

MICRO 2024 · Conference · Lab Publications

一句话：把多执行路径合成引入 RTL 安全验证，自动发现更隐蔽的硬件侧信道。
核心分析：相对 CheckMate 的主要进步是从高层 exploit pattern 到 RTL path-level synthesis，进一步缩短“漏洞发现”到“实现定位”的距离。

16) Memory Consistency Model-Aware Cache Coherence for Heterogeneous Hardware

FMCAD 2024 · Conference · Lab Publications

一句话：让 cache coherence 设计显式感知 MCM 约束，避免“coherence 正确但 consistency 违规”。
核心分析：该文连接了两个常被割裂的层：一致性模型语义与缓存协议实现；对于异构芯片尤为关键。

17) Automatically Uncovering Hardware Side-Channels in Processor RTL with Multi-µPATH Synthesis

OSCAR 2025 · Workshop · CV 来源

一句话：MICRO24 方法在开放架构场景继续扩展，强调自动化 side-channel 挖掘能力。
核心分析：作为 short paper，它体现了方法迁移与可复用性，说明 RTL2MµPATH 并非一次性结果。

18) Formal Foundations of the MemGlue Consistency Protocol for Heterogeneous Hardware

FMSD 2025 · Journal · CV 来源

一句话：为 MemGlue 协议提供更完整的形式化基础，增强理论完备性与可证明性。
核心分析：期刊版本通常承担“理论收敛”角色，本工作正是把 conference 的工程结果提升为更系统的 formal foundation。

19) Specification and Formal Verification of Hardware-Software Contracts for High-Assurance Computer Architectures

IEEE Computer 2025 · Magazine · Stanford Profile

一句话：总结硬件-软件契约的规格化与验证路线，形成高保障架构的统一叙事。
核心分析：它不是“新算法”论文，而是路线图论文；对你后续追踪价值在于把多个零散成果归入可执行的 research program。

整体脉络总结（2015-2025）
  • 阶段一：MCM 形式化验证（ArMOR/TriCheck/ILA-MCM）建立跨层一致性方法论。
  • 阶段二：把形式化思路扩展到硬件安全（CheckMate/MeltdownPrime/Contracts）。
  • 阶段三：推进 RTL 级自动化与可扩展验证（RTL synthesis、Multi-µPATH、MemGlue）。
  • 阶段四：降低 formal 入口成本（NL→SVA / NL→Temporal Logic）以提升生产效率。
