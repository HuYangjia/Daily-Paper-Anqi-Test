逐篇分析（补全集）

1) Protean: A Programmable Spectre Defense (HPCA 2026)

Lab Publications
  • 问题：Spectre 防护策略碎片化、难随场景动态调整。
  • 方法：可编程防护机制，把防御策略参数化并可组合。
  • 学习要点：防护从“固定补丁”转向“可编排策略层”。

2) Automating Requirements Formalization (ICSE 2026)

Lab Publications
  • 问题：需求到形式规格转换耗时且语义易偏差。
  • 方法：LLM + distinguishing traces 做语义校验闭环。
  • 学习要点：规格生成必须配“可区分轨迹”做反例校正。

3) Analyzing and Exploiting Branch Mispredictions in Microcode (arXiv 2025)

Lab Publications
  • 问题：microcode 层分支预测失误可形成新攻击通路。
  • 方法：系统分析失预测利用条件并构造利用路径。
  • 学习要点：安全边界要下探到 microcode 级细节。

4) VeriCoder: Enhancing LLM-Based RTL Code Generation ... (arXiv 2025)

CV
  • 问题：LLM 生成 RTL 正确性不可控。
  • 方法：引入 functional correctness validation 反馈环。
  • 学习要点：代码生成和形式验证应“共设计”而非串行。

5) Automatically Uncovering Hardware Side-Channels ... (OSCAR 2025)

CV
  • 问题：侧信道路径空间大、手工分析难覆盖。
  • 方法：基于 Multi-µPATH 合成自动发现路径。
  • 学习要点：自动路径合成是 RTL 安全评估关键能力。

6) Formal Foundations of MemGlue ... (FMSD 2025)

CV
  • 问题：异构一致性协议需要更严格理论基础。
  • 方法：给出 MemGlue 协议形式化语义与性质证明框架。
  • 学习要点：期刊版常用于沉淀“可证明边界”。

7) Specification and Formal Verification of HW-SW Contracts ... (IEEE Computer 2025)

Stanford Profile
  • 问题：契约式验证如何形成可推广方法论。
  • 方法：总结 contract 规范、验证与系统落地路径。
  • 学习要点：适合作为你建立“个人验证框架”的纲领文。

8) RTL2MµPATH (MICRO 2024)

Lab Publications
  • 问题：RTL 层攻击路径难穷举。
  • 方法：多路径合成 + 安全验证任务实例化。
  • 学习要点：连接“实现细节”与“可验证安全性”。

9) Memory Consistency Model-Aware Cache Coherence ... (FMCAD 2024)

Lab Publications
  • 问题：coherence 与 MCM 不协同会出现隐藏违规。
  • 方法：MCM-aware 协议设计与形式化检查。
  • 学习要点：协议正确性必须做跨层语义验证。

10) Translating NL to Temporal Logics with LLMs and Model Checkers (FMCAD 2024)

Lab Publications
  • 问题：自然语言需求难直接进入模型检查。
  • 方法：LLM 转换时序逻辑 + model checker 验证回路。
  • 学习要点：自动规格化必须和验证器耦合，避免“幻觉规格”。

11) Serberus: Protecting Cryptographic Code from Spectres ... (SP 2024)

Lab Publications
  • 问题：密码代码在推测执行下泄漏风险高。
  • 方法：编译期防护 + 硬件前提验证。
  • 学习要点：软件防护需配硬件假设验证才可靠。

12) Model Selection for Latency-Critical Inference Serving (EuroSys 2024)

Lab Publications
  • 问题：延迟敏感推理服务模型选择难兼顾成本与 SLA。
  • 方法：系统层选择策略与性能权衡建模。
  • 学习要点：展示 Trippel 研究在系统优化方向的延展。

13) nl2spec (CAV 2023)

Lab Publications
  • 问题：非结构化需求难直接转时序逻辑。
  • 方法：交互式 NL→TL 流程，用人机回合减少语义漂移。
  • 学习要点：“交互式”通常比一次性生成更稳。

14) G-QED (DAC 2023)

Lab Publications
  • 问题：复杂加速器验证覆盖不足。
  • 方法：Generalized QED 提升预硅验证覆盖。
  • 学习要点：验证方法需要面向复杂系统交互演化。

15) Design for Hardware Memory Model Verification / Mitigating Spectre / TAGS (PLARCH 2023)

CV
  • 问题：架构设计缺乏验证友好与事件序统一框架。
  • 方法：三篇 workshop 从 MCM 验证、Spectre 协同防护、分布式事件序框架补齐方法面。
  • 学习要点：这是“方法论工具箱”条目，适合用来构建你的实践 checklist。

16) RecShard / RecSSD / Failure-Tolerant Training / SLO-NNs（2021-2022）

Lab Publications
  • 问题：推荐系统在存储、训练恢复、SLO 约束下的系统瓶颈。
  • 方法：分别从内存优化、近数据处理、容错训练、SLO-aware 设计给出工程方案。
  • 学习要点：虽非 strict formal 主线，但帮助理解作者系统研究广度。

17) Opening Pandora's Box (ISCA 2021)

Lab Publications
  • 问题：微架构泄漏途径存在系统性未知空间。
  • 方法：系统化归纳和实证，揭示新的泄漏机制。
  • 学习要点：安全研究要重视“机制枚举 + 统一分类”。

18) Dissertation + SoK + BMVC/ArXiv Reverse Engineering Branch（2019-2021）

Dissertation
  • 问题：并发正确性与安全验证如何统一，及跨域安全（视觉特征逆向）问题。
  • 方法：博士论文给出总框架；SoK 与后续工作扩展到 broader security surfaces。
  • 学习要点：把主线方法迁移到新场景是高价值能力。
