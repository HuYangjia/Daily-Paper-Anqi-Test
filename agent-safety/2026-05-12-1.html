摘要（中文翻译，忠实原文）

多 Agent LLM 系统引入了一个新的安全风险：被某个 agent 访问到的敏感信息，会通过共享上下文传播，并在下游输出中重新出现——即使没有显式的对抗意图。我们把这一现象形式化为"传播放大（propagation amplification）"：每跨越一次 agent 边界，敏感内容被反复曝露给下游生成器，泄漏风险随之累积。

已有防御——基于 prompt 的护栏、静态模式匹配、LLM-as-judge 过滤——并不为此场景设计：它们要么在生成之后才介入，要么主要依赖表面字符串模式，要么以巨大延迟为代价、且不对生成过程本身建模。

为此，我们提出 PRISM，一种把"凭证泄漏"视为生成期顺序风险累积的实时防御。在每个 decoding step，PRISM 融合 16 个信号——覆盖词法、结构、信息论、行为与上下文五个维度——计算校准后的风险分，并通过绿/黄/红三档风险区间实施逐 token 干预。

核心观察是：凭证再现往往在生成动力学层面有可测量的前兆——表现为熵塌缩（entropy collapse）与logit 浓度上升。结合 identifier-pattern 等文本结构线索，这些时间序列信号能在 secret 完全重构之前就给出预警。

在覆盖 13 个攻击类别、3 个压力级别、4-agent 异构流水线的 2,000 任务对抗 benchmark 上，PRISM 取得 F1 = 0.832（precision = 1.000，recall = 0.712），任务级泄漏率 0.0%，输出 utility 保持 0.893。相较最强 baseline Span Tagger（F1 = 0.719，泄漏率 15.0%），PRISM 展现实质性提升。

核心内容解读（背景·方法·差异）

解决了什么问题：过去两周的 agent safety 工作主要在两条线推进——SafeHarbor 用分层记忆 guardrail、Chain of Risk 在 LRM 思维链层面控风险、Skills as Verifiable Artifacts 把 skill 当不可信代码做 capability gate。这些都聚焦"单 agent / 单 skill"层面的 safety。PRISM 攻的是另一条线：multi-agent pipeline 中"信息流"的安全——当一个 agent 看到 secret，它会被反复传给下游生成器，即使每个 agent 都没有恶意。这是把 agent safety 从"单调用 guardrail"推进到"信息流（information-flow）safety"的关键一步，对应了 OS 级 capability/IFC 的经典概念在 agent 系统的 LLM-native 实现。

核心方法（生成期信号 + 三档干预）：

  • "传播放大"形式化：把"经过 N 个 agent 后的泄漏概率"建成一条递增的累积曲线，每个 agent 边界都是一个累积点。这是一个非常简洁却新颖的现象命名——它使得"凭证泄漏 ≠ 一次性事件"变成可量化的风险累积过程。
  • 16 维信号融合：覆盖五大类——(1) 词法（identifier 模式、长度异常）、(2) 结构（key-value 排版、URL 嵌入）、(3) 信息论（熵、logit 浓度）、(4) 行为（生成节奏变化）、(5) 上下文（前文出现 secret 命名空间）。每步 decoding 给出一个校准的风险分。
  • 关键时间序列洞察：论文指出 LLM 在即将复现 secret 时会出现可测量的熵塌缩 + logit 集中——这是模型"高置信回忆某段固定内容"的指纹。把它和 identifier 模式拼起来，就能在 secret 完全重构之前预警。
  • 红黄绿三档干预：绿区放行；黄区软干预（重写、占位符）；红区硬阻断（抑制 token / 回退）。这是一种逐 token 的、以风险分驱动的动态约束解码——不是 post-generation 的 filter，而是 in-the-loop 的 controller。
  • 4-agent 异构流水线 benchmark：2,000 任务 × 13 攻击类别 × 3 压力级别。这个规模足以做出 F1 / 泄漏率 / utility 三轴 trade-off 的 ROC，是当前 multi-agent 安全 benchmark 中最系统化的一个。

核心数据亮点 / 关键论点
  • F1 = 0.832, precision = 1.000, recall = 0.712——precision 1.0 在生产级 secret 防泄漏里是关键指标（误杀 0），与 utility 0.893 的高保留率形成强 trade-off。
  • 任务级泄漏率 0.0%（vs Span Tagger 15.0%）——在 2,000 任务对抗 benchmark 上零泄漏，意味着 PRISM 的"预测性"信号至少在 benchmark 范围内做到了完整 coverage。
  • 16 维信号、5 大类：lexical / structural / information-theoretic / behavioral / contextual——是当前在 in-loop secret detection 中维度最齐全的特征工程。
  • 关键洞察：熵塌缩 + logit 集中——LLM 在"高置信回忆 secret"时生成动力学有可测量前兆，这是信息论 + 行为学的桥接观察，未来可推广到其他 high-confidence "记忆复现" 风险（PII / 训练数据复现）。
  • 13 攻击类别 × 3 压力级别 × 4 agent 异构流水线——把"传播放大"做成可重放压力实验，是一份高质量的多 agent 信息流 safety benchmark。
  • 方法学意义：把 information-flow control（IFC）经典思想以 LLM-native 方式落地到 decoding 阶段，相比 prompt-level / post-hoc filter 第一次把"safety-by-design"做到 token 颗粒度。
对你三个研究方向的启发
Harness Engineering

PRISM 的 16 维生成期信号给 harness 引入一个"per-token diagnostic surface"：你正在做的 ZipperQuant W4A4 e2e benchmark 完全可以借鉴它的熵塌缩 + logit 浓度作为quantization regression detector——当 W4A4 模型在某个 token 上熵分布与 BF16 显著偏离（比如塌缩到错误 token），可以即时报警，比单纯比 perplexity 颗粒度细很多。这给 ZipperEngine 提供了一个"online quality monitor"的设计模板：在 decode 阶段以极低开销做 logit 分布对比，比 offline benchmark 更接近 production 信号。

Agent Skills Safety

PRISM 把 safety unit 推进到 per-token，是和今日 Harness 方向 ComplexMCP 形成天然组合的关键拼图：ComplexMCP 揭示 over-confidence 是失败 root cause，PRISM 给 over-confidence 提供了 generation-time 的可观测信号（entropy collapse）。把两者拼起来，可以构建一个完整的 "stress harness + per-token guardrail" pipeline——在 ComplexMCP 的 seed-driven 故障注入下，让 PRISM 风险分作为 capability gate 触发器，自动在 over-confidence 高发段切换到"verify-then-act"模式。这是把 information-flow safety 从凭证泄漏延伸到 agent 行为泄漏的自然路径。

Safety Benchmark

2,000 任务 × 13 攻击 × 3 压力 × 4 agent 流水线给 multi-agent safety benchmark 提供了一个直接可学的形态："agent 拓扑 × 攻击类型 × 压力级别"三维 grid。今日 Benchmark 方向 Agent-ValueBench 已经在做"价值在 4 主流 harness 中的潮汐"——把 PRISM 的 4-agent pipeline 作为额外 harness 配置嵌入，可以测出"价值一致性 vs 信息流泄漏"两条 safety 维度是否独立，这是更接近真实 production 部署的 multi-dimension safety 评估。Precision = 1.0 同时 recall < 1.0 的 trade-off 也提示一个 benchmark 报告原则：把 false positive 与 false negative 分开报，而不是混在一个 F1 数。
