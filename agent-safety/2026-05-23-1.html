用对抗机器学习的几何语言把"消除拒绝方向"重写为针对线性探针的潜空间逃避攻击——既解释了现有 ablation 为何能跑通，也解释了它为何停在边界，进而做出"越过边界推入合规区"的强化版攻击，在 15 模型上拿下 SOTA ASR。
摘要（中文翻译，忠实原文）

安全对齐过的语言模型被训练用来拒绝有害请求，但拒绝行为可以通过操纵其内部表示来抑制。已有方法的做法是从模型激活中"消除拒绝方向"——目标是把"应该拒绝"的成分从 residual stream 中移除。这些方法在经验上有效，但缺乏对所诱导的潜空间变换的原理性解释，也没说清楚"为什么这样就能抑制拒绝"。

本文把"拒绝抑制"重新定义为针对一个线性探针的潜空间逃避攻击——这个线性探针被训练用来区分"应该拒绝的 prompt"和"应该回答的 prompt"。在这一视角下，先前工作所谓的"均值差异方向（mean difference direction）"自然地恰好定义了这样一个探针，而对它的"消除"正是对决策边界的最小置信度投影——也就是经典对抗机器学习里的"最小置信度逃避攻击"。

这个视角不仅解释了既有方法的经验成功，也揭示了一个关键限制：逃避停在决策边界处，而真正"让模型回答"需要把表示进一步推到 合规区。本文据此提出受控潜空间逃避攻击（Controlled Latent-space Evasion），以可控的置信度把表示越过边界推入合规域。在 15 个模型（指令调优、多模态、推理）上达成 SOTA ASR，超越了 refusal-direction ablation baseline 与专门 jailbreak baseline。

核心内容解读（背景·方法·差异）

解决了什么问题（why now）：近 12 个月的 LLM 安全圈最大公约数式认知是 "refusal direction 可以被 ablate 掉"——但所有这条线的工作（refusal-direction ablation、Diff-of-means、orthogonalization、abliteration）都没在数学上解释为什么。本文把这件事拉回到 2014 年以来的对抗机器学习经典框架：所谓 "consistent representation that supports refusal" 就是一个线性可分超平面，"消除拒绝方向" 就是对这条超平面做最小投影——这是 evasion attack 的最基本几何形态。这条主线跟 5-13 SKILL.md 攻击（86%/77.6%/100% 绕过 ClawHub skill 治理层）、5-19 Memory Laundering（sub-threshold 在 summary 之后逃过监测）、5-20 ECA（authorization-failure 视角的几何化）形成对照——前几篇是系统级攻击，本文是表征级几何攻击；二者打的是同一面墙的不同位置。

核心方法：从"消除"到"越界投影"

  • 步骤 1：把拒绝抑制等价为一个 evasion attack。设线性探针 p(x) = w·h(x) + b，训练它区分 refusal vs answer prompt 的隐藏表示 h。"消除拒绝方向" 就是把 h 投影到 p(h)=0 这条决策边界——即"最小置信度逃避攻击"。
  • 步骤 2：揭示 mean difference direction 的几何身份。mean-of-answer minus mean-of-refusal 这个常用方向 正好就是 Fisher 判别意义下这条探针的法向量——这给一直缺乏理论支撑的"diff-of-means abliteration" 一个干净的几何身份证。
  • 步骤 3：识别 ablation 的内禀限制。ablation 是把 h 投到 边界，而不是 越过边界进入合规域——这正解释了为什么有时 ablation"绕过了拒绝、但模型还是嘟囔"——表示停在边界附近，模型既不拒绝也不流畅作答。
  • 步骤 4：Controlled Latent-space Evasion。在 ablation 之外加一个可控置信度幅度 ε，把 h 沿同方向继续推到 p(h)=+ε——这就是合规域的内部点，模型从 "不拒绝" 升级为 "主动顺畅作答"。这是经典 adversarial ML 里"high-confidence adversarial example"在 LLM 内部表征空间的对应物。

Agent Skills Safety（直击）

对 agent safety 是最直接的弹药：5-13 SKILL.md / 5-19 Memory Laundering / 5-20 ECA 都是系统级攻击——SKILL 注册表绕过、context summary 漂洗、authorization 错位；本文给了一个表征级解释——所有这些攻击之所以能成功，根本上是因为"refusal vs answer" 在隐空间是线性可分的，从而 evasion 必然存在。这意味着：纯系统层防御 + 模型表征层防御必须并举——光靠 CUGA 的 5 检查点 (5-21) 或 SKILL.md 的 governance 层挡不住表征级攻击；反之，纯表征防御也挡不住系统级攻击。这是把"agent safety"从单一栈往多层栈纵深防御推进的明确路标。

Safety Benchmark

对 benchmark 是非常清晰的方法论指引：所有 safety benchmark 都应该报告 "在线性探针下 refusal 是否线性可分" 这个 meta-metric——这是 jailbreak / abliteration / evasion 一切攻击的可行性下界。具体可借鉴：(1) 在 5-21 GrandGuard、5-20 POLAR-Bench 一类细粒度 benchmark 里加 "Linear Probe Separability" 列；(2) 把"受控 ε 越界"做成一个新的对抗鲁棒性 benchmark 维度——量化"模型在多大置信度逃避下还能撑住"；(3) 把 ablation baseline 与本文的 controlled evasion 一并作为 jailbreak benchmark 的下限基线。
