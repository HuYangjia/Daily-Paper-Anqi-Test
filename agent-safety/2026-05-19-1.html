摘要（中文翻译，忠实原文）

LLM agent 越来越依赖持久状态——transcript、summary、检索上下文、记忆 buffer——来支撑长程交互。这意味着安全不再只取决于单次模型输出，还取决于 agent 存了什么、之后又怎么复用。我们研究一种被称为 memory laundering（记忆洗钱）的失败模式：有毒或对抗性上下文被压缩成看上去无毒的 memory summary——标准检测器再也看不出它有毒——但其中保留的敌意框架或冲突结构仍会持续影响未来的生成。

使用配对反事实多 agent rollout，我们证明：来自有毒源头的 memory summary 即便在常见毒性阈值之下，也会相对于匹配的中性基线显著抬升下游毒性。为度量这一隐藏影响，我们提出 Sub-threshold Propagation Gap (SPG) 指标——在被部署 monitor 判为"安全"的 memory state 条件下，下游行为差异有多大。

实验表明：毒性沿不同的状态通道传播——raw transcript reuse 驱动显性下游毒性，compressed memory 携带隐藏的 sub-threshold 影响。我们进一步发现：缓解效果高度依赖干预位点——在 summarization 之前清洗有毒状态能大幅降低传播 gap，而只清洗已完成的 summary会让"洗白后的影响"原样保留。

这些结果表明：记忆增强 agent 的安全应该被当作"演化中的上下文上的状态控制问题"来处理——sanitization 必须发生在不安全信息被压缩成持久记忆之前。

核心内容解读（背景·方法·差异）

解决了什么问题（why now）：过去半个月 Agent Safety 主线密集打多个攻击面——5-12 凭证泄漏的 PRISM、5-13 SKILL.md 供应链攻击、5-14 多 agent 多模态 HAM³ 攻击、5-15 tool-use 知-行 gap、5-18 Ensemble Monitoring。它们的共同前提都是 monitor 看得见输出——只要监测器对当前 token / 当前 prompt 做出了不毒判断，系统就被认为安全。本文打掉的恰恰是这个前提：当上下文被压缩进 memory summary，毒性会被 LLM 的 abstraction 抹平到检测阈值之下，但"敌意框架 / 冲突结构"这种深层 affordance 没有消失——它继续以 sub-threshold 的形式渗到下游决策。这是把"safety vs. monitor"的对抗，从 token level 转移到了 state level / abstraction level。

核心方法：paired counterfactual + SPG

  • 双通道实验：把 agent 的状态分两路——（a）raw transcript reuse（直接复用对话），（b）compressed memory（先 summary 再复用）。配对反事实地比较"有毒来源 vs. 匹配的中性基线"在两通道下的下游毒性。
  • Sub-threshold Propagation Gap (SPG)：核心新指标。条件是"被部署 monitor 判为安全的 memory state"，测量是"在这条件下，下游行为相对中性基线的偏移量"。SPG > 0 = monitor 看不见但毒性已经在传。这个指标第一次把"洗白后的隐性影响力"做成可量化的标量。
  • 干预位点消融：作者比较"先清洗有毒状态再 summarize"vs."只清洗 summary"两种缓解。前者大幅压低 SPG，后者几乎没用。这一对照直接给出工程指南：sanitization 必须前置到 summarization 之前——监督下游 summary 是没有用的，因为伤害已经被压成结构性。
  • 多 agent 设置：实验用配对的 multi-agent rollout，避免单 agent 设置下的偶然性，确保 SPG 不是 noise。

核心数据亮点 / 关键论点
  • memory laundering 现象首次实证：toxic-origin summary 即便在常用毒性阈值下被判"安全"，仍能在 paired counterfactual 中显著拉高下游毒性——这把"safety bypass via abstraction"从理论假说变为可测现象。
  • SPG 指标：定义清晰、条件明确（基于 deployed monitor 的 "safe" 判决）、可作为 monitor 评估的新档——任何 memory-aware agent 都应当报告 SPG，而不只是输出毒性。
  • 双通道差异：transcript reuse 驱动 overt toxicity，compressed memory 携带 hidden sub-threshold influence——明确告诉工程方"压缩才是真正的安全风险"，而不是 reuse 本身。
  • 位点决定缓解效果：清洗 summary 几乎无效，必须在 summarization 之前清洗 state——这是最强的工程指南，可立刻反映在 LangChain / MemGPT / OpenAI Memory 的 pipeline 设计上。
  • 把 safety 重定义为 state-control problem：作者明确说"safety should be treated as a state-control problem over evolving context"——这句话量级和 Anthropic 的"AI control"框架一样大，标志着 memory-aware safety 进入新范式。
  • 方法学外推：SPG 的"条件式 gap"思想可以推广到任何"被压缩/被抽象后的影响力测量"——例如 RAG 中召回的有毒文档、code agent 中长上下文里的污染样本、tool-use agent 中长记忆里的策略漂移。
对你三个研究方向的启发
Agent Skills Safety（直击主线）

这是把"持久状态"立为 agent skill 安全的一等关注面的关键论文。结合 5-13 SKILL.md 供应链攻击与 5-18 Ensemble Monitoring，可以画出一张完整威胁地图：（输入侧）skill 注册攻击 → （执行侧）token-level monitor → （状态侧）memory laundering。三道防线缺一不可。建议在你的"agent skill 安全综述"里把 SPG 作为"状态侧"的标志性指标——因为它第一次让"看不见的影响"可量化。落地建议：任何 skill 在被注册到 agent 长记忆前，都要做pre-summarization sanitization；任何 evaluation harness 也都要附带 SPG 评测。

Harness Engineering

SPG 给了 harness 一种新维度——"被压缩动作前后的下游影响差"。这条思路可以直接搬到你的 KV-cache offload runtime：当某些 layer 的 cache 被驱逐/压缩到 CPU，它对未来 layer 的"行为偏移"是否仍然存在？换言之，"cache eviction 后的下游 logits 偏移"就是 KV 版的 SPG。把它作为 ablation 的一个新坐标轴，可以让你在 LRU/Random/FIFO/hu_fixed 几种策略之间区分"看似 hit-rate 一样但语义损失不同"的微妙差异。配合今日 harness 选定 ANNEAL 的 typed edit + canary，整个 offload runtime 就能做到：先用 SPG 测语义损失、再用 ANNEAL 把损失大的 layer 写回成 typed policy patch。

Safety Benchmark

对 benchmark 的启发：当前所有 agent benchmark 都没有持久状态污染轴——它们假设每个 task 都是 fresh start。SPG 给出一个新的 benchmark dimension：在 N 轮交互中注入有毒源、让 agent 自然地 summary，然后测下游若干轮的行为偏移。把这一轴接到今日 benchmark 选定的 TOBench / 之前的 SaaS-Bench / ClawForge 上，可以一次性暴露"长记忆型 agent 的状态污染敏感度"。值得作为 next-gen agent safety benchmark 的强 baseline。
