摘要（中文翻译，忠实原文）

由 LLM 驱动的工具调用 Agent 系统正在被广泛部署到 Web、App、操作系统、交易等场景中。然而，现有安全 benchmark 仍然侧重显性危险，可能夸大模型对"欺骗性 / 模糊性"轨迹的判断能力。为弥合该 gap，我们提出 ROME（Red-team Orchestrated Multi-agent Evolution）——一个受控的 benchmark 构造管线，它把已知不安全轨迹改写成更具欺骗性的评估实例，同时保留底层的风险标签。从 100 条不安全源轨迹出发，ROME 产生 300 条挑战实例，覆盖三类典型的欺骗模式：上下文模糊、隐性风险、捷径决策。

实验显示，这些挑战集会显著拉低安全判断准确率，且"隐性风险"案例对前沿模型仍非平凡。我们另外提出 ARISE（Analogical Reasoning for Inference-time Safety Enhancement）——一种"检索引导的推理时增强"方法：从外部类比库中取出 ReAct 风格的安全轨迹作为结构化推理范例注入，无需重训就能改善安全判断；但作者特别指出，ARISE 应被理解为针对特定任务的"鲁棒性增强"，而不是独立的安全保证。ROME 与 ARISE 共同构成在欺骗性分布漂移下对 Agent 安全判断力进行压力测试和提升的实用工具。

核心内容解读（背景·方法·差异）

解决了什么问题：过去一年的 Agent Safety 主线在不停"加防护"——Symbolic Guardrails 加策略层、Owner-Harm 把"伤主"列为缺口、Human-Guided Harm Recovery 把回滚做成 post-execution，再到昨日 BOA 把"采样估计"换成"likelihood-budget 搜索"。但所有这些工作都依赖"显性危险"——挂载策略、定义违规分类、训练判别器。本文提出一个尖锐问题：当危险被刻意包装得"看起来正常"时，模型还能不能识别？这正是过去 benchmark 集体忽视的盲区。

方法 1 — ROME 改写管线（数据侧）：

步骤做什么关键约束
① Source 选取从既有 benchmark 抽取 100 条已标注的不安全轨迹保留原始风险标签 (label-preserving)
② Multi-agent 改写多个 LLM agent 协作把同一条轨迹"演化"成 3 个变体底层风险不变，但表面更欺骗
③ 三类欺骗模板上下文模糊 / 隐性风险 / 捷径决策分别对应 OOD 三种形式
④ 验证回路红队 LLM 复检"风险还在 + 表面足够欺骗"过滤无效改写，保留 300 条

方法 2 — ARISE 类比推理（推理侧）：

  • 外部类比库：预先维护一组 ReAct 风格的"安全推理范本"（含原始 query、危险点剖析、安全行动）。
  • 检索-注入：测试时根据当前 trajectory 检索相似类比，作为 in-context exemplar 注入到 Agent 的判断 prompt 里。
  • 定位：明确不替代训练时的安全对齐，只在推理时把"我以前看过类似的危险"这种结构化先验补回来。

与现有工作的核心差异：

  • 对比 HarmBench / SafetyBench / TrustLLM：那些是"静态题库 + 显式 unsafe 标签"；ROME 第一次把"label 不变、表面变深"作为评估维度，揭示模型在 OOD 下的脆弱性。
  • 对比 Symbolic Guardrails：Guardrails 在策略层拦截显性违规；ROME 暴露出"策略层根本没看到危险"的样本类型——这是上层防护的盲区诊断。
  • 对比 BOA（昨日）：BOA 用 likelihood-budget 搜索已知不安全分布的尾部；ROME/ARISE 处理的是 OOD 不安全分布——即从未见过的"伪装危险"。两者一个深耕已知，一个跨越未知。

核心数据亮点 / 关键论点
  • 100 → 300 受控放大：label-preserving 改写让 benchmark 容量 ×3，且不需要新标注成本——这是数据工程上的实用突破。
  • "判断准确率显著下降"：在 ROME 改写后的挑战集上，前沿模型的安全判断准确率出现可观下滑（具体百分比详见原文 Table）；这是迄今为止最直接的"欺骗性 OOD 失效"证据。
  • "隐性风险"是终极硬骨头：论文明确指出 hidden-risk 子集对前沿模型仍是非平凡问题——这正是下一轮安全研究的明确靶子。
  • ARISE 是"无重训"补丁：仅靠类比库 + 推理时检索，就能在不动模型权重的前提下提升判断质量，部署友好。

Harness Engineering

ROME 的"label-preserving 受控改写"本质上是一种 harness 自演化的 benchmark 端实现——可与 AHE 的观测驱动 harness 演化、Last Harness 的 Meta-Evolution 自然耦合。建议把 ROME 的多 agent 改写循环抽象成 harness 的"题库自动扩增"模块，使评测题库随被测模型一起进化，避免数据污染。

Agent Skills Safety

本文与 BOA 在维度上互补：BOA 解"已知分布的尾部估计"，ROME/ARISE 解"未知分布的判别能力"。将两者拼起来，即"已知尾部 × 未知 OOD" 的双轴 Agent 安全度量，这可能成为 2026 年 Agent Safety 测度框架的新模板。后续工作建议把 ARISE 类比库做成"安全模式的 episodic memory"，与 Human-Guided Harm Recovery 的 post-execution 反思形成闭环。

Safety Benchmark

对 Safety Benchmark 设计者最直接的提醒：静态题库正在以指数级速度被前沿模型 saturate。ROME 给出的"受控改写流水线"可以作为"动态扩增"模块嵌入既有 benchmark（如 HarmBench、SafetyBench、CSTM-Bench），使得 benchmark 不再是一锤子买卖。结合今日 Benchmark 报告的 Dreadnode 红队，可以形成"自动改写 + 自动红队"的双引擎评估管线。
