一句话核心贡献
为多轮 coding agent 提供白盒、可解释、单层 10 维子空间 的运行时安全干预——基于步级隐藏表征探针检测 + 子空间方向 broadcast 干预，让 ASR 平均从 85.99% → 13.36%（−72.63 pp），并首次提供提前一步 lookahead 风险预判（平均 94.54% 准确率），证明 multi-turn agent 的"欺骗 / 攻击 / 服从—拒绝" 取向由少量低维方向 中介，把 mechanistic interpretability 从 single-turn QA 推到 multi-turn agent 执行轨迹。
摘要（中文翻译，忠实原文）

基于 LLM 的 coding agent 展现出显著的自主能力，但也在与外部环境的多轮交互中引入了重大的安全与误用风险。现有安全机制主要依赖外部 guardrail，对执行过程中的细粒度行为控制能力有限。同时，近期关于 LLM 安全的mechanistic interpretability 工作大多局限于single-turn 或 jailbreak 风格的问答场景，限制了它们捕捉 multi-turn agent 执行中不断演化的风险动态的能力。本文从内部视角 研究多轮 coding agent 的安全。我们提出 AgentLens（Mechanistic Subspace Intervention and Steering），一个白盒防御框架，为 coding agent 实现运行时安全检测和表征级缓解。与传统 agent guardrail 不同，AgentLens 从步级 隐藏表征检测有害执行状态，并通过在单层内的 10 维子空间 上干预，缓解不安全行为。为支持此研究，我们引入了 Mechanistic Agent Safety（MAS）benchmark，包含 194 个任务上跨 LLaMA-3.1-8B / Qwen-2.5-7B / Gemma-2-9B 的完整注释多轮执行轨迹。大量实验证明 AgentLens 实现了强劲的安全检测性能，提供了 lookahead 风险预判的初步证据，并大幅降低 coding agent 的有害动作，为应用 mechanistic interpretability 到动态 LLM agent 安全奠定基础。代码：github.com/EddyLuo1232/AgentLens。

核心内容解读（白盒姿态 / 单层 10 维子空间 / MAS benchmark / lookahead 预判 / 因果验证）

立论：multi-turn agent 安全需要白盒+表征级方法。2024–2026 H1 主流 agent safety 路径有两条：(1) 外部 guardrail（ShieldAgent、policy reasoning 等）——基于输出文本拦截，缺乏对执行中间状态的细粒度感知；(2) 单轮 / jailbreak 的 mechanistic interpretability（Arditi et al. 2024 "refusal 单方向中介"，Lieberum et al. 2024 Gemma Scope）——只覆盖 QA 场景，捕捉不到 agent 多轮执行中的风险动态。AgentLens 团队的关键观察是：multi-turn coding agent 的"欺骗 / 攻击 / 服从—拒绝" 取向并不是分散在整个网络，而是由少量低维方向中介——这把 Arditi et al. 2024 单轮 refusal 单方向的姿态推到 multi-turn 全执行轨迹。

方法一：步级线性探针 + Top-K 子空间识别。对每一步 t、每一层 ℓ，提取最后一个 token 的隐藏表征 z_t^(ℓ) = H_t^(ℓ)[−1]。训练二分类线性探针 g_φ^(ℓ)，参数为权重矩阵 W ∈ ℝ^(2×d)。每个隐藏维度 k 的类别判别分数 定义为 |W_{1,k} − W_{0,k}|（1=有害 / 0=良性）。选 Top-K 维度（K=10）构成集合 𝒦，构造方向向量 v ∈ ℝ^d：v_k=1 if k∈𝒦, else 0。逐层探针：LLaMA-3.1-8B 的最强层为第 12 层（准确率 99.15%），Qwen-2.5-7B 为第 13/14/15/17 层（97.46%），Gemma-2-9B 为第 14 层（95.35%）——三模型一致地"早期-中期层"达到最强性能。

方法二：子空间方向 broadcast 干预。检测阶段：若 ŷ_t = g_φ^(ℓ)(z_t^(ℓ)) = 1（识别为有害），触发缓解。缓解阶段：对该层所有 token 位置广播方向向量——H̃_t^(ℓ)(α) = H_t^(ℓ) + α · 1_{n_t} · v^T。α 是干预强度（连续可调）。α 的选择：LLM 评判器对候选动作打分，安全分 s(a_t(α)) ∈ [0,10] + 效用分 u(a_t(α)) ∈ [0,10]，最优 α* = argmax [λ_s · s + λ_u · u]，其中 λ_s=0.6 / λ_u=0.4（论文锁定值）。关键工程优势：只在单层 上的10 维子空间 做线性干预——白盒、可解释、可移植、几乎零推理开销。

Benchmark：MAS（Mechanistic Agent Safety）。194 任务（恶意 98 + 良性 96），6:4 训测划分（任务级，确保测试任务不入训）。风险类别：来源 AdvCUA / Safe-OS / AgentBench，覆盖 10 类 MITRE ATT&CK 框架下的 OS 级攻击技术。标注流程：(1) 严格标准——恶意任务中直接实例化攻击的步骤标 1，安全意识/拒绝的步骤标 0；良性任务全部 0；(2) 混合管道——启发式关键词匹配 + Claude Opus 4.6 / Gemini 3.1 Pro 初步标注 + 人工纠正；(3) 反事实增强——构造良性对标任务模仿恶意行为的执行模式与语法结构。

主要结果一：安全检测性能。当前步（current-step）：LLaMA 99.15% acc / 99.17% F1；Qwen 97.46% / 96.25%；Gemma 95.35% / 86.67%。相对最强基线分别 +2.99 / +2.54 / +5.82 pp。Lookahead（用有害步前一步 的隐藏表征预测下一步标签）：LLaMA 92.31%，Qwen 96.77%，平均 94.54%——证明中间层表征已经包含"下一步将要做坏事" 的预测信号。论文谨慎将其解释为可行性证据而非鲁棒性声明（自然 lookahead 样本稀疏，因为开源 coding agent 长程规划仍有限）。

主要结果二：风险缓解（ASR 大幅下降）。原始 ASR：LLaMA 73.80%，Qwen 91.30%，Gemma 92.86%，平均 85.99%。AgentLens 缓解后：LLaMA 16.70%，Qwen 4.35%，Gemma 14.29%，平均 13.36%。平均降低 72.63 pp。生成崩溃率（即 agent 失去生成能力）：LLaMA 28.30%，Qwen 13.04%，Gemma 14.29%，平均 18.54%——远低于 RepE / Self Reminder 等基线。横向对比：RepE 在 Qwen / Gemma 上 ASR 仍达 73.91% / 71.43%；Self Reminder 64.29–82.61%；AgentLens 全部压到 16.70% 以下。

主要结果三：负向干预的因果验证 + 分布外测试。把方向向量取反（−α），检验是否能诱发 有害行为：LLaMA 的 16 个拒绝步骤上，反向干预 ASR=100.00%，崩溃率仅 6.25%；Gemma 27 个拒绝步骤上反向 ASR=63.00%，崩溃率 0.00%——直接证明该 10 维子空间因果 中介 agent 的拒绝/服从取向。分布外：恶意 prompt injection 攻击下，原始 ASR=86.7%，AgentLens 干预后 6.7%（崩溃率 0%），证明子空间方向对未见过的攻击模式仍然有效。

方法学姿态：refusal 单方向 → multi-turn 多步 10 维子空间。Arditi et al. 2024 在 single-turn QA 上证明"refusal 由单一方向中介"；AgentLens 把这条姿态推到 multi-turn coding agent——发现 (a) 仍是低维（K=10）；(b) 仍是单层（早中期）；(c) 信号还能提前一步 反映未来风险。这与 6-17 Rift"残差秩冲突签名"（用残差秩比检测欺骗）、6-18 SRP（预训练嵌入安全反思）、6-09 ELK（欺骗不可能定理）形成"内部表征—梯度记忆—训练阶段—理论边界" 的 2026 H1 agent safety 内部视角四件套。

对你三个研究方向的启发
Harness Engineering

AgentLens 与今日 #1 AOHP 形成"表征—系统" 双层防御——AOHP 提供 OS 级信息流隔离作为河道，AgentLens 提供模型内单层 10 维子空间作为河中闸门。对 harness 工程而言，这意味着 2026 H2 的multi-layer defense harness 应同时包含 (a) OS information flow control；(b) representation-level steering；(c) external guardrail；(d) action-level rollback。从 KV-cache 系统层毕设角度，AgentLens 的step-level hidden representation probing 是一种低开销在线度量 思想——你可以借鉴此姿态为 cache 系统设计"step-level cache health probing"：每一推理步用线性探针从中间表征（如 attention map / KV residual）读出 cache miss 的低维子空间签名，把 cache 决策从规则驱动 升到表征驱动。K=10 维 / 单层 / 几乎零开销 的工程范式特别适合系统层移植。

Agent Skills Safety（直接对应）

AgentLens 给 2026 H1 agent safety "内部表征四件套" 提供了最完整的白盒+多轮+可干预 范式——与 6-17 Rift（残差秩冲突签名诊断欺骗）、6-18 SRP（预训练阶段嵌入安全反思）、6-09 ELK（理论上的欺骗不可能定理）共构 2026 上半年最重要的方法谱系。直接行动建议：(1) 子空间转向的可解释性—— Top-K 维度的 SAE 解释（参考 SAGE @ EACL 2026 Industry）可帮助理解到底是哪些语义概念（如"权限提升" / "用户欺骗" / "数据外泄"）落在这 10 维子空间里；(2) 跨 skill 子空间复用—— skill 与 skill 间是否共享子空间，决定是"一刀切" 干预还是 "per-skill" 干预；(3) lookahead 风险预判 在 skill 调用前一步就能预测，意味着可以构造"preemptive abort" 协议——agent 调用 skill 前自检，命中 lookahead 高风险信号即拒绝；(4) 反向干预导致 100% ASR 是双刃剑—— 模型权重一旦泄露，攻击者可构造跨平台越狱，这反向论证了模型权重保密 + 表征级访问控制的必要性。

Safety Benchmark

AgentLens 的 MAS benchmark 是方向 #3 评测面下沉的最新案例——把评测维度 从 final-step outcome（HarmBench / AgentDojo）下沉到step-level annotation（每条多轮轨迹的每一步 都打标签）。这与 6-18 SciRisk-Bench"10 风险维度 × 学科 × 子学科"、6-17 SEAGym"5 视图评测面"共同构成 2026 H1 evaluation depth 的三次下沉。统一启发：未来 safety benchmark 应同时具备 (a) step-level annotation（MAS）；(b) 风险维度交叉（SciRisk-Bench）；(c) 5 视图评测面（SEAGym）；(d) 反事实对照（MAS 良性对标任务模仿恶意语法结构）。AgentLens 的反事实增强 思想尤其关键——构造与恶意任务语法结构相同但意图良性的任务，可以直接证明探针不是在拟合表层文本，而是在拟合意图差异。这给 benchmark 设计提供了"语法 vs 意图 解耦"的新质量标准。与今日 #3 "Benchmark Illusion"（4 大轴弱耦合 / 单一意识评分不可靠）形成互补：MAS 通过 step-level + 反事实让 benchmark 的单一指标可靠性 提升一个层级。
