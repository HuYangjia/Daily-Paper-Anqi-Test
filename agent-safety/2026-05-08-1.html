一句话核心贡献
用分层记忆把"该不该拒"的细粒度规则在线生长，把 over-refusal 与 under-defense 第一次同步压住。
摘要（中文翻译，忠实原文）

随着基础模型快速演进，LLM Agent 的工具使用能力越来越强；与此同时，攻击者也能更轻易地操纵 Agent 调用工具来产生有害输出。现有防御机制虽然有效，却普遍受困于"过度拒绝"（over-refusal）问题——安全严格度提高的同时，benign 任务上的可用性被牺牲。

本文提出 SafeHarbor：一个为 LLM Agent 建立精确决策边界的新框架。不同于静态指南，SafeHarbor 通过增强对抗生成来抽取上下文感知的防御规则；并设计了一个局部分层记忆系统用于动态规则注入，从而提供 training-free、高效、即插即用的防御方案。此外，作者还引入一种基于信息熵的自演化机制，通过节点的动态分裂与合并不断优化记忆结构。

大规模实验显示：SafeHarbor 在两端——模糊的 benign 任务与显式的恶意攻击——均达到 SOTA：在 GPT-4o 上 benign 效用峰值达到 63.6%，同时对有害请求的拒绝率超过 93%。源码已在 github.com/ljj-cyber/SafeHarbor 公开。

核心内容解读（背景·方法·差异）

解决了什么问题：过去半年的 Agent guardrail 主线（参见本仓库归档：Symbolic Guardrails、BOA、ROME × ARISE、AgentTrust）正在收敛到一个共同问题：静态规则（policy / signature / 形式化守卫）覆盖率不够 → 实际部署只能调严，调严就 over-refuse。SafeHarbor 给出的解法不再是"再写一份更完备的规则"，而是让规则在线生长：每碰到一类新攻击，就让 guardrail 产出一条上下文感知的细粒度规则，存进分层记忆，下次类似上下文再被命中。

核心方法（三件套）：

  • ① 增强对抗生成抽规则：不依赖人工写策略，而是用对抗 prompt 生成器去试探 Agent 的边界，把"什么样的请求该被拦"具体化为上下文感知的规则文本（context-aware defense rules）。
  • ② 分层记忆动态注入：把规则按上下文相似度组织成局部层级化记忆，运行时根据当前请求只把相关的小子集 inject 到 guardrail prompt 里——避开了"规则越多 prompt 越长越容易 over-refuse"的死循环。
  • ③ 信息熵自演化：动态对记忆节点做分裂（信息密度过高）/合并（冗余），让记忆树随时间增长仍保持均匀的信息熵——这是与传统 RAG 检索 cache 的关键差别（RAG 不会主动合并冗余条目）。
  • ④ Plug-and-play、training-free：不需要 fine-tune 被保护的 Agent，只需要包一层 guardrail。

从本仓库 Agent Safety 这条主线来看，SafeHarbor 是第一个把"规则生长"与"规则裁剪"做成一个闭环的工作——以前的工作要么只让规则增长（容易 over-refuse），要么只让规则裁剪（容易 under-defend）。而 ICML 2026 的录用与同期 GPT-4o 同时拿到 63.6% utility / > 93% refusal 的实测数字，意味着这套范式已经达到可工程部署的量级。

核心数据亮点 / 关键论点
  • GPT-4o benign utility = 63.6%：在被保护后仍保留高可用性，相比硬 guardrail 普遍出现的 utility < 50% 显著领先。
  • refusal rate > 93%：对有害请求保持 SOTA 拦截率，与 benign utility 同步达成（不靠"调严"妥协）。
  • training-free：不需要 fine-tune 被保护模型，可直接套在 GPT-4o / Claude / 开源模型上。
  • 分层记忆动态规则注入：避免了静态 prompt 越长越钝化的常见失败模式。
  • 信息熵 split / merge：让记忆树随时间增长仍保持均匀信息密度，是与 RAG cache 的核心区分。
  • 已 ICML 2026 录用：顶会同行评议背书，方法论稳健性与可复现性双重确认。

Harness Engineering

SafeHarbor 与今日 Harness 报告 Token Arena 联读后会发现一个工程问题：guardrail 的 over-refusal 测量本身依赖 endpoint，而 endpoint 漂移高达 12 pp——SafeHarbor 报告的 "GPT-4o 63.6% utility" 在不同 SKU 下可能上下浮动 5-10 pp。建议把 SafeHarbor 的评测扩展为 endpoint 矩阵，作为 guardrail harness 工程化的下一个里程碑。

Agent Skills Safety

SafeHarbor 直接补齐了本仓库 Agent Safety 主线的最后一环。Symbolic Guardrails 给出"形式化"的静态守卫，AgentTrust 给出"runtime"的 4 档启发式裁决，SafeHarbor 给出"分层记忆 + 信息熵"的动态生长规则。三者刚好可以拼成一个三层防御：静态 → 动态生长 → 实时裁决，作为你 Agent Safety 论文的标准 baseline 集合。

Safety Benchmark

SafeHarbor 暴露的 over-refusal/under-defense 双轴指标，应该被纳入今日 Benchmark 报告 XL-SafetyBench 的评测协议——XL-SafetyBench 已经引入 NSR (Neutral-Safe Rate) 区分"原则性拒绝"与"理解失败"，与 SafeHarbor 的 over-refusal 概念高度对齐。建议在你的 safety benchmark 设计中把 (utility, refusal-rate, NSR) 设为强制三轴。
