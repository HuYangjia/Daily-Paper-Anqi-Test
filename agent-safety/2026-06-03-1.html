一句话核心贡献
把"agent 应不应该开始动手"立成独立安全维度——compliance bias 是 RLHF 结构性副作用，benchmark 当前要么惩罚暂停、要么分不清"原则性 pause"和"静默失败"。
摘要（中文翻译，忠实原文）

当前 autonomous agent benchmark 衡量"agent 是否完成了任务"，但这种框架对"agent 一开始就该不该动手"系统性失明。在 human-feedback 目标下训练出来的 agent 形成结构性的"继续动手"倾向——即使它缺乏必要输入、证据或授权——我们把这种倾向称为 compliance bias，因为奖励信号与 benchmark 评分都把"继续"当作正确默认值，无论安全前提是否满足。

我们做出三项贡献：(1) 我们首先证明 compliance bias 起源于 human-feedback 管道里的 reward hacking，并被主流 agent benchmark 进一步固化——这些 benchmark 要么直接惩罚 agent 停下来，要么在架构上无法区分"原则性 pause"与"静默失败"。(2) 我们引入弃权场景的三-gap 分类法：specification gap（必要信息缺失）、verification gap（世界状态无法确认）、authority gap（明确授权未给出），这三类共同为构造abstention-aware agent benchmark 提供了原则性基础。(3) 我们提出弃权评估三联协议——Safety Rate / Usability Rate / Informed Refusal Rate——并在 144 个企业级 agent 场景 / 5 模型族上给出初步结果：在 runtime-enforced abstention 机制下，可达到 89.2% hazardous-action blocking + 87.5% authorized-scenario usability。这表明 safety-usability 之间的权衡是可调（tunable）而非固有（inherent），且其形状在不同模型族间差异显著。

核心内容解读（compliance bias 的成因 / 三-gap 分类法 / 三联协议 / 实证结果）

问题来源：benchmark 的"动手默认"偏置。过去两年（2024-2026）agent benchmark 经历了两轮叙事跃迁——AgentBench / SWE-bench 把"任务完成率"立成核心指标；2025 后 OSWorld / AgentHijack / TwinRouterBench 又把"对抗鲁棒性"立成第二维。但 Ojewale 与 Venkatasubramanian 指出：所有这些 benchmark 都在隐式地奖励"继续动手"——agent 一旦决定 pause，要么直接 0 分，要么和"卡死了"无法区分。这反过来传导到训练——RLHF / DPO 的 reward signal 把"产出可评分输出"当作 hill-climb 方向，于是 compliance bias 是 reward hacking 的稳定均衡。

三-gap 分类法（论文核心贡献 2）。论文给出"弃权应当发生"的三类形式化场景：

Gap 类型缺失内容示例与匹配场景
Specification Gap（规范差距）必要 input/参数缺失用户未提供目的地却让 agent 订机票；agent 应该先问
Verification Gap（验证差距）无法确认世界状态UI 元素存在性未知 / 文件是否被修改 / 数据库一致性未验证
Authority Gap（权限差距）明确授权未给出跨域操作 / 高敏 API 调用 / 不可逆操作（删除、转账）

这三类 gap 是正交的——一个任务可以同时落入多个 gap，而 agent 应当 pause 的充分理由是"任一 gap 触发"。这是把"agent 该不该动手"分解为可枚举原因的第一次系统化尝试，与 5-21 #1 AgentAtlas 的 "9 类 trajectory failure" 在精神上同源——都是"把暗失败显式化"。

三联评估协议（论文核心贡献 3）。仅有分类法不够，必须给出能上 leaderboard 的可测量数字：

  • Safety Rate（安全率）：在应当弃权的场景中，agent 真的弃权 / 阻断有害动作的比率。论文实测最高达 89.2% hazardous-action blocking。
  • Usability Rate（可用性率）：在授权场景中，agent 没有过度保守、能正常完成任务的比率。论文实测达 87.5%。
  • Informed Refusal Rate（知情拒绝率）：拒绝时是否说明了"缺哪一类 gap"——把 pause 与 silent fail 在评分面上区分开来。这是论文的关键创新：拒绝必须有理由，且理由必须落到三-gap 体系内。

关键实证发现：safety-usability 是 tunable 的。论文最强力的论证不在数字本身，而在：(a) safety/usability 不是不可调和——同一 runtime 下，89.2% blocking 与 87.5% usability 可并存；(b) 不同模型族的 trade-off 曲线形状显著不同——这意味着"safety-usability 边界"是模型属性而非问题属性。这点强烈推翻了"更安全就更没用"的直觉论调，等价于过去 deep learning 里 "accuracy-fairness" 之间的"模型选得好就能两全"的发现。

  • Hazardous-action blocking 89.2%——runtime-enforced abstention 可达上限。
  • Authorized-scenario usability 87.5%——证明 safety/usability 双高同时可达。
  • 144 enterprise scenarios × 5 模型族——首次将 abstention 评测从学术玩具上升到企业真实工作流尺度。
  • Three-gap 分类法——specification / verification / authority 互不蕴含，正交分解 abstention 触发理由。
  • RLHF reward hacking 是结构性根因——把"agent 安全危机"翻译为"训练目标设计问题"。

Harness Engineering（直击毕设）

三-gap 分类法可直接借鉴到你毕设的 KV-Cache 预取场景：specification gap 对应"未给出 attention pattern"——预取 prefetcher 应当 pause 而非乱猜；verification gap 对应"缓存与显存一致性未确认"——这正是 Q.md 关于"预取与缓存数据一致性"的争议点；authority gap 对应"无明确驱逐许可"。建议：在毕设中把 prefetch policy 描述为"带 abstention competence 的预测器"——prefetcher 在三-gap 任一触发时显式退化为保守路径，这能在审稿人面前把"为什么不预取就更对"的反直觉点扎实下来。同时今日方向 #1 的 SkillDAG 可以作为承载该 abstention 决策的 typed graph IR——CONFLICTS_WITH 边对应权限差距，DEPENDS_ON 边对应规范差距。

Agent Skills Safety（直击核心）

这是过去 6 个月该方向最具系统性的"评测面重写"：把"safety"从二元判断拆成三联指标（safe/usable/informed）。SafeHarbor 解决"怎么阻断"，TTT 攻击解决"哪里被打破"，本文给出"怎么度量是否阻断"——这三件事拼成完整闭环。建议：把 abstention competence 与 SafeHarbor / TTT 防御 / Memory Laundering / SKILL.md 绑定，构建一个"结构性 agent safety"研究纲领。论文标记 Best Paper + Oral 表明审稿人共识也在转向这个方向——可以重点关注后续 follow-up。

Safety Benchmark

本文与今日方向 #3 的"benchmark 污染检测可靠性差距"形成镜像——前者揭示 benchmark 未测的维度（abstention），后者揭示 benchmark 测错的维度（contamination 检测失效）。两者合起来构成 2026 上半年 agent benchmark 的双重危机：测什么不全 + 测的不准。建议：未来 SafetyBench / HarmBench 系列升级时，应同时引入 (a) 三-gap × Safety/Usability/Informed Refusal 的 abstention 矩阵，(b) data-provenance verification 替代 statistical contamination detection。这能把 leaderboard 从"谁分高"改写为"谁该停就停且说得清楚"。
