摘要（中文翻译，忠实原文）

多模态 agent 用截图、文档、网页来选择工具调用。当一个错误的视觉 claim 触发了点击、邮件、抽取或转账，幻觉就不再是"答案质量错"，而是 authorization 失败。我们把这一失败模式形式化为 hallucination-to-action conversion：一个未被支撑的 perceptual claim 提供了让特权操作"看起来被允许"的前置条件。

我们提出 Evidence-Carrying Agents (ECA)——把模型自由文本视为 inadmissible evidence。ECA 把每一次工具调用分解为 action-critical predicate；从受约束的 DOM/OCR/AX verifier 处获取类型化证书（typed certificate）；最后由确定性 gate仅在证书支持时授予对应特权。该架构不掩盖感知错误——它把"模型不透明信念"显式翻译为命名 verifier、schema、实现的 residual。

对 verifier 做 1,900 攻击的红队把这一 residual 直接暴露：四步 targeted hardening 把 gate bypass 从 15% 降至 1.3%。在 content-derived certificate 的支持下，ECA 在 200-task 端到端 pipeline 上拿到 0% 的 unsafe-action 率（Wilson 95% 上界 2.67%），在 120-task 浏览器 PoC 上同样 0%（上界 4.3%）。在 500 条 stratified task key 上的 HACR 审计显示：未被支撑的 action-critical claim 在 naive agent 中 100.0% 走到 unsafe execution、prompt-only 防御 49.6%，而 ECA 不会。Oracle-certificate 在 7,488 条 GPT-5.4 trace 上做 gate 正确性 sanity check，同一威胁模型下 neural-judge 基线仍可被绕过。结论一句话：模型语言可以提案动作，但只有外部证据才能授权动作。

核心内容解读（背景·方法·差异）

解决了什么问题（why now）：过去三周 agent safety 主线在围绕"谁在监控、监控什么、监控之后干嘛"展开——5-13 SKILL.md 的供应链攻击（86%/77.6%/100% 绕过）、5-14 HAM³ 的多模态多 agent 三层 ASR 78.3%、5-15 知-行 gap（26.5–54% mismatch）、5-18 ensemble monitoring 的 diversity > scale、5-19 Memory Laundering 的 SPG。这些工作共同指向一个底层困境："模型说什么"不能直接当 evidence 用。ECA 给出的解法非常干净——直接把这一前提从架构层面写死：模型只能"提案动作"，授权必须来自外部 verifier 输出的 typed certificate。

核心方法：HACR 形式化 → 类型化证书 → deterministic gate

  • 形式化 hallucination-to-action conversion (HACR)：每一个工具调用被分解成action-critical predicate——执行该动作所需的所有感知前提，全部以 typed predicate 列出。模型不能"说一句就行"，必须每一条 predicate 都拿到证书。
  • 受约束 verifier 输出 typed certificate：DOM verifier（结构）/ OCR verifier（文本）/ AX verifier（accessibility tree）三类专用 verifier，不知道 agent 的目标，只在自己的本体里给出"该 predicate 是否成立"的类型化证明。这把 perception error 从"agent 说错了"转成"verifier 输出了什么、residual 在哪儿"。
  • Deterministic gate：拿到所有 predicate 的证书后，gate 用固定规则判定是否允许执行——不再依赖 LLM 自我审查。这是"可证明 trust"而不是"prompt 能不能哄住"。
  • 四步 targeted hardening：作者通过 1,900 红队攻击迭代出四个具体加固点（论文给出 verifier-by-verifier 的 hardening 清单），把 gate bypass 从 15% 砸到 1.3%——这是有据可查的残差缩减，而非靠"再调一下 prompt"。
  • Wilson 上界报告：所有"0%" 都附 Wilson 95% 上界（2.67% / 4.3%）——这是 agent safety 论文里少见的、直接给出概率上界的 statistical 严谨度。

对你三个研究方向的启发
Harness Engineering

ECA 给 harness 评测面引入新一档："是否有 typed certificate 支撑每一次 action"。当前所有 agent harness（GAIA / τ-bench / BFCL / 你毕设里的 cache 仿真）都默认 agent action 是被允许的——但如果引入"未拿到证书的 action 一律视为失败"这一硬性 axiom，很多看起来高分的 trace 会立刻退分。这可以被嫁接到 KV-cache prefetch 决策上：把"该 prefetch 哪 K 行"视为一个需要被 verifier（带宽 budget 校验、稀疏 mask 一致性、命中 oracle）授权的动作——任何 prefetch 决策都必须凭证书走 deterministic gate，否则不下发。这样可解释性和可控性都会上一档。

Agent Skills Safety（直接对位）

ECA 是 agent safety 主线的关键一帧。结合 5-13 SKILL.md 攻击、5-15 知-行 gap、5-19 Memory Laundering，可以画出统一图景：不可信源（用户输入、tool 输出、记忆、压缩 summary、skill registry）一律不能直接成为 action 授权依据；必须经 typed verifier 出 certificate。把这条 axiom 同时套到 skill registry 上——SKILL.md 的每条声称能力都必须对应一个 verifier 输出的 capability certificate；套到 memory 上——任何被压缩进 summary 的 claim 都必须保留 certificate 链路；套到 multi-agent A2A——peer model 的输出必须经本地 verifier 才能进入 deterministic gate。这把 agent safety 从"加监控"升级到了"加授权契约"。

Safety Benchmark

对 benchmark 方向，ECA 给出 "HACR audit"这一新 first-order 评测维度：在每一条 trace 上检查"unsupported action-critical claim 是否走到 unsafe execution"。把这一指标嫁接到 BenchJack（5-14）/ DTap（5-7）/ POLAR-Bench / TOBench，可以一次性把"reward hack"和"hallucination-to-action"区分开——前者是 metric 漏洞、后者是授权漏洞。把 Wilson 上界写法纳入 leaderboard 报告规范，也能让 benchmark 圈停止"我跑出 0% 攻击成功率"的不规范声明。
