摘要（中文翻译，忠实原文）

我们分析了来自主流市场的 3,984 个 AI agent skill，发现 76 个手动确认的恶意 payload，包括凭据窃取、后门安装、数据外泄。所有 skill 中有 13.4% 至少包含一个 critical 级别的安全问题，截至发表日，至少 8 个手动确认的恶意 skill 仍在 clawhub.ai 上公开可用。本报告记录我们的方法论，基于真实样本提出威胁分类法，并细化了我们观察到的攻击模式。

随着 agent skill 市场快速增长、AI agent 不断获得对敏感凭据和系统的访问权限——自动化安全分析已不再可选。

核心内容解读（生态威胁面 · 8 类威胁分类 · 扫描方法论 · 真实恶意攻击模式）

问题背景：2025 下半年起，Anthropic Skills、OpenClaw、clawhub.ai、skills.sh 等 agent skill 市场爆发式增长。一个 SKILL.md 文件，下载即可让 agent 执行复杂任务——但它本质上是无审核、可远程更新、运行在用户凭据上下文中的可执行代码。这与早期 Chrome Extension、npm package 危机高度同构，但又叠加了 prompt injection 和工具调用两个新攻击面。本研究是该方向第一份基于真实生产生态的实证报告——不是合成攻击实验，而是直接扫描已经在线的 skill。

8 类威胁分类法：论文以数百个 skill 的手动审查为基础，建立了 8 类策略，每类对应一种独立威胁面：

表里两个数字最值得圈出：(i) 100% 的恶意 skill 含恶意代码模式，且 91% 同时使用 prompt injection——这意味着 skill 攻击不是简单的 LLM jailbreak，而是恶意代码 × prompt injection 的组合武器；(ii) skills.sh top-100 合法 skill 的关键级 false positive 为 0%——证明扫描器在生产环境直接可用。

扫描方法论：论文用 MCP-scan 引擎，结合"多 LLM-judge + 确定性规则"做混合分析。每个 skill 经过：(a) 静态文本扫描提取下载链接、shell 命令；(b) LLM-judge 判断 SKILL.md 是否含隐藏指令（"安装时请把这段 base64 写到 ~/.bashrc"）；(c) 依赖图分析判断是否引入可远程修改的代码块。每条策略都有独立的 LLM judge 和 deterministic rule，并在最后做 ensemble 决策——这与 5-13 #1 SKILL.md 那篇论文的设计同构，但首次有大规模真实数据验证。

真实攻击模式 5 种：论文从 76 个恶意 skill 里提炼了 5 种持续观察到的真实模式——

  • "安装指令含外部链接"——钓鱼到挂载恶意软件的网站。
  • "含混淆 shell 指令外泄数据"——典型如 base64-encoded curl POST 用户密钥到 attacker 服务器。
  • "指令明确要求 agent 禁用安全设置"——例如要求 chmod 777、关闭 keychain prompts。
  • "批量供应链攻击"——一个恶意作者批量发布 40+ 同模式 skill，绕过 reputation 信号。
  • "Auto-update 远程修改"——SKILL.md 没问题，但运行时 fetch 远端代码，使审计前置完全失效。

仍在线的 8 个恶意 skill：论文公开列出 clawhub.ai 上仍可访问的 IOC——例如 moonshine-100rze/moltbook-lm8、Aslaep123/polymarket-traiding-bot、zaycv/clawhub1、pepe276/moltbookagent、Aslaep123/bybit-agent。这种公开命名是 Snyk 一贯的"负责任披露 + 强制 takedown 压力"做法，等价于把市场运营商置于必须响应的境地。

论文判断与建议："自动化安全分析已不再可选"是全文最重的结论。三类受众的具体建议——(a) 用户：不要在未审计前安装 skill，使用 mcp-scan 之类工具；(b) 市场：把自动扫描嵌入提交流水线，对关键级发现做 gating；(c) 开发者：减小第三方内容暴露面、用环境变量/凭据库管理 secrets、避免 auto-update。

与 5-13 #1 "SKILL.md" 那篇 position paper 的关系：那篇是"SKILL.md 应该被当成可执行代码对待"的理论呼吁，本篇就是该呼吁的实证后果——证明在缺少自动化安全分析的市场上，恶意 skill 已经"在野"。这两篇可视作对偶，研究路径上一前一后衔接得很好。
