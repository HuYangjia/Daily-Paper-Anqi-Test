摘要（中文翻译，忠实原文）

随着 Model Context Protocol（MCP）把工具调用作为自主 agent 的标准化界面广泛铺开，它同时引入了一个关键且未被审查的攻击面：错误处理回路（error-handling loop）。我们假设：工具错误消息具有隐式权威（implicit authority）——它们会触发模型的纠错推理模式，从而绕过标准的安全启发式。

我们提出 VATS（Vulnerability Analysis of Tool Streams）——一个变异驱动的框架，沿 7 个结构化与语言学维度系统化地演化对抗 payload。在 Gemini 3.1 Pro / GPT-5.5 / GLM-5.1 / Qwen3-Coder 4 个前沿模型上的实验显示：错误路径注入相对于标准 indirect prompt injection（IPI）的成功率提升 3 倍，在受控评估中合规率最高达到 100%。

我们将结构性定位（structural positioning）——把指令"夹心式" 嵌入错误上下文中——隔离为所有测试模型上最有效的攻击向量。我们发现生产框架的 guardrails 可以缓解这些漏洞，但模型层本身的固有易感性，对自定义 agentic workflow 仍然构成系统性风险。

核心内容解读（错误路径攻击面 / 7 维变异 / 夹心定位 / 与 ECA / Cold-Start / Attack Selection 的关系）

立论：错误处理是 agent safety 上的"暗物质"。Agent safety 圈在 2025 H2-2026 H1 的研究主线，一直在正向攻击面上转：5-13 SKILL.md 注册可信、5-19 memory laundering、5-21 CUGA Governance by Construction、5-25 ECA / Hallucination as Exploit、6-08 Attack Selection。VATS 第一次把视角扭转到"反向通道"——agent 接收工具返回这一段。核心赌注：模型对"错误信息"有结构化先验偏置——它会立刻进入"纠错推理"模式，并对错误中携带的字符序列赋予比正常上下文更高的指令权重。这种先验偏置就是论文标题里的 implicit authority。

7 维变异演化：MCP agent 时代的红队工程模板。论文核心方法不是单点攻击，而是一个变异驱动的进化框架，沿 7 个结构化与语言学维度系统化生成 payload：

维度（推断）语义对照"夹心定位"贡献
结构性位置指令在 error message 内的相对顺序（前 / 中 / 后）最强单维度——把指令包在 error 上下文之间收益最高
消息语气（mood）命令式 / 祈使式 / 解释式 / 警示式提高 agent "解读为合理修复指引" 的概率
错误类型伪装HTTP 5xx / Timeout / Auth / Schema mismatch 等选择最易激活纠错推理的错误类
编码格式纯文本 / JSON / YAML / 半结构化覆盖 MCP 协议各种 trace serializer
身份伪装"system" / "retry hint" / "upstream notice"诱发"这是协议层而不是用户输入" 的错误归因
修辞策略请求式 / 紧急式 / 礼节式 / 反讽式跨语种迁移基础
扰动密度注入 payload 占整条 error 的 token 比例测试饱和度阈值

"夹心定位"是新的 universal pattern。论文的最强发现：把恶意指令夹在 error 上下文的中间段，对 4 个前沿模型同时有效——这与 5-25 ECA / Hallucination as Exploit 把 hallucination 重写为授权失败形成强对称：ECA 看到"正确产物绕过授权"，VATS 看到"正确返回回路本身就是授权通道"。两篇背靠背把"可信通道边界"在 MCP 时代立成 agent safety 第一议题。

与今日 #2 候选 Cold-Start Safety Gap 的对仗。同日 Cold-Start Safety Gap 揭示 agent 在会话最初几步最不安全——表征还没漂移到"安全对齐区域"。VATS 暴露的是另一个时间窗：每次工具调用后的错误响应处理瞬间，模型立刻进入纠错推理子状态，与会话首部一样可视为临时性安全降级窗口。两条工作合起来构成 agent 时间维度上的"双脆弱窗"图谱——会话开始时不安全，错误响应后也不安全。

与 6-08 Attack Selection 的呼应。6-08 Attack Selection 把红队决策分解为 start/stop 两 policy，并指出"攻击时机选择" 即可将安全度压低 20-28 pp。VATS 给出了"start policy 应该挑何时" 的具体语义答案：挑工具错误返回的瞬间——因为那一瞬间模型本身在"积极找解释" 而不是"守规则"。两篇连起来：Attack Selection 给方法学，VATS 给战术；安全评估的 ASR 上界因此还能继续下沉。

缓解：guardrails 是创可贴而不是治本药。论文有意识地报告了一个"软结论"——生产 framework guardrails 可以缓解，但模型层本身的固有易感性对自定义 agentic workflow 构成系统性风险。这意味着：不能依赖 wrapper 解决。可能的治本方向：(1) 把"error 通道" 在协议层独立 sandbox（参考 5-21 CUGA 的 5 检查点架构）；(2) 让模型在 error 上下文上显式降权，类似指令层级里的 system / user / tool 三档（与 6-08 候选 Where Instruction Hierarchy Breaks 互补）；(3) 在错误返回前过 schema gate（与今日方向 #1 SKILL.nb 的 gate 机制天然连接）。

对你三个研究方向的启发
Harness Engineering

VATS 第一次把"错误返回路径"立成 harness 评估的独立通道——正常工具输出和错误工具输出在 MCP agent 协议中走同一个 LLM 上下文窗，但模型对它们的处理方式不一样。建议：在 harness 设计中独立报告"正常路径鲁棒性" 与 "错误路径鲁棒性" 两个指标。这与今日 #1 SKILL.nb 的 gate 机制天然耦合：err-cell 必须独立 gate，而不是和 normal-cell 共享上下文进入下一步。这也直接启发你毕设的 cache eviction 决策：当 prefetch 触发"越界 / 失败" 时，错误回路本身不能携带对下一次预取策略的隐式 instruction——这条原则在 VATS 框架下首次有了 concrete 反面教材。

Agent Skills Safety（直击方向）

VATS 把 agent safety 评估补全到一个关键缺失维度：所有以前的 IPI 工作都假设攻击者只能注入用户提供的工具输入，VATS 证明工具产生的错误输出 同样可控且更具杀伤力。建议：把 AgentHarm / AgentSafetyBench / SODA 等基准升级为"正常 + 错误" 双通道版本——同一个攻击意图，分别从工具输入侧和工具错误侧注入，记录两侧 ASR 差距。这条差距本身就是模型在"错误路径上的隐式权威偏置" 的量化指标。同时把 VATS 与 ECA 联立：ECA 关心"授权失败 → 不被检测的正确产物"，VATS 关心"失败响应 → 被高权重当作 instruction"，它们是同一座房子的正反两个门，agent safety 的 audit 必须同时关上两道门。

Safety Benchmark

VATS 给 benchmark 提了一个全新维度：error-path ASR uplift——攻击者只切换注入位置（从输入侧切到错误返回侧）就能获得多少额外 ASR。这是结构性指标，不依赖具体 payload 内容。建议：未来 agentic safety benchmark（包括今日 #3 MAC-Bench）应同时报告 input-path ASR 与 error-path ASR 的差值。这与 5-23 latent-space evasion 把"潜空间方向" 立成几何指标、6-08 Attack Selection 把"时机" 立成方法学指标，三轴合一构成 agentic safety benchmark 的结构性诊断三件套：位置 + 几何 + 时机。
