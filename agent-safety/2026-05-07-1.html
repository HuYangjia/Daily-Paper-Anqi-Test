一句话核心贡献
Agent 安全的"沙盒/守卫/post-hoc"三类各管一段，AgentTrust 把它们合并到 Tool 调用前的一个裁决面上：allow / warn / block / review。
摘要（中文翻译，忠实原文）

现代 AI Agent 通过工具调用对世界产生真实副作用——文件操作、shell 命令、HTTP 请求、数据库查询。一次不安全的动作就足以造成不可逆伤害：误删、凭据泄露、数据外泄。现有防御不完整：

  • Post-hoc benchmark——只在执行之后测量行为；
  • 静态守卫——遗漏混淆与多步上下文；
  • 沙盒——约束代码在哪里跑，但不理解动作意味着什么。

本文提出 AgentTrust——一个运行时安全层，在执行前拦截 Agent 工具调用并返回结构化裁决：allow / warn / block / review。AgentTrust 由四个组件组成：shell 反混淆归一化器、SafeFix（更安全替代建议）、RiskChain（多步攻击链检测）、cache-aware LLM-as-Judge（处理歧义输入）。

作者发布两个 benchmark：300 场景 × 6 类风险的内部 benchmark 与独立构造的 630 场景真实对抗 benchmark。在内部 benchmark 上仅靠产线规则集即可达 95.0% 总裁决准确率 / 73.7% 风险等级准确率，端到端延迟为低毫秒级；在 630 场景外部 benchmark 上（评估时使用了打补丁的规则集，不主张零样本），AgentTrust 达 96.7% 总裁决准确率，其中对 shell 混淆攻击约 93%。AgentTrust 在 AGPL-3.0 下发布，并提供一个 Model Context Protocol (MCP) 服务器，可被任意 MCP 兼容 Agent 直接接入。

核心内容解读（背景·方法·差异）

解决了什么问题：过去一年这条 Agent Safety 主线已经把"事故什么时候发生"切成了三段研究：执行前（Symbolic Guardrails 在策略上"可证明"，但需要预先有 policy）、执行中（Owner-Harm、OS-BLIND 揭示运行时空白）、执行后（Human-Guided Harm Recovery）。AgentTrust 第一次给"执行前 N 毫秒"这个时间窗设计了一个完整、可部署、毫秒级延迟、覆盖混淆/多步/歧义的运行时安全层，并且不预设事先有 policy——它自己就承担"现场判断"的角色。

四大组件如何协同：

组件功能解决的现有空白
Shell Deobfuscation Normalizer把混淆 shell 命令归一化（base64/eval/aliasing/quote-tricks）静态规则误判混淆为良性
SafeFix对潜在风险动作给出"更安全替代"（如 rm → trash, curl|sh → 可审计安装器）沙盒/守卫只能 block，无法引导回正轨
RiskChain检测跨步骤的攻击链（如读凭据 → 改 .ssh/config → 外发）Symbolic Guardrails 假设 policy 已知
Cache-aware LLM-as-Judge对仅靠规则无法判定的歧义动作做语义裁决，并复用判定缓存朴素 LLM-judge 延迟过高、规则系统覆盖不足

与现有工作的核心差异：

  • 对比传统沙盒（Docker/seccomp/firejail）：沙盒约束能力面但不理解语义；AgentTrust 在调用层做语义级 verdict。
  • 对比 Symbolic Guardrails：后者要求预先存在显式策略；AgentTrust 自带 LLM-judge 兜底，无 policy 也能上线，可与 Symbolic Guardrails 互为前后端（Symbolic 提供高保证主干，AgentTrust 处理 policy 真空）。
  • 对比 BOA：BOA 把"安全度量"改成搜索（评估侧），AgentTrust 把"安全干预"做成实时拦截（部署侧）——评估-部署两条腿同时落地。
  • 对比 post-hoc benchmark（HarmBench/AgentBench-Safety）：它们事后量化伤害；AgentTrust 让伤害根本不发生。
  • 开源策略：AGPL-3.0 + MCP server——任何 MCP 兼容 Agent（含 Claude Desktop/IDE/自研）都可零改动接入，这是迄今为止最低门槛的运行时安全层接入方式。

核心数据亮点 / 关键论点
  • 内部 benchmark 95.0%：300 场景 × 6 类风险，仅靠产线规则集（无 LLM-judge 兜底）。
  • 外部 benchmark 96.7%：独立构造 630 场景对抗集；论文明确披露使用打补丁规则集，不主张零样本——这种诚信披露在该领域并不常见。
  • shell 混淆攻击 ~93% 拦截：这是现有静态守卫最薄弱的一环；AgentTrust 的 Deobfuscation Normalizer 单点收益显著。
  • 低毫秒级 E2E 延迟：对线上 Agent 来说"零感知拦截"——cache-aware LLM-as-Judge 的关键工程贡献。
  • AGPL-3.0 + MCP：第一份"开箱即用"的运行时 Agent 安全层；对 MCP 生态有可能形成事实标准位。
  • 结构化 verdict（4 类）：allow/warn/block/review 设计远比"true/false"灵活——为人类介入（review）和自动改写（SafeFix）留足空间。

Harness Engineering

AgentTrust 的 verdict 流可以直接接入今日 Harness 报告 Uno-Orchestra 的奖励函数：把"被 block / warn 次数"作为编排策略的硬约束/软约束信号——让"安全编排"的目标显式进入 RL 而不是事后修正。这种"runtime safety verdict ↔ RL 奖励"的耦合，会成为下一阶段 production harness 的标配。

Agent Skills Safety

AgentTrust 让 Agent Safety 真正进入"工程可消费"阶段——AGPL-3.0 + MCP server 意味着研究产出能立即变成生态共享件。下一步该做的是把 Symbolic Guardrails 的可证明策略与 AgentTrust 的 LLM-judge 兜底正交合层（symbolic 主干 + LLM 兜底 + 缓存）；同时把 Human-Guided Harm Recovery 的偏好学习反哺 SafeFix 的"安全替代生成"。

Safety Benchmark

"内部 300 + 外部 630 + 不主张零样本"是一个值得普及的 benchmark 报告范式——明确区分 in-distribution 与 wild distribution、明确披露规则是否打过补丁。和今日 Benchmark 报告 DTap 的 14 域 × 50 环境结合，可形成"DTap 出题 + AgentTrust 应试"的端到端 red-team 闭环：DTap 自动注入攻击向量，AgentTrust 给出 verdict，verdict 错误回流为新 benchmark 样本。
