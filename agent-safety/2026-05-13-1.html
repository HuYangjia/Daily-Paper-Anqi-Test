一句话核心贡献
把 SKILL.md 当作"操作性文本"而非被动文档：在 Discovery / Selection / Governance 三阶段揭示 86%–100% 的语义攻击面。
摘要（中文翻译，忠实原文）

自治 AI agent 越来越多地通过 Agent Skills 来扩展能力——这是一种模块化的文件系统包，其 SKILL.md 文件描述 agent 应当何时、如何使用该技能。该设计支持可扩展、按需的能力增长，但同时引入了一类语义供应链风险（semantic supply-chain risk）：自然语言元数据与指令本身可以影响哪些 skill 被收录、被展示、被选用、被加载。

我们针对 Agent Skill 生命周期中三个面向 registry 的阶段研究"SKILL.md-only 攻击"，使用真实 ClawHub skills 与真实 registry 机制：

(1) Discovery 阶段：短文本触发器可以操纵基于嵌入的检索系统，提升对抗性 skill 的曝光度——pairwise win rate 最高 86%，Top-10 出现率 80%。

(2) Selection 阶段：仅靠 description 文本框架，就能让 agent 在功能等价的 skill 之间偏好对抗性变体——平均 77.6% 配对试验选中对抗性 skill。

(3) Governance 阶段：语义规避策略让恶意 skill 避开 blocking verdict 的概率达到 36.5%–100%。

总体来看，SKILL.md 不是被动文档，而是塑造 agent "找谁、信谁、用谁"的操作性文本。

核心内容解读（背景·方法·差异）

解决了什么问题：过去两个月 Agent Safety 方向的核心进展集中在运行期：AgentTrust 给 tool call 加 allow/warn/block/review；SafeHarbor 做分层记忆 guardrail；Skills as Verifiable Artifacts 给 skill 加 trust schema 与 capability gate。但所有这些工作都假设 skill 本身是可信注册的对象。本文把镜头拉到了上游——skill 还没被 agent 调用之前，registry 这一层就已经被自然语言操纵了。这与 Agent Skills 兴起本身（Anthropic 等已把 SKILL.md 作为开放协议）形成时间上的同步攻击面。

三阶段攻击方法（Discovery → Selection → Governance）：

  • Discovery（被检索到）：在 SKILL.md 描述里嵌入短文本触发器，操纵 embedding-based retriever。结果：pairwise win rate 最高 86%，Top-10 placement 达 80%。这是 retrieval 层"被推上货架"的能力——比传统 SEO 更精细，因为它直接作用于 embedding 几何。
  • Selection（被 agent 选用）：当 retrieval 同时返回多个功能等价 skill，agent 用 description 做选择。仅通过措辞框架（framing）——不更改功能——可让 agent 在 77.6% 的配对试验中选中对抗版。这是 LLM 在元数据层偏好可被操控的直接证据。
  • Governance（绕过审查）：面对带语义审查的 governance 检查器，恶意 skill 用语义规避策略（rephrasing / abstracting / role-shifting 等）将"blocking verdict"绕过率推到 36.5%–100% 区间。这意味着当前 skill registry 的语义守卫并不可靠。
  • 使用真实 ClawHub skills：不是合成 toy registry，而是用真实在用的 skill 包与 registry 机制——研究外部有效性显著强于 lab-only 攻击。
  • SKILL.md-only 假设：所有攻击都不动技能代码、不修改运行期；只动 SKILL.md 文本。这是把"语义层"作为单独攻击面隔离出来——比代码植入更隐蔽，也更难被传统软件供应链工具检出。

核心数据亮点 / 关键论点
  • Discovery 阶段：短文本触发器使对抗 skill 在 embedding 检索中 pairwise win rate 最高 86%，且 80% 进入 Top-10——直接威胁所有"按相似度排序"的 skill registry 设计。
  • Selection 阶段：description-only 措辞框架使 agent 在 77.6% 的配对试验中偏向恶意变体，且功能完全等价。这是首次量化"agent 在 skill 元数据上的偏好可被零代码改动操纵"。
  • Governance 阶段：语义规避策略将 governance 拦截率从 0% 推到 36.5%–100%，说明当前 LLM-as-policy 的审查器不能独自承担 skill 注册守门人的角色。
  • 真实环境：使用真实 ClawHub skills 与真实 registry 机制，而非合成 toy——攻击的外部有效性是该工作最大的方法学贡献。
  • 三阶段视角：把过去散落在"prompt injection / SEO / typosquatting"的攻击叙事统一到一条 lifecycle 链上，给后续 skill registry 标准制订提供了清晰防御坐标。
  • 方法学定位：把"自然语言元数据"作为独立攻击面从代码与 prompt 中剥离——这与今日 Harness 选定的 ED 工作（评估元数据可被识别）共同指向"semantic-layer security" 这一正在出现的新范式。
对你三个研究方向的启发
Harness Engineering

SKILL.md 攻击天然要求一类新的 harness：skill registry harness——量化 retrieval / selection / governance 三阶段在不同 framing / trigger / 规避策略下的 ASR-utility 曲线。可与今日 Harness 选定的 Evaluation Differential 直接合并：每条 SKILL.md 都应在 harness 输出里被标注 ED 类型（被审计时看似良性 / 部署时被选用），这就把 skill registry 安全做成"带条件性声明"的评测体系。对 ZipperQuant 类工作的启发：你的量化模型同样要在 deployment 端被一个 skill registry 调用——如果 W4A4 模型对 SKILL.md 措辞偏置比 BF16 更敏感，那就是 quality regression 的一种新刻画维度。

Agent Skills Safety

这是自 OS-BLIND 之后最具威胁的 agent safety 论文——它把"skill ecosystem"作为整体威胁面而非单点攻击对象。直接催生四个具体研究问题：(1) 如何设计 SKILL.md sanitiser——在不破坏功能描述的前提下抹除 trigger / framing 偏置；(2) 如何把 Skills as Verifiable Artifacts 的 trust schema 扩展到 description text 而不仅是接口；(3) 如何让 capability gate 在注册阶段就预运行（pre-admission audit）；(4) 是否需要类似 npm 的 skill provenance + reproducible build 体系。这四个问题中任何一个都足以成为独立 NeurIPS/USENIX 主题。

Safety Benchmark

这篇本身就建立了一个可以拓展的 benchmark：以三阶段（Discovery / Selection / Governance）×多种攻击策略×真实 registry 为坐标。今日 Benchmark 选定的 TRIVIA+ 是 RAG 检测 benchmark——其方法论（desiderata + 噪声标签变体）可以平移过来：给 skill registry 攻击 benchmark 设计一组"noise schedule + label noise"以衡量防御方法在真实部署条件下的鲁棒性。再加上 ED 类型标注，就能形成"skill-registry-RedTeam-Bench"——这是当前缺口最明显的 safety benchmark 类别之一。
