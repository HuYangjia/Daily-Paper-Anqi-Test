摘要（中文翻译，忠实原文）

多模态多智能体系统（MM-MAS）因其在不同模态间进行复杂推理与协调的能力而日益受到关注。当这些系统在规模与功能上不断扩张，研究其潜在漏洞的重要性也随之上升。然而，已有针对多 agent 系统的对抗性攻击研究主要聚焦在单 agent 或 单模态 设定，MM-MAS 的脆弱性在很大程度上仍未被探索。

为弥合这一空白，我们提出 HAM³——一个面向 MM-MAS 的层次化攻击框架，把攻击解耦成三个互联的层级：

  • 在 感知层（perception layer），HAM³ 通过扰动视觉输入、文本输入、以及融合后的视觉-文本表征发起攻击；
  • 在 通信层（communication layer），它进行通信级攻击——污染消息内容、操纵交互拓扑（如共享上下文、通信链路），从而扭曲集体信息流；
  • 在 推理层（reasoning layer），它干预每个 agent 的认知 pipeline，偏置推理轨迹，并最终影响最终决策。

我们在 GQA benchmark 上评估 HAM³，所用多 agent 系统建立在 ReAct、Plan-and-Solve、Reflexion 三种不同 reasoning 范式上。实验显示：HAM³ 实现了高达 78.3% 的攻击成功率，其中推理层攻击最有效；并且超过一半的成功攻击让多个 agent 同时给出一致的错误答案。这些发现为构建更鲁棒、可解释的多 agent 智能提供了有价值的洞见。

核心内容解读（背景·方法·差异）

解决了什么问题（why now）：过去半个月 Agent Skills Safety 已经把"注册期"（Skills as Verifiable Artifacts）、"调度期"（PRISM）、"语义供应链"（SKILL.md）依次外化。但这些工作有一个共同假设——系统是单模态、文本主导的。MM-MAS 是一个截然不同的攻击面：感知和通信本身就是攻击载体。HAM³ 是首篇把 MM-MAS 攻击系统化分解成三层并量化每层贡献的工作；这与现有 prompt-injection / jailbreak 主导的安全文献是正交的。

核心方法（perception · communication · reasoning 三层 + 跨范式评估）：

  • 感知层（Perception Layer）：分别扰动视觉输入、文本输入、以及视觉-文本融合表征——这把传统对抗样本从"图像扰动"扩展到"跨模态融合扰动"，是 MM-MAS 独有的攻击面。
  • 通信层（Communication Layer）：两类攻击同时存在——消息内容污染（在共享 context 中插入误导事实）和拓扑操纵（修改通信链路，让某 agent 被孤立或被强制信任某 agent）。这与传统单 agent prompt injection 不同，攻击的是群体信息流。
  • 推理层（Reasoning Layer）：直接干预每个 agent 的 cognitive pipeline，偏置 reasoning trajectory。摘要明确说"reasoning-layer attacks 最有效"——这是该论文最反直觉的发现：花精力做对抗样本可能不如直接污染思维过程。
  • 跨 reasoning 范式：在 ReAct（act/think 交错）、Plan-and-Solve（先 plan 再 execute）、Reflexion（带反思）三种范式上同时测——这相当于对"是否有 reasoning step 即更鲁棒"做了一个反命题验证。
  • GQA benchmark：选 GQA（图问答）作为评测域，因其同时考验 visual grounding + 多步 reasoning，是 MM-MAS 的典型场景。

核心数据亮点 / 关键论点
  • 78.3% Attack Success Rate：HAM³ 在 GQA + MM-MAS 上的最高攻击成功率，跨 ReAct / Plan-and-Solve / Reflexion 三大 reasoning 范式都成立——这意味着"给 agent 加 reasoning step"并不天然增加鲁棒性。
  • reasoning-layer 攻击最强：在三层中，推理层而非感知层贡献最大攻击成功率。这是反直觉的——对抗样本社区长期认为感知层最危险，HAM³ 把"防御重心"明确推向 cognitive pipeline。
  • 50%+ 攻击让多 agent 集体犯一致错误：这是论文最重要的安全启示——多 agent 投票/共识机制不仅不能稀释错误，反而被攻击者反向利用。这直接挑战了"agent ensemble = 自然防御"的设计假设。
  • 跨 paradigm 普适性：ReAct（执行-思考交错）、Plan-and-Solve（先 plan 后 act）、Reflexion（带反思）三大范式在 HAM³ 下都受同等程度影响——意味着"选哪种 reasoning paradigm"几乎不是 MM-MAS 安全的杠杆。
  • 三层联合而非分立：HAM³ 强调三层"互联"而非独立——单层防御不足以阻断攻击，因为攻击可以跨层传播。这给 agent safety 防御设计提了一个相当硬的约束：必须做 vertical defense。
  • CVPR 2026 接收：在视觉/多模态顶会获得认可，意味着该工作的工程严谨度（perturbation budget、evaluation 协议、复现性）经过严格审稿——是目前为数不多同时被 CV 与 agent safety 社区接受的工作。
对你三个研究方向的启发
Harness Engineering

HAM³ 与今日 Harness 选定的 TRIAGE 形成"红队对位"：TRIAGE 测前瞻性元认知 plan的最优性，HAM³ 揭示plan 阶段如果存在多模态/通信扰动 会让多 agent 集体偏离。两者拼起来给 harness 工程一个新课题——"adversarial-aware metacognition harness"：在 TRIAGE 设定下注入 HAM³ 三层扰动，看 plan 阶段的 efficiency ratio 是否退化。这对你的 ZipperQuant W4A4 e2e benchmark 也是重要参考：W4A4 在多模态多 agent 场景下的鲁棒性 vs 单模态 LLM 上的鲁棒性可能完全不同，不能直接外推——这值得在毕设论文里写进 limitation 一节。

Agent Skills Safety

HAM³ 完成了 agent safety 攻击面的"下移闭环"：5-11 注册（Skills as Verifiable Artifacts）→ 5-12 调度（PRISM）→ 5-13 语义供应链（SKILL.md）→ 5-14 多模态执行（HAM³）。跨层防御从此成为不可回避的工程命题——单层 filter 防不住跨层攻击。这条经验可以直接用到你 ZipperQuant 量化的安全性论证：weight outlier、激活 outlier、KV-cache outlier 三类异常各对应一类潜在"植入面"，必须验证组合后是否依然鲁棒，而非分别证明。"50%+ 集体一致错误"也提醒——agent ensemble / 多专家投票 / multi-rollout 等机制都可能成为放大器而非稀释器。

Safety Benchmark

HAM³ 给 safety benchmark 社区留下了三个尚未被解决的"层级化设计要求"：(1) 多模态 + 多 agent 场景的 benchmark 极少（GQA 只是单跳样本）；(2) "集体一致错误"作为新的安全度量从未在 HarmBench/SafetyBench 出现；(3) reasoning paradigm 不应作为防御变量进入 benchmark 评分。今日 Benchmark 选定的 BenchJack 和 HAM³ 共享一个母题——benchmark 自身需要 secure by design。一个直接的研究契机：把 BenchJack 的 reward-hacking taxonomy 与 HAM³ 的三层攻击 taxonomy 合并，做一个统一 benchmark security taxonomy——这是 NeurIPS 2026 datasets & benchmarks track 的高潜力题。
