摘要（中文翻译，忠实原文）

基础模型的最新进展已把 LLM 从被动对话系统变成能够推理和工具执行的自主 agent。这些能力释放了重要的实用价值，但也引入新的安全风险——攻击者可能操纵 agent 在真实环境中执行有害动作。现有防御策略可以缓解这些威胁，但常常难以平衡安全与效用，导致对良性用户请求的过度拒绝（over-refusal）。

为缓解这一权衡，本文提出 SafeHarbor——一个为 LLM agent 建立精确决策边界的新颖框架。SafeHarbor 不依赖静态守则，而是通过增强对抗生成提取上下文感知的防御规则。我们设计了一个本地分层记忆系统用于动态规则注入，提供 training-free、高效、即插即用 的解决方案。此外，我们还引入基于信息熵的自演化机制，通过动态节点分裂与合并持续优化记忆结构。

大量实验表明 SafeHarbor 在模糊良性任务和明确恶意攻击上均达到 SOTA，特别是在 GPT-4o 上达到峰值良性效用 63.6%，同时对有害请求维持超过 93% 的稳健拒绝率。源代码已公开在 github.com/ljj-cyber/SafeHarbor。

核心内容解读（背景·方法·差异）

解决了什么问题：这条研究线在过去两个月反复指向同一痛点——agent guardrail 的"safety-utility 跷跷板"。SKILL.md（5-13） 揭示静态守则被 86% / 77.6% / 100% 绕过，CUGA（5-21） 用 5 个结构化检查点 + policy-as-code 但仍是离线规则集，ECA（5-20） 用 typed certificate gate 把 unsafe 打到 0% 但需要应用侧重写。SafeHarbor 选了第三条路：既不重训模型权重、也不重写应用接口，而是把 guardrail 做成一个自演化的本地分层记忆系统，通过对抗生成和信息熵自动产出 / 调整规则。

核心方法：三件正交配合的机件

  • 增强对抗生成（Enhanced Adversarial Generation）——不靠人工写守则，而是让对抗生成器主动产生"边界 case"，从这些 case 蒸出"上下文感知"的防御规则。这是 SkillSafetyBench（5-12）所说"本地化非用户攻击可以持续诱导不安全行为"的反向解药——既然攻击面是本地工件 / 技能上下文，规则就必须按上下文动态生成。
  • 本地分层记忆系统（Local Hierarchical Memory）——把规则组织成树状层级，按动态查询路径激活相关规则注入到 agent prompt 中。"本地" + "分层" + "动态注入" 三件一起保证了 training-free + plug-and-play 这两个工程属性。
  • 信息熵自演化（Entropy-Based Self-Evolution）——以记忆树各节点的熵为信号，过高即分裂（说明该节点覆盖太杂），过低即合并（说明冗余），让记忆结构自己漂移到与威胁分布匹配的形态。这一点借鉴了 b-tree / lsm-tree 的运行时再平衡思想，但应用到了"规则空间"。

对你三个研究方向的启发
Harness Engineering（直击毕设）

SafeHarbor 的 "分层记忆 + 熵驱动分裂/合并" 是一份对你毕设双水位线 KV-cache 管理的极佳灵感互文：(1) 把"热缓存 / 温缓存 / 冷卸载"看作三层记忆树，根据"访问熵"动态分裂或合并，是双水位线策略的逻辑等价物；(2) "training-free + plug-and-play" 这个工程美学正好对应你毕设强调的"不改 DSA 模型权重，只改 KV runtime"边界；(3) 论文给出"同时优化 efficiency-side 与 safety-side"的双指标双 SOTA，可借鉴为你 hit-rate / bandwidth 双轴报告模板的思路。可在论文 chapter 6 引为"运行时数据结构自演化"的相关工作。

Agent Skills Safety（核心）

SafeHarbor 是这一波 guardrail 路线里最完整的一份"第三条路答卷"——不重训、不改应用、自演化。具体可记忆 4 件：(a) 增强对抗生成 = 自动出 SKILL.md 类 的攻击 case 反向蒸规则；(b) 分层记忆树 = 把 CUGA 的 5 检查点软化为可动态生长的层级；(c) 熵自演化 = 给 ECA 的 typed certificate 提供"规则空间维护"的运行时机制；(d) ICML 2026 接收意味着这条路线已被会议侧承认。建议把 SafeHarbor + ECA + CUGA + SKILL.md 串成你 6 月小组分享的"guardrail 三路线"主线。

Safety Benchmark

对 benchmark 设计的启发：必须同时报告 safety-axis 与 utility-axis，不能只看一个。SafeHarbor 给出 63.6% / 93%+ 这对双指标对 benchmark 协议设计是直接示范——GrandGuard（contextual risk 三级）、POLAR-Bench（5×5 诊断面）、即将到来的 Boiling the Frog（多轮 ASR）都应在报告时把 over-refusal / utility 一同报告，否则方法论将不可比较。SkillSafetyBench（5-12, 155 cases）和 SafeHarbor 形成攻击-防御的最佳配对。
