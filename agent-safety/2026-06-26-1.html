摘要（中文翻译，忠实原文）

现代开放世界 agent（如 OpenClaw）展现出强大的跨环境执行能力，但同时引入了广泛的新型安全风险源。与此同时，前沿 AI 模型大幅降低了攻击门槛，让现有 agent 对齐框架在真实部署中显得不足。为应对这些新兴威胁，我们提出一个轻量、可扩展的 agent 安全对齐框架。具体地，我们更新 agent 安全分类法，使其涵盖 Codex 与 OpenClaw 执行场景中涌现的风险。我们进一步构建分类法引导的数据引擎 + 影响函数净化，仅用约 1k 样本就训出 AgentDoG 1.5 的轻量版本（0.8B / 2B / 4B / 8B 参数），达到与领先闭源模型（如 GPT-5.4）相当的性能。基于 AgentDoG 1.5，我们构建了一个极高效的 agentic 安全 SFT 与 RL 训练环境，让 Docker 级环境的部署开销降低两个数量级。最后，我们把 AgentDoG 1.5 部署为训练免费 的在线护栏，做实时安全审核。大量实验表明，AgentDoG 1.5 在多样化、复杂交互的 agent 场景中达到了 state-of-the-art。所有模型与数据集均开源发布。

核心内容解读（三维分类法 / 影响函数净化 / FSM-RL 环境 / 训练免费护栏）

立论：agent safety 已经进入 OpenClaw / Codex 时代——分类法必须更新。2024–2025 H1 的 agent safety 链条主要围绕"tool-use safety + prompt injection"。但 2026 H1 真正的攻击面已经从单轮 prompt 拓展到跨会话持久状态污染、技能/插件供应链、依赖项 MCP 入侵、仓库工件注入、破坏性工作区变更——这些都是 OpenClaw（开放世界 agent）与 Codex（代码执行 agent）专属的新攻击面。AgentDoG 1.5 直接更新分类法，保留 1.0 的三维分解（Risk Source × Failure Mode × Real-world Harm），但为两类新场景扩展叶子类别——这一点直接对应 6-25 SkillSafetyBench 提出的"非用户三位置攻击面" 分类，但更工程化（5,973 个独特工具与 MCP 服务器，15 来源 × 21 模式 × 11 危害）。

分类法的三维结构 — 这是 AgentDoG 1.5 全篇的"骨架"。

  • Risk Source（15 种）：用户指令 / 工具描述 / 环境观察 / 持久状态 / 运行时反馈 / 仓库工件 / agent 自身推理 / ……
  • Failure Mode（21 种）：错误工具调用 / 越权操作 / 未验证外部信息 / 不安全命令执行 / 未验证的成功声明 / ……
  • Real-world Harm（11 种）：隐私泄露 / 系统完整性损害 / 财务损失 / 物理-心理-声誉损害 / 新增治理级危害（合规、法律、可审计性）

OpenClaw 场景新增叶子：发送者/会话身份模糊、持久内存污染、技能/插件供应链入侵、策略优先级误读、审批绕过、操作范围越权、跨工具攻击链、跨通道路由错误、不安全的无人值守自动化；Codex 场景新增叶子：仓库工件注入、依赖/MCP 供应链入侵、破坏性工作区变更、不安全 shell/script 执行、提示注入（精炼）、损坏的工具反馈、未授权信息泄露（覆盖仓库密钥/环境变量/凭证文件）。

影响函数净化：把 32,787 样本压到 ~1,000。这是 AgentDoG 1.5 最关键的方法学创新。给定参考检查点 θ̂，对每条候选样本计算偏好感知影响函数得分：先算目标响应梯度 ĝ(q,y) = ∇θℓ̄(q,y;θ)|θ=θ̂，再用模型对正确响应的归一化偏好 π̂q = p̄(y+|q) / (p̄(y+|q) + p̄(y−|q)) 作权重，聚合护栏方向 ĝguard = (1/|Qsafe|) Σ π̂q(ĝ(q,y+) − ĝ(q,y−))，最后用 sπ(z) = ĝz⊤ĝguard 给每条候选打分。只保留与护栏方向最对齐的样本，从而：(a) 训练样本压到 ~1,000，训练成本下降 30+ 倍；(b) 缓解对虚假模式的过拟合（避免"看似拒绝实际是关键词匹配" 的浅层 alignment）。这与 6-23 AgentLens 的"refusal subspace 干预" 思想是同一方向的对偶——一个在数据层用影响函数过滤，一个在表征层 用 subspace 干预。

RL 环境：有限状态模拟器替代 Docker。AgentDoG 1.5 的另一个工程创举是用LLM 生成的有限状态 Python 模拟器替代 Docker 环境：单台 8 核机器支持 10,000+ 并发 agent 环境，峰值内存严格 ≤ 2.5 GB，1,000 并发工具调用下延迟稳定。奖励模型由 AgentDoG 1.5 自身担任外部 judge，R = U（clean）/ S（malicious-query）/ 0.25·U·S + 0.25·S + 0.5·U（env-injection）。算法用 GDPO（Group Reward-Decoupled Normalization PO），把 Docker-RL 的两个数量级开销削掉。这一点对你做 KV-cache / SGLang 系统毕设有直接借鉴价值——FSM 简化 RL 环境的思想可平移到 KV-cache 策略搜索的训练环境。

SFT 应用效果（Table 4 摘要）—— alignment 不掉点。用 AgentDoG 1.5-Filt 数据微调 Qwen3.5-4B：AgentHarm Harm Score 57.49 → 20.32（−65%）；Refusal Rate 28.41 → 75.00（+165%）；AgentSafetyBench SR 34.37 → 53.23；AgentSecurityBench ASR 90.39 → 23.82；BFCL 通用能力 76.04 → 81.12（不仅没掉，反而 +5）。这是safety-utility Pareto 同时改善 的稀有数据点，证明影响函数净化得到的是真正的安全核心，没有伤害通用能力。

位置：与 SkillSafetyBench / AgentLens 共构 H1 末"攻击面 × 表征防御 × 训练对齐" 三联拍。6-23 AgentLens 解决白盒表征防御（subspace 干预，ASR 86 → 13.4 pp）；6-25 SkillSafetyBench 定义新攻击面（skill-mediated）；今日 AgentDoG 1.5 提供端到端 alignment 框架（数据净化 + RL 训练 + 在线护栏）。三篇形成 H1 末 agent safety 的完整工程谱系。

对你三个研究方向的启发
Harness Engineering

AgentDoG 1.5 的"FSM 替代 Docker" 思想直接相关你 KV-cache / SGLang 毕设：(1) 训练环境 == 数据生成器 + reward judge 的二元抽象——你做 KV-cache 策略搜索时不需要真的跑 vLLM 全栈，而是可以用 FSM 模拟请求序列 / KV 命中 / cache eviction，几个数量级降低 RL 训练成本；(2) 影响函数净化 思想可平移到 KV-cache 训练—— 给 cache 策略每条 trajectory 计算影响函数得分，只保留与系统性能梯度 最对齐的样本；(3) 三维分类法（来源 × 模式 × 危害） 是设计 KV-cache benchmark 的极好模板—— 把 KV-cache 失败拆成"触发来源（workload pattern） × 失败模式（eviction-miss / sparse-mismatch / numerical-error） × 业务影响（latency / throughput / quality）"；(4) 训练免费在线护栏 思想—— KV-cache 系统也需要训练免费的运行时监控（如不依赖再训练的 sparse pattern detector），这是 SGLang / vLLM 当前明显的缺口；(5) 49 作者 / 多机构合作模式—— 是中国 AI Safety 工程化的范本，与今日 NOVA 腾讯工业落地形成"学界基础设施 + 业界部署" 双门。

Agent Skills Safety（直击本方向）

AgentDoG 1.5 是 H1 末 agent safety 链条的"集大成者"——它把 6-23 AgentLens / 6-25 SkillSafetyBench 的散点能力系统化、规模化、工程化。统一启发：(1) 分类法是基础设施—— 三维分类法（15×21×11）是后续所有评测 / 训练 / 部署的共享语言，意味着 agent safety 论文必须先承诺一个分类法 vocabulary；(2) OpenClaw / Codex 新风险叶子 是 2026 H2 的关键研究增长点—— 跨会话身份模糊、持久内存污染、技能/插件供应链、仓库工件注入 等位置，是 jailbreak 之外的结构化攻击面，与 SkillSafetyBench 的 skill-mediated 视角 mutually reinforcing；(3) 训练成本下降 30+ 倍 意味着 H2 安全研究的入门门槛大幅降低——本科生可以用 ~1,000 样本和 4B 模型做出可发表的工作；(4) safety-utility Pareto 同时改善（BFCL 76 → 81，Refusal 28 → 75）的反例性数据直接反驳了"safety hurts capability" 的悲观叙事；(5) 与 AgentLens 形成"数据净化（这里）+ 表征干预（那里）" 的方法论双门，研究路线既可以走数据侧也可以走表征侧。

Safety Benchmark

AgentDoG 1.5 给 Safety Benchmark 提供了三条新原则：(1) benchmark 不应该只测 ASR / SR 单标量，必须三维报告（risk source × failure mode × real-world harm）—— ATBench 三维诊断让小模型反超大模型，正说明聚合标量隐藏了真实能力差异；(2) 训练-评测一体化—— FSM 训练环境同时也是 evaluation environment，避免 train/eval 分布偏差；(3) BFCL 通用能力同时上升 提示 safety benchmark 必须配置utility track，否则无法识别"过度保守" 的护栏；(4) 与 6-25 Hacker-Fixer Loop 形成姊妹—— 一个加固 verifier（外部），一个净化数据（内部），共同把 benchmark 从 leaderboard 升级为方法学基础设施；(5) 49 作者 / 多机构合作模式 是 H2 大规模 safety benchmark 的标准模板—— 单机构无法覆盖 5,973 工具与 MCP 服务器的多样性。
